Llama-3.1-8B-Instruct layer 16 · batch 32 · seq 18 · BFP8 KV cache · one Blackhole P300 · single traced decode step. Both configs differ only by advisor_exact_residual_chain.

Provenance — what optimized took from the advisor, and how sure we are

There is no separate "pre-optimized" git commit — the file was committed whole. The version before the advisor is preserved as the dram_sharded candidate still in test_optimized_decoder.py. Diffing it against optimized in code gives the attribution below. The distinction that matters: did the advisor cause the choice, or does the value just coincide with what was already there?
