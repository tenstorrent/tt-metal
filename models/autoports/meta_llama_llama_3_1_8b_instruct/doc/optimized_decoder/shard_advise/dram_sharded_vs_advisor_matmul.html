DRAM-sharded vs. advisor_1d decode matmuls

Llama-3.1-8B-Instruct layer 16 · batch 32 · seq 18 · BFP8 KV cache · one Blackhole P300. Both configs are all-BFP4/LoFi with the 32-way residual chain; they differ only by decode_matmul_strategy. This isolates strategy at the shipped precision — the combination the bring-up log never tested.

At matched precision, DRAM-sharded (32-core) is faster: 0.6965 ms vs advisor_1d 0.7101 ms — ~2% / 14 µs, reproducible, identical accuracy.

What the bring-up log did — and missed

The log did tweak DRAM-sharded knobs (work_log:156–159): all-role core geometry 32/16/8, down-only and gate/up geometry, in0_block_w=8, and all-BFP8/HiFi2. Its best DRAM-sharded decode was 0.803 ms, so it kept advisor_1d. But every dram sweep ran at the dram precision (BFP8 attention/O/down) — so QKV read 2× the bytes and the strategy never got a fair shot. It compared dram@BFP8 (0.803) against advisor@BFP4 (0.713–0.755): precision confounded with strategy. Running dram_sharded at the shipped all-BFP4 precision (this page) flips the result.

This is a greedy-search artifact: at original precision advisor_1d (0.827) beat the DRAM-sharded rewrite (0.895), so advisor_1d was "retained as search base" (work_log:131) and every later precision reduction was layered only onto it — the strategy was frozen before the precision that flips it. It also runs against the governing rule the log cites: optimize/SKILL.md OPT-015 states the shard-advise 1D-mcast config is "one candidate … not a DRAM-sharded replacement: it regresses DRAM-BW-saturated decode matmuls and wins mainly on few-core, BW-starved, or interleaved baselines — measure it against the DRAM-sharded candidates and keep the winner (OPT-004)." The advisor QKV at 27% BW is exactly the case OPT-004 says DRAM-sharded should win; the log kept the 1-D config anyway because its A/B was stale.

Honest caveats

  • Margin is small (~14 µs / 2%) but reproducible and in the opposite direction from the log's conclusion.
  • Only 32-core dram wins; 16-core is tied and 8-core fails — so the result is geometry-sensitive.
  • Single-replay device totals (690 vs 686 µs) carry per-op jitter on small ops; the authoritative signal is the 10/100 wall-clock and the stable matmul device times.
  • Scope: layer 16, batch 32, seq 18, BFP8 cache, single device. Not swept across batch/seq/multichip.

Reproduce: add final_candidate.with_changes(decode_matmul_strategy="dram_sharded") as a policy and run test_optimized_decoder_perf / test_candidate_executes_real_optimized_path. Raw reports: optimized_full.txt, dram_all_bfp4_full.txt.
