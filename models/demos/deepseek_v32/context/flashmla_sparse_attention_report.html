FlashMLA sparse attention

DeepSeek's deployed DSA attention kernels: flash_mla_sparse_fwd (prefill, bf16) and flash_mla_with_kvcache(indices=…) (decode, fp8 paged cache). Sources: flash_mla/flash_mla_interface.py, csrc/sm90/prefill/sparse/phase1.cuh, tests/ref.py, docs/20250929-hopper-fp8-sparse-deep-dive.md. The contract that matters: top-k indices replace the mask — attention runs over κ gathered tokens, never length T; complexity O(S·κ·H·D).

1. API

No causal flag: causality is already inside the indices (the indexer only proposes past tokens).

Decode: flash_mla_with_kvcache(q, k_cache, indices=…, topk_length=…, attn_sink=…, extra_k_cache=…)

q [B, s_q, H, 576]; fp8 paged cache [num_blocks, 64, 1, 656 B]; indices [B, s_q, κ] flat token id = page·block_size + offset (block_table not needed); returns out [B, s_q, H, 512] + lse. Split-K over κ across SMs; metadata reused via FlashMLASchedMeta. extra_k_cache/extra_indices attend to a second cache.

512 × fp8 latent NoPE — 4 tiles of 128
4 × fp32 scale
64 × bf16 RoPE (never quantized)

656-byte fp8 token format. In-kernel: fp8→bf16 dequant (×scale) then bf16 MMA for QK and PV.

2. Semantics (tests/ref.py)

$$ P_{s,h,j}=\sigma\,\langle q_{s,h},\,kv[\,\mathrm{idx}_{s,j}\,]\rangle,\quad P_{s,h,j}=-\infty\ \text{if idx invalid};\qquad O = \mathrm{softmax}_j(P)\cdot kv[\mathrm{idx}][:512],\ \ \mathrm{lse}=\log\!\sum_j e^{P} $$
g = kv[indices.clamp(0)]                    # [S, κ, 576] — gather
P = q.float() @ g.mT * sm_scale; P[invalid] = -inf
out = softmax(P, -1) @ g[..., :512]; lse = logsumexp(P, -1)

3. Kernel (SM90 prefill): gather is the producer, attention never sees T

Grid = S·(H/64) CTAs (one per query token × 64-head half); 3 warpgroups, κ consumed in B_TOPK = 64-token blocks, double-buffered:

# producer WG2 (128 thr = 16 groups × 8): replaces TMA — KV is gathered, not contiguous
for blk in topk_blocks(κ, 64, step=2):
  t = ldg(indices[s, blk*64 + row]); valid = 0 <= t < T            # int gather
  cp_async_128b(smem[buf][row], kv + t*stride, pred=valid)          # 2 bufs × 9 tiles (64×64)
  smem.is_valid[buf][row] = valid; mbarrier.arrive()

# consumers WG0+WG1 (64 heads, head-dim split, sM/sS exchanged via smem)
for blk in topk_blocks:
  P  = wgmma(sQ, sK[blk])                                # [64h, 64tok]
  P  = valid ? P*scale : -inf                            # mask, base-2 exp
  m,l,O = online_softmax_rescale(P, m, l, O)             # classic flash recurrence
  O += wgmma(S, sV_local) + wgmma(sS_other, sV_remote)   # V = K[:512], split L/R
out = O / l;  lse = log l + m;  store(out, lse, max_logits)
Invalid slots become −∞ before softmax (zero weight, exact); the predicate also kills the load (no IMA). Decode adds: fp8 dequant on CUDA cores is slower than the MMAs, so two 64-head CTAs in a cluster each dequant half the tokens and swap via distributed smem (crossover) — 410 TFLOPS vs 250 without. Plus split-K over κ with lse-merge.

4. Pipeline (steady state, prefill)

WG2: ldg idx + cp.async gather
WG0/1: QKᵀ wgmma
mask→softmax→rescale (sM swap)
P·V wgmma (local + remote S)
gather QK softmax PV
