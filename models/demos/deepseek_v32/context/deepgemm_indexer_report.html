DeepGEMM fused indexer — fp8_mqa_logits

The deployed GPU implementation of the DeepSeek-V3.2 “lightning indexer” score (I[s,t] = Σ_h relu(q·k)·w · s_k). Sources: csrc/apis/attention.hpp (host API), deep_gemm/include/deep_gemm/impls/sm90_fp8_mqa_logits.cuh (Hopper kernel; SM100/FP4 variants exist), reference semantics in tests/test_attention.py::ref_fp8_mqa_logits. DeepGEMM calls the op MQA logits: one shared key (MQA-style), many query heads, output is logits — top-k is not fused.

2. Semantics

$$ \mathrm{logits}[s,t]=\Big(\sum_{h=1}^{H}\mathrm{relu}\big(\langle q_{s,h},\,k_t\rangle\big)\cdot w_{s,h}\Big)\cdot s_k[t] \quad \text{for } ks_s \le t < ke_s,\ \ -\infty \text{ otherwise} $$
# exact reference (tests/test_attention.py)
score  = einsum('shd,td->hst', q.float(), k.float())
logits = (score.relu() * weights.T[:, :, None]).sum(dim=0) # [S,T]
logits = logits.masked_fill(~(ks[:,None] <= t) | ~(t < ke[:,None]), -inf)

3. Kernel: one fused GEMM with the head axis folded away

The fusion trick: \(QK^\top\) is launched as one GEMM with \(M = S\!\cdot\!H\), \(N = T\), \(K = D\) — heads stacked into rows — but heads never reach memory: relu·w·Σ over H runs in registers on the GEMM accumulator before the [BLOCK_Q × BLOCK_KV] tile is stored.

# persistent grid: each SM strides over query blocks; warp-specialized producer/consumer.
BLOCK_QH, BLOCK_KV = 128, 256;  BLOCK_Q = 128 // H

if warp.is_producer:                              # TMA warp
  for qb in myblocks(sm_idx, kNumSMs):
    tma(Q[qb], W[qb])                               # double-buffered Q stages
    win = [min(ks[qb]) // 4 * 4, max(ke[qb])]       # only the visible KV window!
    for kv in blocks(win, BLOCK_KV): tma(K[kv], s_k[kv])   # kNumKVStages-deep pipeline
else:                                              # math warpgroups
  for qb in myblocks(...):
    w = ld_shared(W[qb])
    for kv in window_blocks:
      acc = wgmma(K_smem, Q_smem)                       # [BLOCK_KV, BLOCK_Q*H] (KV=M, QH=N)
      logit = relu(acc).mul(w).sum(heads) * s_k         # registers + 2 shfl_xor
      if ks[s] <= t < ke[s]: logits[s, t] = logit     # predicated store
A/B swap: KV is the M-side, Q·H the N-side, so each warp owns 16 KV rows; the H heads of one query land in one lane → head reduction = in-thread adds + 2 shfl_xor, no smem trip. Window: the GEMM loop runs only over [min ks, max ke) of the block — DSA causality/sparsity becomes skipped tiles (TFLOPS scales with valid keys); −∞ fill is a separate tiny clean_logits kernel.

5. Where the op sits in the indexer diagram (§A.3)

Assume fp8_fp4_mqa_logits exists. It replaces the entire score block of the indexer diagram — the fan-in of the three lanes plus masking. Everything upstream stays the projection front-end (TT or PyTorch); top-k stays downstream. Batch is folded: rows are B·S.
