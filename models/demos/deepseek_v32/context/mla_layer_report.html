DeepSeek V3.2 MLA — the DSA layer as an op graph

Source: models/demos/deepseek_v32/tt/mla/mla.py (the ttMLA subclass) + tt/ops.py (thin wrappers over the now-landed C++ ops), exercised by tests/test_mla.py::test_v32_mla_vs_cpu_reference and …_chunked_…. V3.2 MLA = the v3 dense MLA (deepseek_v3_d_p) plus DeepSeek Sparse Attention (DSA): a lightning indexer proposes top-k keys, and ring SDPA is swapped for sparse_mla over just those keys. Nodes below are ops; edges are tensors labelled with their shapes; node colour is the distribution (see legend). Edges animate in the direction data flows.

Updated 2026-06-16 to reflect the device-op port (status.md Step 6) and the on-device gather (Step 7): the indexer is fully on device (stems, RoPE, key cache, indexer_score, topk_large_indices), sparse_mla is the ttnn.transformer.sparse_sdpa device op run SPMD per chip, and the latent prefix it consumes is now gathered entirely on device (no host readback) — with the indices SP-reshard done via ttnn.mesh_partition. The DSA path no longer touches the host. This report focuses on the chunked-prefill path.

The one structural fact. forward() is a router (mla.py:226). When the global end position is ≤ index_topk (2048) the selection is a no-op (every causal key fits in top-k), so it falls straight through to the unchanged v3 dense super().forward — and on the chunked path (kv_actual_isl passed) the indexer K-cache is still written (mla.py:231-232) so later chunks can score against the early prefix. Only when end_pos > 2048 does the real DSA path run: _indexer_topk → _dsa_forward.

Layout marker (orthogonal to colour). 🟫 on an edge = that tensor is ROW_MAJOR; unmarked = TILE. The DSA device ops are ROW_MAJOR (indexer_score emits RM logits → topk_large_indices RM uint32 indices → sparse_sdpa RM q/kv/idx/out); inside sparse_mla the absorbed q is converted TILE→RM on the way in and the output RM→TILE on the way out for the wkv_b2 linear. Edge labels carry the symbolic shape; the ↳ line below it is the concrete shape for the §7 runtime (50k cached + 5k chunk, sp=8, tp=4, 50k=50·1024, 5k=5·1024: S/sp=640, dim/tp=1792, H/tp=32, glob=5120, end_pos=56320, k=2048). Note the SP all-gathers in the indexer are real at sp=8 — per-device tensors are S/sp=640 until the gather, then jump to the global glob=5120.

The DSA path is now fully on device (status.md Step 6 + Step 7). The three ops in tt/ops.py that were host/composed workarounds are thin wrappers over C++ device ops: indexer_logits→ttnn.experimental.deepseek.indexer_score, topk_indices→ttnn.experimental.topk_large_indices, sparse_mla→ttnn.transformer.sparse_sdpa. Step 7 removed the last host hop: the latent prefix feeding sparse_mla is gathered on device (to_memory_config ND→interleaved → SP all_gather → typecast bf8→bf16 → to_layout TILE→RM → on-device block-cyclic un-rotate), and the indices SP-reshard uses ttnn.mesh_partition (the inverse of all-gather). No tensor leaves the device. Remaining C++ follow-up (perf, not correctness): the SP-ring + full device-residency that would avoid materializing the full-T replica (backlog 12) — see the dedicated kvpe_sparse_pipeline_report.html.

4. The indexer — selecting the top-k keys (_indexer_topk)

Three stems run off the same hidden_states: the K stem (cached, runs every chunk), the Q stem (reuses v3's q_a latent then the indexer's wq_b), and weights_proj (per-head score weights). Their dot product gives index logits; indexer_score fuses the causal mask, and topk_large_indices yields the indices. Fully on device — stems, RoPE, key cache, logits and top-k — no host (status.md backlog 6/19).

WeightLayout on deviceWhy

The score math (ops.indexer_logits): $$ \text{logits}[s,t] = \sum_{h} w[s,h]\cdot \mathrm{relu}\big(\langle q[s,h],\,k[t]\rangle\big) $$ The reference fuses this as DeepGEMM fp8_mqa_logits; the port now calls the merged ttnn.experimental.deepseek.indexer_score C++ op (bf16, no fp8/Hadamard yet) — causality is fused inside the op from chunk_start_idx (future/pad columns set to -inf), so the caller no longer adds a triu mask. Output is [1,1,Sq,Skv] bf16 ROW_MAJOR, chaining straight into topk_large_indices; the -inf columns survive there as the 0xFFFFFFFF sentinel (contiguous descending tail), which sparse_mla masks. Keys are shared across all H_idx heads (MQA), so a single _index_kbuf serves every head.

5. Sparse attention forward (_dsa_forward)

Structurally this is v3's forward with the ring SDPA replaced by ops.sparse_mla (now the ttnn.transformer.sparse_sdpa device op): Q is built absorbed (latent 512 ++ rope 64 = 576), KV is built + cached exactly as v3, then attention runs over only the k selected latents. Like v3's chunked path it keeps attention latent and applies wkv_b2 after. The chunked cache write is update_padded_kv_cache (block-cyclic across SP); the latent prefix for the op is then gathered and un-rotated to natural order on device (_gather_kvpe_prefix), with no host round-trip.

What sparse_mla actually does

FlashMLA sparse contract: no causal flag — the indices already encode causality (via the 0xFFFFFFFF sentinel; start_pos is vestigial/ignored). Per query row it gathers its k selected latents and runs a tiny absorbed-MQA online-softmax: $$ O_{h,s} = \mathrm{softmax}_j\!\Big(\text{scale}\cdot\langle q_{h,s},\,kv[\mathrm{idx}_{s,j}]\rangle\Big)\cdot kv[\mathrm{idx}_{s,j}][{:}512] $$ V is the first 512 dims of the same latent (MQA). This is now the merged ttnn.transformer.sparse_sdpa device op (Blackhole single-chip kernel), invoked SPMD across the SP×TP mesh: each chip runs the kernel over its own q shard, so q's distribution is preserved end-to-end — heads on TP (dim1), sequence on SP (dim2), output the same. Preconditions the producer guarantees: per-chip H a multiple of 32 (128/tp=4=32), indices uint32 with sentinels as a contiguous tail, every row ≥1 valid key, all valid indices <T. The op is 🟫 ROW_MAJOR-only: the wrapper converts the absorbed q TILE→RM in and converts the latent output RM→TILE out for wkv_b2. Both the kv prefix and the indices SP-reshard are produced on device (the latent gather of §5's graph; ttnn.mesh_partition for indices) — no host hop. The remaining C++ follow-up is perf only: an SP-ring + full device-residency that avoids materializing the full-T replica (backlog 12).

7. Worked example — 50k cached, processing a fresh 5k chunk

Concrete trace with kv_actual_isl = 51200 (50·1024), this chunk = 5120 tokens (5·1024), on an 8×4 mesh (sp=8, tp=4). Because kv_actual_isl is passed, this is the chunked path. (The code's DSA path is validated at sp=1 (1×4, single-shot + chunked) and sp=2 (2×2 single-shot); 2×2 chunked is currently guarded back to 1×4 over the open block-cyclic read issue (status.md 5.5). sp=8 here is illustrative — it exercises the general SP wiring, where the indexer all-gathers and the indices SP-reshard become real, not no-ops.)

The selection, in one line. That gap is DeepSeek Sparse Attention; it widens with context (at 50k it is already 27.5×, at 128k it is ~64×).
