## Key Findings

1. **Time conv (384→768) baselines were catastrophically bad.** The old default `(32,32,1,1,1)` used C_out_block=1 (single channel per iteration). Optimal blockings use C_out_block=96-384, giving **19-635x speedups**.

2. **Smaller C_out_block helps smaller shapes.** On 480p uncached at latent resolution (15×13), C_out_block=32 beats 128 for 384→384 (2.27x).

3. **Optimal blockings are mesh-specific.** The best (H_out_block, W_out_block) depends on per-device spatial dims which change with mesh factoring.

4. **720p baselines are already near-optimal for large shapes.** At 96→96 and 96→32 with large spatial dims (186×162), the baseline (96,96,1,8,8) is already optimal or within 1%.

5. **Non-power-of-2 spatial values cause device hangs.** Values like 13, 20, 23, 26, 52 hang the conv3d kernel on BH p150b. Only power-of-2 spatial blocks are safe.
