## SFPU Kernel Implementation
This section provides a dedicated deep dive into the underlying SFPU kernel function that the compute kernel dispatches to.

### Unary Dispatch Summary
- **UnaryOpType**: `HARDSIGMOID`
- **Compute kernel**: `ttnn/cpp/ttnn/operations/eltwise/unary_ng/device/kernels/compute/eltwise_sfpu.cpp` (default path via `unary_ng` factory)
- **SFPU_OP_CHAIN_0 expansion**: `hardsigmoid_tile_init(); hardsigmoid_tile(0);`
- **Dispatch path**: `unary_ng` (not old `unary`). `HARDSIGMOID` is registered in `unary_ng_op_utils.cpp` line 117, not in the old `unary_op_utils.cpp`.

### Call Chain

1. **`hardsigmoid_tile(idst)`** (API header `activations.h:27-29`): Wraps the call in the `MATH(...)` macro (active only on TRISC_MATH). Calls `llk_math_eltwise_unary_sfpu_hardsigmoid<APPROX, ckernel::ActivationType::Hardsigmoid>(idst)`.

2. **`llk_math_eltwise_unary_sfpu_hardsigmoid<APPROX, Hardsigmoid, 8>(dst_index)`** (LLK dispatch `llk_math_eltwise_unary_sfpu_activations.h:22-27`): Casts the SFPU function to a `void (*)()` function pointer and delegates to `_llk_math_eltwise_unary_sfpu_params_<APPROX>(...)`, passing the cast pointer to `_calculate_activation_<APPROX, ActivationType::Hardsigmoid, 8>`, `dst_index`, and default `VectorMode::RC`.

3. **`_llk_math_eltwise_unary_sfpu_params_<APPROX>(...)`** (params dispatch `llk_math_eltwise_unary_sfpu_params.h:13-73`): Sets DEST write address, sets address mode base, stalls SFPU until MATH is ready. In `VectorMode::RC`, loops over 4 faces calling the SFPU function once per face, with `TTI_SETRWC` (WH) or `_llk_math_eltwise_unary_sfpu_inc_dst_face_addr_()` (BH) to advance between faces.

4. **`_calculate_activation_<APPROX, ActivationType::Hardsigmoid, 8>()`** (core SFPU `ckernel_sfpu_activations.h:86-97`): Loops 8 iterations per face. Each iteration loads `dst_reg[0]` into a `vFloat`, calls `ActivationImpl<APPROX, Hardsigmoid>::apply(v)`, stores the result back, and increments `dst_reg`.

5. **`ActivationImpl<APPROX, ActivationType::Hardsigmoid>::apply(v)`** (core SFPU `ckernel_sfpu_activations.h:50-57`): Computes `tmp = v * vConstFloatPrgm0 + vConstFloatPrgm1` (i.e. `x/6 + 0.5`) then clamps to [0, 1] via `_relu_max_body_(tmp, 1.0f)` from `ckernel_sfpu_relu.h`.

**Init path**: `hardsigmoid_tile_init()` → `llk_math_eltwise_unary_sfpu_hardsigmoid_init<APPROX>()` → `llk_math_eltwise_unary_sfpu_init<SfpuType::hardsigmoid, APPROX>(_init_hardsigmoid_<APPROX>)`. The generic init calls `_llk_math_eltwise_unary_sfpu_init_<SfpuType::hardsigmoid>()` (configures SFPU config reg, address modes, resets counters), then calls `_init_hardsigmoid_<APPROX>()` which loads slope=1/6 into `vConstFloatPrgm0` and offset=0.5 into `vConstFloatPrgm1`.

### Parameters Dispatch Summary

- **Vector mode**: `VectorMode::RC` (default, processes all 4 faces of the tile)
- **Operation invocation**: The params dispatch loops over 4 faces. For each face, it calls the SFPU function (which internally loops 8 iterations), then advances the DEST address by one face stride. On Wormhole, face advance is `TTI_SETRWC(CLR_NONE, CR_D, 8, 0, 0, SET_D)` twice (each advances 8 sfpi rows = 16 physical DEST rows). On Blackhole, face advance is `_llk_math_eltwise_unary_sfpu_inc_dst_face_addr_()` which calls `math::inc_dst_addr<8>()` twice.
- **DEST address progression**: Standard DEST progression (ITERATIONS=8 per face, `dst_reg++` per iteration, SETRWC/inc_dst_addr between faces). Within each face iteration, `dst_reg++` advances 1 sfpi row (2 physical DEST rows, 32 elements). The address mode for SFPU is `ADDR_MOD_7` with `dest.incr = 0` (addressing is managed by `dst_reg++` in SFPI code, not by hardware auto-increment). No special `ADDR_MOD_6` is configured for hardsigmoid (the `SfpuType::hardsigmoid` case does not match any of the special `if constexpr` conditions in `eltwise_unary_sfpu_configure_addrmod`).

### Annotated SFPU Kernel Source

The hardsigmoid kernel uses SFPI abstractions (`sfpi::vFloat`, `dst_reg`, `v_if`/`v_endif`), so Style A applies.

```cpp
// File: tt_metal/third_party/tt_llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_activations.h

// Specialization for HARDSIGMOID activation
template <bool APPROXIMATION_MODE>
struct ActivationImpl<APPROXIMATION_MODE, ActivationType::Hardsigmoid> // APPROXIMATION_MODE=false (unused by this impl)
{
    static inline void apply(sfpi::vFloat& v)
    {
        // tmp = x * (1/6) + 0.5  — linear transform for hardsigmoid
        // vConstFloatPrgm0 = 1/6 = 0.1666666716337204f, vConstFloatPrgm1 = 0.5f (set by _init_hardsigmoid_)
        sfpi::vFloat tmp = (v * sfpi::vConstFloatPrgm0) + sfpi::vConstFloatPrgm1; // → SFPMAD (mul), SFPMAD (add)
        v                = _relu_max_body_(tmp, 1.0f); // clamp to [0, 1]
    }
};

// Generic activation dispatch loop (parameterless variant used by hardsigmoid)
template <bool APPROXIMATION_MODE, ActivationType ACTIVATION_TYPE, int ITERATIONS>
inline void _calculate_activation_() // APPROXIMATION_MODE=false, ACTIVATION_TYPE=Hardsigmoid, ITERATIONS=8
{
#pragma GCC unroll 8
    for (int d = 0; d < ITERATIONS; d++) // 8 iterations per face
    {
        sfpi::vFloat v = sfpi::dst_reg[0]; // SFPLOAD: load 32 elements from current DEST position
        apply_activation<APPROXIMATION_MODE, ACTIVATION_TYPE>(v); // calls ActivationImpl::apply
        sfpi::dst_reg[0] = v; // SFPSTORE: write result back to DEST
        sfpi::dst_reg++; // advance 1 sfpi row (2 physical DEST rows = 32 elements)
    }
}

// Init function: loads constants into programmable SFPU registers
template <bool APPROXIMATION_MODE>
void _init_hardsigmoid_()
{
    // For hardsigmoid slope is 1/6, FP32 IEEE 754 representation.
    sfpi::vConstFloatPrgm0 = 0.1666666716337204f; // slope = 1/6
    sfpi::vConstFloatPrgm1 = 0.5f;                // offset = 0.5
}
```

```cpp
// File: tt_metal/third_party/tt_llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_relu.h

// Helper: clamp value to [0, threshold]
sfpi_inline sfpi::vFloat _relu_max_body_(sfpi::vFloat val, sfpi::vFloat threshold) // threshold = 1.0f
{
    sfpi::vFloat result = val;
    v_if (result > threshold) // CC set by comparing result vs threshold (1.0f)
    {
        result = threshold; // clamp high: result = 1.0
    }
    v_endif;
    v_if (result < 0.0f) // CC set by comparing result vs 0.0
    {
        result = 0.0f; // clamp low: result = 0.0
    }
    v_endif;
    return result;
}
```

### SFPU Instructions Used

The kernel uses SFPI C++ abstractions which compile to the following underlying SFPU instructions:

| SFPI Abstraction | Emitted Instruction(s) | Description |
|------------------|----------------------|-------------|
| `sfpi::dst_reg[0]` (read) | **SFPLOAD** | Load 32 elements (2 physical rows) from current DEST position into LREG |
| `v * vConstFloatPrgm0` | **SFPMAD** | Multiply-accumulate: `v * (1/6) + 0.0`. There is no dedicated float multiply — it is `SFPMAD(a, b, 0)` |
| `... + vConstFloatPrgm1` | **SFPMAD** | Add via multiply-accumulate: `result * 1.0 + 0.5`. Float addition is `SFPMAD(a, 1.0, b)` |
| `v_if (result > threshold)` | **SFPSETCC** (implicit via comparison) | Set per-lane condition codes based on comparison result |
| `result = threshold` (conditional) | **SFPMOV** or **SFPLOADI** (CC-guarded) | Conditional assignment — only executed on lanes where CC is set |
| `v_if (result < 0.0f)` | **SFPSETCC** (implicit via comparison) | Set per-lane condition codes for negative check |
| `result = 0.0f` (conditional) | **SFPLOADI** (CC-guarded) | Load immediate zero into result on lanes where CC is set |
| `v_endif` | **SFPENCC** | Clear/restore condition code state |
| `sfpi::dst_reg[0] = v` (write) | **SFPSTORE** | Store 32 elements from LREG back to current DEST position |
| `sfpi::dst_reg++` | (address counter increment) | Advance DEST read/write pointer by 1 sfpi row (2 physical rows) |

### SFPU Register Usage

| Register | Usage |
|----------|-------|
| **DEST register** | Input/output tile data. Addressed via `dst_reg[0]`, advances by `dst_reg++` per iteration |
| **LREG0** | Working register — holds the current element vector (`v`). Loaded from DEST, manipulated, stored back |
| **vConstFloatPrgm0** | Programmable constant register — loaded with slope `1/6 = 0.1666666716337204f` during init |
| **vConstFloatPrgm1** | Programmable constant register — loaded with offset `0.5f` during init |
| **Condition Code (CC)** | Per-lane predication register — set by `v_if` comparisons (`>`, `<`), cleared by `v_endif`. Used to gate the clamping assignments |

### Address Mode Configuration

The address mode for hardsigmoid is configured during `_llk_math_eltwise_unary_sfpu_init_<SfpuType::hardsigmoid>()`, which calls `eltwise_unary_sfpu_configure_addrmod<SfpuType::hardsigmoid>()`.

Since `SfpuType::hardsigmoid` does not match any of the special-case `if constexpr` conditions (only `topk_local_sort`, `typecast`, `unary_max/min` have special address modes), the only address mode configured is the default:

**ADDR_MOD_7** (both Wormhole and Blackhole):
```
addr_mod_t {
    .srca = {.incr = 0},
    .srcb = {.incr = 0},
    .dest = {.incr = 0},
}
```

This means **no hardware auto-increment** on DEST addresses. All DEST address progression is handled explicitly by the SFPI `dst_reg++` abstraction within the kernel loop, and by `TTI_SETRWC` (WH) or `math::inc_dst_addr<8>()` (BH) between faces in the params dispatch.

The configuration is identical between Wormhole and Blackhole hardware generations.
