## SFPU Kernel Implementation
This section provides a dedicated deep dive into the underlying SFPU kernel function that the compute kernel dispatches to.

### Unary Dispatch Summary
- **UnaryOpType**: `LGAMMA`
- **Compute kernel**: `ttnn/cpp/ttnn/operations/eltwise/unary/device/kernels/compute/eltwise_sfpu.cpp`
- **SFPU_OP_CHAIN_0 expansion**: `lgamma_tile_init(); lgamma_tile(0);`

#### Approximation Mode Resolution
There are TWO independent controls for approximation. Both are reported below.

| Control | Value | Source |
|---------|-------|--------|
| `math_approx_mode` (ComputeConfig) | `false` | `get_op_approx_mode(LGAMMA)` in `unary_op_utils.cpp` -- falls through to `default: return false` |
| Template parameter (SFPU_OP_CHAIN) | none (uses `APPROX` directly) | `get_op_init_and_func()` returns `"lgamma_tile_init();"` and `"lgamma_tile({idst});"` -- no parameterized template argument |
| Effective SFPU path | `APPROXIMATION_MODE=false`; the `_sfpu_reciprocal_` helper is instantiated with `max_iter=1` (hardcoded, independent of APPROXIMATION_MODE); `_calculate_log_body_no_init_` has no APPROXIMATION_MODE branching | `lgamma_tile()` calls `llk_math_eltwise_unary_sfpu_lgamma<APPROX>()` where `APPROX=false` is generated by `genfiles.cpp` from `math_approx_mode`. The `calculate_lgamma` function is templated on `APPROXIMATION_MODE` but the current implementation does not branch on it. |

### Call Chain
1. `SFPU_OP_CHAIN_0` expands to `lgamma_tile_init(); lgamma_tile(0);` in `eltwise_sfpu.cpp`.
2. `lgamma_tile(idst)` (in `lgamma.h`) calls `llk_math_eltwise_unary_sfpu_lgamma<APPROX>(idst)` guarded by `MATH((...))`.
3. `llk_math_eltwise_unary_sfpu_lgamma<APPROXIMATE>` (in the LLK dispatch header) calls `_llk_math_eltwise_unary_sfpu_params_<APPROXIMATE>(ckernel::sfpu::calculate_lgamma<APPROXIMATE, ITERATIONS>, dst_index, vector_mode)`.
4. `_llk_math_eltwise_unary_sfpu_params_` (in the params dispatch) sets up DEST addressing, stalls for SFPU, iterates over faces in `VectorMode::RC` mode (4 faces), and for each face calls `calculate_lgamma<false, 8>()`.
5. `calculate_lgamma` (in `ckernel_sfpu_lgamma.h`) executes the Lanczos approximation using `_sfpu_reciprocal_<1>` (from `sfpu/ckernel_sfpu_recip.h`) and `_calculate_log_body_no_init_` (from `sfpu/ckernel_sfpu_log.h`).

### Parameters Dispatch Summary

- **Vector mode**: `VectorMode::RC` (default) -- all 4 faces of the tile are processed.
- **Operation invocation**: The params dispatch calls `calculate_lgamma<false, 8>()` once per face, for 4 faces total. Each call processes 8 SFPI iterations (one face of 256 elements).
- **DEST address progression**: Standard DEST progression (ITERATIONS=8 per face, `dst_reg++` per iteration, `SETRWC` between faces). On Wormhole, `set_addr_mod_base()` selects ADDR_MODs 4-7 (base=1), and `ADDR_MOD_7` is configured with `dest.incr=0` (SFPI manages its own addressing via `dst_reg++`). On Blackhole, the same `ADDR_MOD_7` with `dest.incr=0` is used. Between faces, `_llk_math_eltwise_unary_sfpu_inc_dst_face_addr_()` advances by 16 physical DEST rows (two `inc_dst_addr<8>()` calls).

### Annotated SFPU Kernel Source

This kernel uses **Style A: SFPI-based kernel** -- it exclusively uses SFPI abstractions (`vFloat`, `dst_reg`, `v_if`/`v_endif`).

The lgamma kernel depends on two shared helper functions:
1. `_sfpu_reciprocal_<1>` from `ckernel_sfpu_recip.h` -- computes reciprocal with 1 Newton-Raphson iteration
2. `_calculate_log_body_no_init_` from `ckernel_sfpu_log.h` -- computes natural log without requiring programmable constant init

**Note on architecture differences**: The Wormhole and Blackhole `ckernel_sfpu_lgamma.h` files are identical. However, the shared `_sfpu_reciprocal_<1>` helper differs between architectures:
- **Wormhole**: Software Newton-Raphson using `setman`/`SFPNOT`/`SFPMAD` chain
- **Blackhole**: Hardware-accelerated `approx_recip()` (`SFPARECIP` instruction) followed by Newton-Raphson refinement

The `_calculate_log_body_no_init_` function is identical across both architectures.

```cpp
// File: tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_lgamma.h

// lgamma(x) = ln(|Gamma(x)|)
// Implementation notes, see the original file for more details
template <bool APPROXIMATION_MODE, int ITERATIONS = 8>
inline void calculate_lgamma() { // APPROXIMATION_MODE=false, ITERATIONS=8
    constexpr float half_ln_2pi = 0.918938531357171f;

#pragma GCC unroll 0
    for (int d = 0; d < ITERATIONS; d++) {
        sfpi::vFloat x = sfpi::dst_reg[0]; // SFPLOAD: load 32 elements from DEST

        // Lanczos series: 1 + c1/x + c2/(x+1) + c3/(x+2) + c4/(x+3)
        sfpi::vFloat series = sfpi::vConst1; // series = 1.0
        series = series + 76.18009172947146f * _sfpu_reciprocal_<1>(x); // SFPMAD chain + reciprocal
        series = series + -86.50532032941677f * _sfpu_reciprocal_<1>(x + sfpi::vConst1); // x+1 via SFPMAD, then recip
        series = series + 24.01409824083091f * _sfpu_reciprocal_<1>(x + 2.0f); // x+2 via SFPMAD, then recip
        series = series + -1.231739572450155f * _sfpu_reciprocal_<1>(x + 3.0f); // x+3 via SFPMAD, then recip

        sfpi::vFloat t = x + 4.5f; // SFPMAD (x * 1.0 + 4.5)
        sfpi::vFloat log_t = _calculate_log_body_no_init_(t); // natural log of t
        sfpi::vFloat log_series = _calculate_log_body_no_init_(series); // natural log of series

        // result = (x - 0.5) * log(t) - t + 0.5*ln(2*pi) + log(series)
        sfpi::vFloat result = (x - 0.5f) * log_t - t + half_ln_2pi + log_series; // chain of SFPMAD instructions

        // Special cases: lgamma(1) = 0, lgamma(2) = 0
        v_if(x == sfpi::vConst1) { result = 0.0f; } // CC-guarded: SFPSETCC(EQ), SFPLOADI(0.0), SFPSTORE
        v_endif;
        v_if(x == 2.0f) { result = 0.0f; } // CC-guarded: SFPSETCC(EQ), SFPLOADI(0.0), SFPSTORE
        v_endif;

        sfpi::dst_reg[0] = result; // SFPSTORE: write 32 elements back to DEST
        sfpi::dst_reg++; // advance DEST pointer by 1 sfpi row (2 physical rows)
    }
}

template <bool APPROXIMATION_MODE>
inline void lgamma_init() { // APPROXIMATION_MODE=false
    _init_sfpu_reciprocal_<APPROXIMATION_MODE>(); // Programs vConstFloatPrgm0/1/2 for reciprocal (WH only)
}
```

**Helper: `_sfpu_reciprocal_<1>` (Wormhole B0)**

```cpp
// File: tt_metal/third_party/tt_llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_recip.h

template <int max_iter = 2>
sfpi_inline sfpi::vFloat _sfpu_reciprocal_(const sfpi::vFloat in) { // max_iter=1 (called with <1>)
    // Scale input to [1,2) range with negative sign via SFPSETMAN
    sfpi::vFloat negative_x = sfpi::setman(sfpi::vConstNeg1, sfpi::reinterpret<sfpi::vInt>(in));

    // Quadratic initial estimate: y = k2 - k1*x + k0*x**2
    sfpi::vFloat y = sfpi::vConstFloatPrgm1 + sfpi::vConstFloatPrgm0 * negative_x; // SFPMAD

    // Scale factor via SFPNOT (bit inversion of exponent)
    sfpi::vInt scale_bits = ~sfpi::reinterpret<sfpi::vUInt>(in); // SFPNOT

    // Continue quadratic: y = k2 + y * negative_x
    y = sfpi::vConstFloatPrgm2 + y * negative_x; // SFPMAD

    // Clear mantissa of scale factor via SFPSETMAN
    sfpi::vFloat scale = sfpi::setman(sfpi::reinterpret<sfpi::vFloat>(scale_bits), 0);

    // Newton-Raphson iteration 1: t = 1.0 + negative_x * y
    sfpi::vFloat t = sfpi::vConst1 + negative_x * y; // SFPMAD

    // Scale adjustment: scale *= 0.5
    scale *= 0.5f; // SFPMAD

    // Continue NR: y = y + y * t
    y = y + y * t; // SFPMAD

    // max_iter=1, so second NR iteration is skipped

    // Apply scale and match input sign
    y = y * scale; // SFPMAD
    y = sfpi::setsgn(y, in); // SFPSETSGN

    return y;
}
```

**Helper: `_calculate_log_body_no_init_`**

```cpp
// File: tt_metal/third_party/tt_llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_log.h

sfpi_inline sfpi::vFloat _calculate_log_body_no_init_(sfpi::vFloat base) {
    // Normalize base to [1,2) range
    sfpi::vFloat x = setexp(base, 127); // SFPSETEXP: set exponent to bias (127)

    // 3rd order polynomial approx via Horner's method -- 4 SFPMAD instructions
    sfpi::vFloat series_result = x * (x * (x * 0x2.44734p-4f - 0xd.e712ap-4f) + 0x2.4f5388p+0f) - 0x1.952992p+0f;

    // Extract and convert exponent to float
    sfpi::vInt exp = exexp(base); // SFPEXEXP: extract exponent with de-bias
    v_if (exp < 0) { // CC: check sign of exponent
        exp = sfpi::setsgn(~exp + 1, 1); // negate: SFPNOT + SFPIADD(+1) + SFPSETSGN
    }
    v_endif;
    sfpi::vFloat expf = int32_to_float(exp, 0); // SFPCAST: int32 to float32

    // Combine: ln(x) = exp*ln(2) + series_result
    sfpi::vFloat vConstLn2 = 0.692871f; // SFPLOADI (loaded as immediate)
    sfpi::vFloat log_result = expf * vConstLn2 + series_result; // SFPMAD

    // Handle ln(0) = -inf
    v_if (base == 0.0f) { // CC: equality check
        log_result = -std::numeric_limits<float>::infinity(); // SFPLOADI(-inf)
    }
    v_endif;

    return log_result;
}
```

### SFPU Instructions Used

| Instruction | Usage in lgamma | Description |
|-------------|----------------|-------------|
| `SFPLOAD` | `dst_reg[0]` read | Load 32 elements from DEST row into LREG |
| `SFPSTORE` | `dst_reg[0] = result` write | Store LREG back to DEST row |
| `SFPMAD` | All `vFloat + vFloat` and `vFloat * vFloat` operations | Fused multiply-add; dominates this kernel due to Lanczos polynomial evaluation, reciprocal NR iterations, and log polynomial |
| `SFPLOADI` | Float constant loads (e.g., `76.18f`, `0.5f`, `4.5f`, `0.0f`, `-inf`) | Load 16-bit immediate to LREG (two instructions for 32-bit float) |
| `SFPSETMAN` | `setman()` in reciprocal helper | Set mantissa field of float, used to normalize input to [1,2) range and to clear mantissa of scale factor |
| `SFPNOT` | `~reinterpret<vUInt>(in)` in reciprocal helper | Bitwise NOT, used to compute `255 - exponent` for scale factor |
| `SFPSETSGN` | `setsgn(y, in)` in reciprocal helper | Set sign bit to match input sign |
| `SFPSETEXP` | `setexp(base, 127)` in log helper | Set exponent field to specified value, normalizing input to [1,2) |
| `SFPEXEXP` | `exexp(base)` in log helper | Extract exponent field with de-bias (subtract 127) |
| `SFPCAST` | `int32_to_float(exp, 0)` in log helper | Convert integer exponent to float for subsequent arithmetic |
| `SFPIADD` | `~exp + 1` (negation) in log helper | Integer add used for 2's complement negation of negative exponents |
| `SFPSETCC` | `v_if(x == ...)` conditions | Set condition code for equality comparisons (special cases x=1, x=2 and log base case x=0) |
| `SFPENCC` | `v_if`/`v_endif` expansion | Enable/disable condition code for predicated execution regions |
| `SFPCOMPC` | Implicit in `v_if`/`v_endif` expansion | Complement condition code (used internally by SFPI CC stack management) |
| `SFPPUSHC` | Implicit in nested `v_if` expansion | Push CC state for nested conditional regions |
| `SFPPOPC` | Implicit in `v_endif` expansion | Pop CC state to restore prior conditional context |

**Blackhole-specific additional instruction:**

| Instruction | Usage | Description |
|-------------|-------|-------------|
| `SFPARECIP` / `SFPNONLINEAR` | `approx_recip(x)` in BH reciprocal | Hardware-accelerated reciprocal approximation (~7-bit precision), followed by Newton-Raphson refinement |

### SFPU Register Usage

| Register | Usage |
|----------|-------|
| **DEST** | Source and destination for tile data. Each SFPI iteration accesses 32 elements (2 physical rows x 16 elements). `dst_reg[0]` loads input `x`, and `dst_reg[0] = result` stores the lgamma result back. |
| **LREG0-LREG3** | General-purpose registers used extensively for intermediate values. The kernel has high register pressure due to multiple live values: `x`, `series`, `t`, `log_t`, `log_series`, `result`, plus temporaries inside `_sfpu_reciprocal_` (which itself uses `negative_x`, `y`, `scale`, `t`) and `_calculate_log_body_no_init_` (which uses `x`, `series_result`, `exp`, `expf`, `log_result`). |
| **LREG4-LREG7** | Available as additional temporaries. LREG7 is used for indirect addressing mode by SFPMAD when `SFPMAD_MOD1_INDIRECT_VD` is set, but this kernel does not use indirect mode. |
| **vConstFloatPrgm0** | Initialized to `0.3232325f` (WH, reciprocal quadratic coefficient k0) or `2.0f` (BH, Newton-Raphson constant) by `_init_sfpu_reciprocal_<false>()` |
| **vConstFloatPrgm1** | Initialized to `1.4545459f` (WH, reciprocal quadratic coefficient k1) by `_init_sfpu_reciprocal_<false>()`; not programmed on BH |
| **vConstFloatPrgm2** | Initialized to `2.1212124f` (WH, reciprocal quadratic coefficient k2) by `_init_sfpu_reciprocal_<false>()`; not programmed on BH |
| **vConst1** | Fixed constant register = `1.0f`, used as the initial value of `series` and in Newton-Raphson (`t = 1.0 + ...`) |
| **vConstNeg1** | Fixed constant register = `-1.0f`, used in reciprocal as the base for `setman` to create negative normalized input |
| **vConst0** | Fixed constant register = `0.0f`, used on BH in reciprocal NR (addend for subtract-via-MAD) |

### Address Mode Configuration

The lgamma operation uses the standard SFPU unary address mode configuration:

| Hardware | ADDR_MOD | srca.incr | srcb.incr | dest.incr | Notes |
|----------|----------|-----------|-----------|-----------|-------|
| Wormhole B0 | `ADDR_MOD_7` (logical 3 + base=4) | 0 | 0 | 0 | SFPI manages DEST addressing via `dst_reg++`; `set_addr_mod_base()` shifts to ADDR_MODs 4-7 |
| Blackhole | `ADDR_MOD_7` | 0 | 0 | 0 | Same configuration; SFPI manages DEST addressing |

No special ADDR_MOD_6 configuration is set for lgamma (the `eltwise_unary_sfpu_configure_addrmod<SfpuType::lgamma>` template does not match any of the `if constexpr` branches that customize ADDR_MOD_6). The kernel relies entirely on SFPI's `dst_reg++` to advance through DEST rows within each face, and the params dispatch's `SETRWC` / `_llk_math_eltwise_unary_sfpu_inc_dst_face_addr_()` to advance between faces.
