## 3. Compute Kernel Analysis

TANHSHRINK has **two** dedicated compute kernel files in the codebase:

### 3.1 Primary Kernel: `tanhshrink_kernel.cpp` (FPU binary approach)

**File:** `ttnn/cpp/ttnn/operations/eltwise/unary/device/kernels/compute/tanhshrink_kernel.cpp`

**Strategy:** Uses FPU `binary_dest_reuse_tiles` with `DEST_TO_SRCB` for the subtraction step.

**Algorithm:**
1. Copy input tile from CB c_0 to DST register 0
2. Apply `tanh_tile(0)` - overwrites DST[0] with tanh(x)
3. Use `binary_dest_reuse_tiles<ELWSUB, DEST_TO_SRCB>` - loads DST[0] (tanh result) into SRCB, unpacks original input from CB c_0 into SRCA, computes SRCA - SRCB = x - tanh(x), result goes to DST[0]
4. Pack DST[0] to output CB c_2

**Key APIs used:**
- `init_sfpu(cb_input, cb_output)` - standard SFPU init
- `copy_tile_init(cb_input)` / `copy_tile(cb_input, 0, 0)` - unpack input to DST[0]
- `tanh_tile_init()` / `tanh_tile(0)` - SFPU tanh on DST[0]
- `binary_dest_reuse_tiles_init<ELWSUB, DEST_TO_SRCB>(cb_input)` - init FPU binary sub with dest reuse
- `binary_dest_reuse_tiles<ELWSUB, DEST_TO_SRCB>(cb_input, 0, 0)` - FPU subtract: input(SRCA) - tanh_result(SRCB from DST)

**CB layout:**
| CB | Index | Role |
|----|-------|------|
| c_0 | Input | Source tensor tiles |
| c_2 | Output | Result tiles (x - tanh(x)) |

**Tile processing loop:**
- Outer loop: `per_core_block_cnt` blocks
- Inner loop: `per_core_block_dim` tiles per block
- `cb_reserve_back` is called once per outer block (before inner loop)
- `cb_push_back` is called once per outer block (after inner loop)
- `cb_wait_front` and `cb_pop_front` are called per tile (inside inner loop)

### 3.2 Secondary Kernel: `tanhshrink_sfpu_kernel.cpp` (SFPU binary approach)

**File:** `ttnn/cpp/ttnn/operations/eltwise/unary/device/kernels/compute/tanhshrink_sfpu_kernel.cpp`

**Strategy:** Uses SFPU `sub_binary_tile` for the subtraction step, operating entirely within DST registers.

**Algorithm:**
1. Copy input tile from CB c_0 to DST register **1** (not 0!)
2. Apply `tanh_tile(1)` - overwrites DST[1] with tanh(x)
3. Copy input tile AGAIN from CB c_0 to DST register **0** (fresh copy of x)
4. Use `sub_binary_tile(0, 1, 0)` - computes DST[0] - DST[1] = x - tanh(x), result in DST[0]
5. Pack DST[0] to output CB c_2

**Key APIs used:**
- `init_sfpu(cb_input, cb_output)` - standard SFPU init
- `copy_tile_init(cb_input)` / `copy_tile(cb_input, 0, 1)` - unpack input to DST[1]
- `tanh_tile_init()` / `tanh_tile(1)` - SFPU tanh on DST[1]
- `copy_tile_init(cb_input)` / `copy_tile(cb_input, 0, 0)` - unpack input again to DST[0]
- `sub_binary_tile_init()` / `sub_binary_tile(0, 1, 0)` - SFPU binary sub: DST[0] - DST[1] -> DST[0]

**Additional includes:**
- `api/compute/eltwise_binary_sfpu.h` - provides `sub_binary_tile`
- `api/compute/copy_dest_values.h` - (included but not used in this kernel)

### 3.3 Comparison of Two Kernel Approaches

| Aspect | tanhshrink_kernel.cpp | tanhshrink_sfpu_kernel.cpp |
|--------|----------------------|---------------------------|
| Subtraction method | FPU binary with DEST_TO_SRCB | SFPU binary (`sub_binary_tile`) |
| DST slots used | 1 (DST[0]) | 2 (DST[0] and DST[1]) |
| Input reads from CB | 2 (once for copy, once for binary_dest_reuse) | 2 (two separate copy_tile calls) |
| Hardware unit for sub | FPU (matrix unit) | SFPU (vector unit) |
| Register pressure | Lower (1 DST slot) | Higher (2 DST slots) |
| Key advantage | Uses FPU for binary, more efficient data reuse | Fully SFPU path, simpler logic |

## 4. SFPU Instruction Usage

### 4.1 tanh_tile (Core SFPU Operation)

**Declaration:** `tt_metal/hw/inc/api/compute/compute_kernel_api.h:178`
```cpp
template <bool fast_and_approx = false>
ALWI void tanh_tile(uint32_t idst) {
    MATH((llk_math_eltwise_unary_sfpu_tanh<fast_and_approx, DST_ACCUM_MODE>(idst)));
}
```

Both kernel variants call `tanh_tile()` with `fast_and_approx = false` (default). This uses the precise tanh implementation.

**Init:** `tanh_tile_init()` must be called before `tanh_tile()`.

### 4.2 sub_binary_tile (SFPU Binary)

**Declaration:** `tt_metal/hw/inc/api/compute/eltwise_binary_sfpu.h:39`
```cpp
ALWI void sub_binary_tile(uint32_t idst0, uint32_t idst1, uint32_t odst) {
    MATH((llk_math_eltwise_binary_sfpu_binop<APPROX, ckernel::BinaryOp::SUB>(idst0, idst1, odst)));
}
```

Used in `tanhshrink_sfpu_kernel.cpp`. Takes two DST register indices and an output DST index. Computes `DST[idst0] - DST[idst1] -> DST[odst]`.

### 4.3 binary_dest_reuse_tiles (FPU Binary with Dest Reuse)

**Declaration:** `tt_metal/hw/inc/api/compute/eltwise_binary.h:248`
```cpp
template <EltwiseBinaryType eltwise_binary_type, EltwiseBinaryReuseDestType binary_reuse_dest>
ALWI void binary_dest_reuse_tiles(uint32_t in_cb_id, uint32_t in_tile_index, uint32_t dst_tile_index);
```

Used in `tanhshrink_kernel.cpp` with `ELWSUB` and `DEST_TO_SRCB`. The current DST value (tanh result) is loaded into SRCB, while SRCA comes from the CB (original input). Result: SRCA - SRCB = x - tanh(x).

## 6. Program Factory Integration

**File:** `ttnn/cpp/ttnn/operations/eltwise/unary/device/unary_program_factory.cpp`

The UnaryProgramFactory:
1. Creates CBs c_0 (input, 2 tiles) and c_2 (output, 2 tiles)
2. Creates reader/writer dataflow kernels (interleaved start_id variants)
3. Calls `utils::get_compute_kernel_path(ops_chain[0].type(), input.dtype())` to determine the compute kernel `.cpp` file
4. Creates the compute kernel with defines from `utils::get_block_defines()`
5. Distributes tiles across cores using `split_work_to_cores`

For tanhshrink, the compute kernel path resolution is critical:
- **Before nuke:** `get_compute_kernel_path(TANHSHRINK)` returned `"tanhshrink_kernel.cpp"` (the FPU binary variant)
- **After nuke:** Falls through to `default`, returning `"eltwise_sfpu.cpp"` (which fails because TANHSHRINK has no `get_op_init_and_func` entry either)

**Compile-time args passed to compute kernel:**
- `per_core_block_cnt` (arg 0) - number of tile blocks for this core
- `per_core_block_dim` = 1 (arg 1) - tiles per block (always 1 in this factory)

## 7. Backward Operation

**File:** `ttnn/cpp/ttnn/operations/eltwise/unary_backward/unary_backward.cpp:939-947`
```cpp
// tanhshrink
// result: torch.square(torch.tanh(input)) * grad_data
std::vector<Tensor> tanhshrink_bw(
    const Tensor& grad, const Tensor& input, const std::optional<MemoryConfig>& output_mem_config) {
    std::vector<Tensor> grad_tensor;
    Tensor tanh_res = ttnn::square(ttnn::tanh(input, output_mem_config), output_mem_config);
    grad_tensor.emplace_back(ttnn::multiply(grad, tanh_res, std::nullopt, output_mem_config));
    return grad_tensor;
}
```

The backward is a **composite** operation using `ttnn::tanh`, `ttnn::square`, and `ttnn::multiply`. It does NOT use the SFPU kernel directly.

Gradient formula: `d/dx[x - tanh(x)] = 1 - sech^2(x) = tanh^2(x)`, so: `grad_out = grad * tanh(x)^2`.

## 8. Implementation Notes for Re-Implementation

### 8.1 Two Valid Implementation Strategies

**Strategy A: SFPU-only (recommended for new implementation)**
Use the standard `eltwise_sfpu.cpp` dispatch via `SFPU_OP_CHAIN_0`. This requires:
1. Adding a `tanhshrink_tile_init()` / `tanhshrink_tile(idst)` pair in a new SFPU header
2. Implementing using `tanh_tile` + `sub_binary_tile` from `eltwise_binary_sfpu.h`
3. Registering in `get_op_init_and_func_default()` and `get_macro_definition()`

**Strategy B: Custom kernel (original design)**
Use a dedicated `tanhshrink_kernel.cpp` with:
1. Adding `case UnaryOpType::TANHSHRINK:` to `get_compute_kernel_path()`
2. The kernel handles its own copy/tanh/sub sequence directly

### 8.2 Key SFPU API Dependencies

The operation depends on these existing, working SFPU/compute APIs:
- `tanh_tile()` / `tanh_tile_init()` - from `compute_kernel_api.h` (always available)
- `sub_binary_tile()` / `sub_binary_tile_init()` - from `eltwise_binary_sfpu.h`
- `binary_dest_reuse_tiles<ELWSUB, DEST_TO_SRCB>()` - from `eltwise_binary.h`

### 8.3 No Dedicated SFPU Kernel Function

Unlike operations such as `sigmoid` or `relu` that have their own `ckernel_sfpu_*.h` files with raw SFPI instructions, tanhshrink does NOT have a dedicated low-level SFPU kernel function. It is composed entirely from existing primitives (`tanh` + subtraction).

There is no `ckernel_sfpu_tanhshrink.h` file. The operation is a **composite of existing SFPU/FPU primitives**, not a new SFPU instruction sequence.

### 8.5 Implementation Recommendation

To re-implement tanhshrink via the standard SFPU dispatch path (rather than restoring the custom kernel path), the implementor needs to:

1. **Add to `get_op_init_and_func_default()`** in `unary_op_utils.cpp`:
   - Create a `tanhshrink_tile_init()` and `tanhshrink_tile(idst)` function pair
   - Return them as the init/func strings

2. **Create a new SFPU header** (e.g., `api/compute/eltwise_unary/tanhshrink.h`):
   - Include `compute_kernel_api.h` (for `tanh_tile`)
   - Include `eltwise_binary_sfpu.h` (for `sub_binary_tile`)
   - Implement `tanhshrink_tile(idst)` as:
     ```cpp
     // Copy DST[idst] to DST[idst+1] (or another register)
     // Apply tanh to the copy
     // Subtract: DST[idst] = DST[idst] - DST[idst+1]
     ```

3. **Add split-include macro** in `sfpu_split_includes.h`:
   ```cpp
   #if SFPU_OP_TANHSHRINK_INCLUDE
   #include "api/compute/eltwise_unary/tanhshrink.h"
   #endif
   ```

4. **Register macro** in `get_macro_definition()`:
   ```cpp
   case UnaryOpType::TANHSHRINK: return "SFPU_OP_TANHSHRINK_INCLUDE";
   ```

Alternatively, the simpler approach is to restore the custom kernel path:
1. Add `case UnaryOpType::TANHSHRINK: return "tanhshrink_kernel.cpp";` to `get_compute_kernel_path()`

This is a one-line fix since both custom kernel files already exist in the codebase.
