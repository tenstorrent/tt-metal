# SFPU Kernel Analysis: `dropout`

## 1. Operation Overview

**Operation name**: dropout
**UnaryOpType enum value**: `DROPOUT` (index 104 in `unary_op_types.hpp`)
**Math definition**: For each element x with probability p, output 0; otherwise output x * scale where scale = 1/(1-p). This implements inverted dropout (scaling at train time so no scaling needed at inference).
**Category**: Infrastructure / stochastic operation (excluded from standard unary nuke — uses PRNG hardware)

### Unique Characteristics
Dropout is unique among SFPU operations because it:
1. Uses the SFPU's hardware **pseudo-random number generator (PRNG)** via `SFPMOV` with special mode bits
2. Requires a **seed initialization** step (`init_prng_seed`) before any tile processing
3. Takes **two parameters** (probability and scale factor) rather than the typical 0-1 parameters
4. Does **not** use the standard `UnaryProgramFactory` / `SFPU_OP_CHAIN_0` dispatch mechanism
5. Is implemented as a **standalone experimental operation** with its own program factory

---

## 2. Architecture Classification

**NOT a standard UnaryProgramFactory SFPU operation.** Despite having a `UnaryOpType::DROPOUT` enum entry:

- There is **no dispatch case** for `DROPOUT` in `unary_op_utils.cpp` (no `SFPU_OP_CHAIN_0` wiring)
- There is **no compute API header** `api/compute/eltwise_unary/dropout.h` in the source tree
- There is **no entry** in `sfpu_split_includes.h` for `SFPU_OP_DROPOUT_INCLUDE`
- The `DROPOUT` enum value exists as an **infrastructure placeholder** in `UnaryOpType`

Instead, dropout is implemented via two independent paths:
1. **`ttnn::experimental::prim::DropoutProgramFactory`** — full production implementation with its own program factory, circular buffers, and compute kernel
2. **LLK test kernel** (`tests/tt_metal/tt_metal/test_kernels/compute/dropout_sfpu.cpp`) — standalone test using `SFPU_OP_DROPOUT_INCLUDE` define

## 3. SFPU Kernel Function: `_calculate_dropout_`

### Source Locations
- **Wormhole B0**: `tt_metal/third_party/tt_llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_dropout.h`
- **Blackhole**: `tt_metal/third_party/tt_llk/tt_llk_blackhole/common/inc/sfpu/ckernel_sfpu_dropout.h`
- Both implementations are **identical**.

### Function Signature
```cpp
template <bool APPROXIMATION_MODE, int ITERATIONS>
inline void _calculate_dropout_(const int iterations, std::uint32_t probability, std::uint32_t scale)
```

### Template Parameters
| Parameter | Description |
|-----------|-------------|
| `APPROXIMATION_MODE` | Not used in the function body (template exists for interface consistency) |
| `ITERATIONS` | Not used in the function body; runtime `iterations` parameter is used instead |

### Runtime Parameters
| Parameter | Type | Description |
|-----------|------|-------------|
| `iterations` | `int` | Number of SFPU iterations (faces per tile, typically 4 for 32x32 tile with 32 SFPU lanes processing 8 rows each) |
| `probability` | `uint32_t` | Drop probability scaled to INT_MAX range: `probability = p * INT_MAX` |
| `scale` | `uint32_t` | Scale factor as bit-cast float32: `scale = bit_cast<uint32_t>(1.0f / (1.0f - p))` |

## 4. SFPU Instruction-Level Analysis

### Register Allocation
| Register | Purpose |
|----------|---------|
| `LREG0` | Input/output: holds tile data loaded from DST, then scaled result |
| `LREG1` | Scale factor (loaded as 32-bit float via two 16-bit halves) |
| `LREG2` | Probability threshold (loaded as 32-bit integer via two 16-bit halves) |
| `LREG3` | Random number generated by PRNG hardware |

### Instruction Sequence (per face iteration)

#### Phase 1: Load constant parameters (before loop, executed once)
```cpp
TT_SFPLOADI(p_sfpu::LREG1, 10, scale & 0xFFFF);      // Load low 16 bits of scale into LREG1
TT_SFPLOADI(p_sfpu::LREG1, 8, scale >> 16);           // Load high 16 bits of scale into LREG1
TT_SFPLOADI(p_sfpu::LREG2, 10, probability & 0xFFFF); // Load low 16 bits of probability into LREG2
TT_SFPLOADI(p_sfpu::LREG2, 8, probability >> 16);     // Load high 16 bits of probability into LREG2
```

#### Phase 2: Per-face loop body (4 iterations for standard 32x32 tile)
```
#pragma GCC unroll 0   // Prevent unrolling to keep code size small
```

**Step 2a: Scale the input value**
```cpp
TTI_SFPLOAD(p_sfpu::LREG0, 0, 3, 0);                                    // Load tile data from DST into LREG0
TTI_SFPMUL(p_sfpu::LREG0, p_sfpu::LREG1, p_sfpu::LCONST_0, p_sfpu::LREG0, 0); // LREG0 = LREG0 * LREG1 (input * scale)
```

**Step 2b: Generate random number**
```cpp
TTI_SFPMOV(0, 9, p_sfpu::LREG3, 8);                  // Generate PRNG uint32 into LREG3 (lreg_c=9, instr_mod1=8 triggers PRNG)
TTI_SFPSETSGN(0, p_sfpu::LREG3, p_sfpu::LREG3, 1);   // Clear sign bit for unsigned comparison with probability
```

**Step 2c: Conditionally zero out (drop) the element**
```cpp
TTI_SFPIADD(0, p_sfpu::LREG2, p_sfpu::LREG3, 10);    // Integer compare: sets CC if LREG3 < LREG2 (rand < probability)
TTI_SFPMOV(0, p_sfpu::LCONST_0, p_sfpu::LREG0, 0);   // Conditional move: if CC set, LREG0 = 0 (drop the element)
TTI_SFPENCC(0, 0, 0, 0);                               // End conditional context (clear CC flags)
TTI_SFPSTORE(0, 0, 3, 0);                              // Store LREG0 back to DST
```

**Step 2d: Advance to next face**
```cpp
sfpi::dst_reg++;   // Move DST pointer to next 8-row face
```

### Total Instructions Per Face: 8
- 1 SFPLOAD (read from DST)
- 1 SFPMUL (scale)
- 1 SFPMOV (PRNG generation)
- 1 SFPSETSGN (clear sign bit)
- 1 SFPIADD (compare)
- 1 SFPMOV (conditional zero)
- 1 SFPENCC (end conditional)
- 1 SFPSTORE (write to DST)

### Total Instructions Per Tile: 4 (setup) + 8 * 4 (faces) = 36 instructions

## 5. PRNG Initialization: `_init_dropout_`

### Source
```cpp
inline void _init_dropout_(const std::uint32_t seed) {
    init_prng_seed(seed);
}
```

### `init_prng_seed` implementation (from `ckernel.h`)
```cpp
inline void init_prng_seed(const std::uint32_t seed) {
    volatile std::uint32_t tt_reg_ptr *cfg = get_cfg_pointer();
    cfg[PRNG_SEED_Seed_Val_ADDR32] = seed;
    // Wait for PRNG to stabilize (600 NOPs)
    for (int i = 0; i < 600; i++) {
        TTI_SFPNOP;
    }
}
```

**Key details**:
- Writes the seed to a hardware configuration register (`PRNG_SEED_Seed_Val_ADDR32`)
- Requires **600 SFPNOP instructions** after seeding for the PRNG state to stabilize
- Must be called **once before any `_calculate_dropout_` calls**
- Same seed produces **deterministic** (repeatable) dropout masks — verified by tests

## 6. Compute Kernel Patterns

### Experimental Production Kernel
**Path**: `ttnn/cpp/ttnn/operations/experimental/dropout/device/kernels/compute/dropout_kernel.cpp`

```
Compile-time args: per_core_block_cnt, per_core_block_dim, int_probability, int_scale_factor
Runtime args: seed (index 0)
CBs: c_0 (input), c_2 (output)
Includes: api/compute/eltwise_unary/dropout.h (missing from source — likely a removed/unreachable header)
```

Pattern:
```
init_sfpu(c_0, c_2)
dropout_kernel_init(seed)
for each block:
    cb_reserve_back(c_2, block_dim)
    for each tile:
        tile_regs_acquire()
        cb_wait_front(c_0, 1)
        copy_tile(c_0, 0, 0)
        dropout_tile(0, probability, scale)
        tile_regs_commit()
        tile_regs_wait()
        pack_tile(0, c_2)
        cb_pop_front(c_0, 1)
        tile_regs_release()
    cb_push_back(c_2, block_dim)
```

### LLK Test Kernel
**Path**: `tests/tt_metal/tt_metal/test_kernels/compute/dropout_sfpu.cpp`

Uses `experimental::CircularBuffer` API and `SFPU_OP_DROPOUT_INCLUDE` define.
```
Compile-time args: per_core_block_cnt, per_core_block_dim, seed, int_probability, int_scale_factor
CBs: c_0 (input), c_16 (output)
```

Pattern:
```
init_sfpu(c_0, c_16)
dropout_kernel_init(seed)
for each block:
    cb16.reserve_back(block_dim)
    for each tile:
        acquire_dst()
        cb0.wait_front(1)
        copy_tile(c_0, 0, 0)
        dropout_tile(0, probability, scale)
        pack_tile(0, c_16)
        cb0.pop_front(1)
        release_dst()
    cb16.push_back(block_dim)
```

## 10. Key Observations for Reference Implementation

### What Makes Dropout Special (vs. standard unary SFPU ops)
1. **Hardware PRNG dependency**: Uses `SFPMOV` with `instr_mod1=8, lreg_c=9` to generate random numbers — this is a hardware feature not used by any other standard unary op
2. **Initialization overhead**: 600 NOP cycles for PRNG seeding (cannot be amortized across tiles)
3. **Conditional execution**: Uses SFPU conditional context (`SFPIADD` sets CC, `SFPMOV` conditionally writes, `SFPENCC` ends context) — a pattern useful for any operation that needs branching per-element
4. **Raw TTI instructions**: Uses `TTI_*` (Tensix Tightly-coupled Instruction) macros instead of SFPI C++ intrinsics — this is a lower-level coding style
5. **Integer comparison**: Probability comparison is done in integer domain (avoiding float comparison imprecision)
6. **Two-phase output**: Scales first, then conditionally zeros — never needs to undo scaling

### SFPI Instruction Patterns Reusable for Other Operations
- **PRNG generation**: `TTI_SFPMOV(0, 9, dest, 8)` — generates random uint32_t per SFPU lane
- **Sign bit manipulation**: `TTI_SFPSETSGN(0, src, dest, 1)` — clears sign bit (useful for absolute value, unsigned comparison)
- **Integer conditional**: `TTI_SFPIADD(0, threshold, value, 10)` — sets condition code for integer less-than comparison
- **Conditional assignment**: `TTI_SFPMOV(0, LCONST_0, dest, 0)` under active condition code — conditional zero
- **End conditional**: `TTI_SFPENCC(0, 0, 0, 0)` — must be called to clear condition context

### Relevance to RReLU
Dropout's PRNG and conditional execution patterns are directly relevant to RReLU, which needs:
- Random number generation per element (for the random slope in negative region)
- Conditional branching (different behavior for positive vs. negative inputs)
- Scaling by a random factor (slope * x for negative x)

The key difference is that RReLU needs random numbers in a **float range [lower, upper]** rather than integer comparison against a threshold, and the condition is based on the **input sign** rather than a random threshold.
