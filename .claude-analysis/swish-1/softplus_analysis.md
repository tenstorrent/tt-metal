## SFPU Kernel Implementation
This section provides a dedicated deep dive into the underlying SFPU kernel function that the compute kernel dispatches to.

### Unary Dispatch Summary
- **UnaryOpType**: `SOFTPLUS`
- **Compute kernel**: `ttnn/cpp/ttnn/operations/eltwise/unary/device/kernels/compute/eltwise_sfpu.cpp`
- **SFPU_OP_CHAIN_0 expansion**:
  - Init: `softplus_tile_init();`
  - Func: `softplus_tile(idst, beta_bits, beta_recip_bits, threshold_bits);`
    where `beta_bits = bit_cast<uint32_t>(beta)`, `beta_recip_bits = bit_cast<uint32_t>(1/beta)`, `threshold_bits = bit_cast<uint32_t>(threshold)`.
    Default params from `string_to_unary_with_param("softplus")`: `beta=1.0f`, `threshold=20.0f`.

### Call Chain

1. `eltwise_sfpu.cpp:kernel_main()` invokes `SFPU_OP_CHAIN_0`, which expands to `softplus_tile_init()` (init pass) and `softplus_tile(idst, beta, beta_recip, threshold)` (per-tile compute pass).
2. `softplus_tile_init()` in `softplus.h` expands via `SFPU_UNARY_KERNEL_INIT(softplus, APPROX)` to `llk_math_eltwise_unary_sfpu_init<SfpuType::softplus, false>()`, which calls `_llk_math_eltwise_unary_sfpu_init_<softplus>()` → configures ADDR_MOD and resets counters.
3. `softplus_tile(idst, beta, beta_recip, threshold)` in `softplus.h` expands via `SFPU_UNARY_THREE_PARAM_KERNEL_FN(calculate_softplus, RC, APPROX, idst, beta, beta_recip, threshold)` to `_llk_math_eltwise_unary_sfpu_params_<false>(ckernel::sfpu::calculate_softplus<false>, idst, VectorMode::RC, beta, beta_recip, threshold)`.
4. `_llk_math_eltwise_unary_sfpu_params_` iterates over 4 faces (RC mode), calling `calculate_softplus<false>(beta, beta_recip, threshold)` for each face and emitting `TTI_SETRWC` between faces to advance the DEST address.
5. `calculate_softplus<false>` loops 8 iterations per face, calling `calculate_softplus_body<false>` and incrementing `dst_reg++` after each iteration.
6. `calculate_softplus_body` scales input by `beta`, conditionally applies the `softplus(x)` helper (which uses `_sfpu_exp_21f_bf16_` or `PolynomialEvaluator::eval`), scales by `beta_reciprocal`, and writes back to `dst_reg[0]`.

---

### Parameters Dispatch Summary

- **Vector mode**: `VectorMode::RC` — all 4 tile faces are processed. Each face is a 16×16 block; the full tile (32×32 = 1024 elements) is covered.
- **Operation invocation**: `_llk_math_eltwise_unary_sfpu_params_` runs a `for (int face = 0; face < 4; face++)` loop. Per face, `calculate_softplus<false>(param0, param1, param2)` is called once; it internally loops 8 iterations via `for (int d = 0; d < ITERATIONS; d++)`.
- **DEST address progression**: Standard DEST progression (ITERATIONS=8 per face, `dst_reg++` per iteration, `TTI_SETRWC(p_setrwc::CLR_NONE, p_setrwc::CR_D, 8, 0, 0, p_setrwc::SET_D)` emitted twice between faces in RC mode). This advances by 1 sfpi row (= 2 physical DEST rows = 32 elements) per `dst_reg++`, covering 256 elements per face.

### Annotated SFPU Kernel Source

#### Helper: `_sfpu_exp_21f_bf16_` (used for negative inputs x < -5)

```cpp
// File: tt_metal/tt-llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_exp.h

// Helper: convert scaled float (val/2^23) to int, constraint: 0 <= val < 128
sfpi_inline sfpi::vInt _float_to_int32_for_exp_21f_(sfpi::vFloat val)
{
    sfpi::vInt exp = sfpi::exexp(val);             // SFPEXEXP: extract biased exponent field
    sfpi::vInt man = sfpi::exman8(val);            // SFPEXMAN: extract mantissa with implicit bit (range [1,2])
    man = sfpi::reinterpret<sfpi::vInt>(           // SFPSHFT: shift mantissa left by exponent
        sfpi::shft(sfpi::reinterpret<sfpi::vUInt>(man), exp));
    return man;
}

// exp21f: 2-step exp via integer bit manipulation + 2nd-degree polynomial correction
// is_fp32_dest_acc_en=false (default): rounds result to BF16 to avoid DST truncation errors
template <bool is_fp32_dest_acc_en>                // is_fp32_dest_acc_en=false
sfpi_inline sfpi::vFloat _sfpu_exp_21f_bf16_(sfpi::vFloat val)
{
    constexpr float ONE_LN2 = 1.4426950216293334961f; // 1/ln(2)
    sfpi::vFloat xlog2 = (val * ONE_LN2 + 127.f);    // SFPMAD: x/ln2 + bias, prepares for 2^(x/ln2)

    sfpi::vFloat threshold_low  = 0.f;
    sfpi::vFloat threshold_high = sfpi::vFloat(255.f);
    sfpi::vec_min_max(threshold_low, xlog2);           // clamp xlog2 to [0, 255] to prevent overflow
    sfpi::vec_min_max(xlog2, threshold_high);

    sfpi::vInt z = _float_to_int32_for_exp_21f_(xlog2); // scale xlog2 by 2^23 as integer

    sfpi::vInt exponential_part = exexp_nodebias(sfpi::reinterpret<sfpi::vFloat>(z)); // integer part: 2^z_i
    sfpi::vInt fractional_part  = sfpi::exman9(sfpi::reinterpret<sfpi::vFloat>(z));   // fractional part mantissa bits

    sfpi::vFloat frac = sfpi::int32_to_float(fractional_part, 0); // SFPCAST: int → float, range [0, 2^23]

    // 2nd-degree Horner polynomial to refine 2^(x_f) over [0, 2^23]
    frac = PolynomialEvaluator::eval(frac, 1.0017248f, 7.839635491371155e-08f, 4.791750143340323e-15f);

    sfpi::vFloat y = sfpi::setexp(frac, exponential_part); // SFPSETEXP: reconstruct float = 2^z_i * 2^z_f

    if constexpr (!is_fp32_dest_acc_en)            // true branch (is_fp32_dest_acc_en=false)
    {
        // Round to BF16 to match DST storage precision and avoid off-by-one rounding errors
        y = sfpi::reinterpret<sfpi::vFloat>(sfpi::float_to_fp16b(y, 0)); // SFPSTOCHRND: stochastic round to BF16
    }

    return y;
}
```

#### Helper: `PolynomialEvaluator::eval` (Horner's method)

```cpp
// File: tt_metal/tt-llk/tt_llk_wormhole_b0/common/inc/sfpu/ckernel_sfpu_polyval.h

struct PolynomialEvaluator
{
    // Recursive Horner evaluation: coeff0 + x*(coeff1 + x*(coeff2 + ...))
    // Each recursive level emits one SFPMAD instruction: result = prev_result * x + coeff
    template <typename U, typename Coefficient0, typename... OtherCoefficients>
    sfpi_inline static constexpr auto eval(U x, Coefficient0 coeff0, OtherCoefficients... other_coefficients)
    {
        return coeff0 + x * eval(x, other_coefficients...);
    }
};
```

#### Helper: `softplus` — main approximation dispatch

```cpp
// File: tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_softplus.h

// Two-region approximation for log(1 + exp(x)):
//   x < -5:  use exp21f algorithm (log(1+exp(x)) ≈ exp(x) for large negative x)
//   -5 <= x < 4: 8-term Horner polynomial (Remez minimax coefficients)
//   x >= 4:  pass-through x (log(1+exp(x)) ≈ x for large positive x)
template <bool is_fp32_dest_acc_en = false>           // is_fp32_dest_acc_en=false
inline vFloat softplus(vFloat x) {
    vFloat result = x;                                 // default: result = x (for x >= 4)
    v_if(x < -5.0f) { result = _sfpu_exp_21f_bf16_<is_fp32_dest_acc_en>(x); }  // SFPSETCC + exp21f
    v_elseif(x < 4.0f) {
        result = PolynomialEvaluator::eval(
            x,
            0.6924354434013367f,       // coeff[0]: constant term ≈ ln(2)
            0.49275708198547363f,      // coeff[1]: linear term ≈ 0.5
            0.12142381817102432f,      // coeff[2]
            0.0031102809589356184f,    // coeff[3]
            -0.00330807245336473f,     // coeff[4]
            -0.00028794066747650504f,  // coeff[5]
            5.3185409342404455e-05f,   // coeff[6]
            7.1853546614875086e-06f,   // coeff[7]
            7.4961114648886e-08f);     // coeff[8]: 8-term polynomial (degree 8)
    }
    v_endif;
    return result;
}
```

#### Core dispatch: `calculate_softplus_body` and `calculate_softplus`

```cpp
// File: tt_metal/hw/ckernels/wormhole_b0/metal/llk_api/llk_sfpu/ckernel_sfpu_softplus.h

// Per-element body: scales x by beta, conditionally applies softplus, scales result by beta_reciprocal.
// APPROXIMATION_MODE is passed through but not used for branching.
template <bool APPROXIMATION_MODE>                    // APPROXIMATION_MODE=false (unused in body)
inline void calculate_softplus_body(const float beta, const float beta_reciprocal, const float threshold) {
    vFloat x = beta * dst_reg[0];           // SFPMAD: x = beta * dst_reg[0] * 1.0 + 0.0
    v_if(x < threshold) {                   // SFPSETCC: guard — threshold=20.0f (default)
        dst_reg[0] = beta_reciprocal * softplus(x); // apply softplus(x), scale by 1/beta, store
    }
    v_endif;                                // SFPENCC: restore condition code
}

// Per-face loop: iterates 8 times, advancing dst_reg by 1 sfpi row each iteration.
// Converts raw uint32 params back to float using Converter::as_float (reinterpret_cast).
template <bool APPROXIMATION_MODE, int ITERATIONS = 8> // APPROXIMATION_MODE=false, ITERATIONS=8
inline void calculate_softplus(uint param0, uint param1, uint param2) {
    const float beta             = Converter::as_float(param0); // reinterpret bits as float
    const float beta_reciprocal  = Converter::as_float(param1);
    const float threshold        = Converter::as_float(param2);
    for (int d = 0; d < ITERATIONS; d++) {          // 8 iterations per face = 256 elements
        calculate_softplus_body<APPROXIMATION_MODE>(beta, beta_reciprocal, threshold);
        dst_reg++;                                   // advance 1 sfpi row = 2 physical DEST rows = 32 elements
    }
}

// Empty init — no LReg constants needed (beta/threshold passed as runtime params)
template <bool APPROXIMATION_MODE>
void softplus_init() {}
```

### SFPU Instructions Used

| Instruction / Intrinsic | Description |
|-------------------------|-------------|
| `SFPMAD` (via `vFloat * vFloat`, `vFloat + vFloat`) | Fused multiply-add: `a * b + c`. Used pervasively — beta scaling, polynomial Horner steps, exp21f intermediate computations |
| `SFPSETCC` / `SFPENCC` (via `v_if`, `v_elseif`, `v_endif`) | Set/enable condition code register; guards writes to DST rows |
| `SFPLOAD` / `SFPSTORE` (via `dst_reg[0]` read/write) | Load/store element from/to DEST register via sfpi row address |
| `SFPEXEXP` (via `sfpi::exexp`) | Extract biased exponent field from float |
| `SFPEXMAN` (via `sfpi::exman8`, `sfpi::exman9`) | Extract mantissa bits (8 or 9 bits) from float |
| `SFPSHFT` (via `sfpi::shft`) | Arithmetic shift of integer vector; used in `_float_to_int32_for_exp_21f_` |
| `SFPCAST` (via `sfpi::int32_to_float`) | Convert integer vector to float; used in exp21f fractional-part reconstruction |
| `SFPSETEXP` (via `sfpi::setexp`) | Set exponent field of float, combining integer and fractional parts in exp21f |
| `SFPSTOCHRND` (via `sfpi::float_to_fp16b`) | Round float to BF16 (stochastic rounding); applied in `_sfpu_exp_21f_bf16_` when `is_fp32_dest_acc_en=false` |
| `vec_min_max` (via `sfpi::vec_min_max`) | Per-element min/max clamp; used in exp21f to clamp `xlog2` to `[0, 255]` |

---

### SFPU Register Usage

| Resource | Role |
|----------|------|
| `dst_reg[0]` | Current tile element in DEST register. Read at start of `calculate_softplus_body`; written with the computed softplus result |
| `dst_reg` (advancing pointer) | Incremented by `dst_reg++` in the outer `calculate_softplus` loop; advances 1 sfpi row (2 physical DEST rows = 32 elements) per iteration |
| LREGs (SFPU local registers) | Used implicitly by SFPI for intermediate values (`x`, `result`, `xlog2`, `z`, `frac`, `y`, `exponential_part`, `fractional_part`). Each `vFloat`/`vInt` variable occupies one LREG |
| `vConst0` | Not used by softplus itself; only used in the legacy `softplus_legacy` helper which is not called in the active path |

### Address Mode Configuration

**Configured by**: `eltwise_unary_sfpu_configure_addrmod<SfpuType::softplus>()` called from `_llk_math_eltwise_unary_sfpu_init_`.

**Wormhole B0** (`tt_metal/tt-llk/tt_llk_wormhole_b0/llk_lib/llk_math_eltwise_unary_sfpu.h`):

| ADDR_MOD | srca.incr | srcb.incr | dest.incr | Used by |
|----------|-----------|-----------|-----------|---------|
| `ADDR_MOD_7` | 0 | 0 | 0 | Set for all unary SFPU ops (base config) |

`ADDR_MOD_6` is **not** set for softplus — it is only configured for `topk_local_sort` (incr=32) and `typecast`/`unary_max`/`unary_min`/`signbit` variants (incr=2).

**Blackhole** (`tt_metal/tt-llk/tt_llk_blackhole/llk_lib/llk_math_eltwise_unary_sfpu.h`): Identical to Wormhole B0 for softplus — only `ADDR_MOD_7` (dest.incr=0) is configured.

**Effective addressing**: The `ADDR_MOD_7` with `dest.incr=0` serves as a neutral baseline — it does not auto-increment DEST between SFPU instructions. The destination advancement is instead handled explicitly:
- **Within a face**: `dst_reg++` in SFPI advances the sfpi pointer by 1 row (2 physical DEST rows) per iteration.
- **Between faces**: `TTI_SETRWC(CLR_NONE, CR_D, 8, 0, 0, SET_D)` is emitted twice by `_llk_math_eltwise_unary_sfpu_params_` (RC mode), advancing by 16 physical DEST rows to jump to the next face's start.
