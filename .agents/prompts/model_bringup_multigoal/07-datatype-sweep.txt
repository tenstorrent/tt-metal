/goal Use $datatype-sweep for the HF_MODEL optimized full model in the current tt-metal checkout.

This goal creates the datatype-sweep state for the repo-local TTNN autoport pipeline. Start from the completed optimized full model and choose the fastest weight, activation, CCL, and KV-cache datatype configuration that still satisfies the required full-model accuracy. Do not begin vLLM integration work in this goal.

Goal completion requirements:
- The baseline optimized full model accuracy and performance are refreshed using the main AIME24 chat-template readiness reference with 100 generated tokens unless a concrete model-specific limit prevents it.
- Full-model candidate results are captured in models/autoports/<model>/doc/datatype_sweep/sweep_results.json and sweep_results.csv, including config id, dtype policy, top-1/top-5/top-100, TTFT, decode t/s/u, command, hardware, mesh, and pass/fail status.
- The selected precision config is written to models/autoports/<model>/doc/datatype_sweep/selected_precision_config.json and is either the model default or a documented config that the next stage can request.
- Two pyplot charts exist: top1_perf_pareto.png and top5_perf_pareto.png. Each chart plots evaluated full-model configs, draws the Pareto frontier, marks the selected point in red, and shows a vertical dotted line at the minimum allowed accuracy.
- models/autoports/<model>/doc/datatype_sweep/README.md and work_log.md summarize thresholds, selected config, rejected configs, Pareto interpretation, commands, plots, performance, accuracy, limitations, and exact artifacts.
- The final selected config is the fastest evaluated config that satisfies the full-model top-1/top-5 gate. If the baseline is still selected, the report proves the likely lower-precision wins were tested and rejected with evidence.

Do not mark this goal blocked just because accuracy, trace replay, watcher, profiler, CCL, cache, MoE or plotting validation currently fails. Those failures are expected sweep work. Before marking blocked, isolate the smallest repro, compare against the optimized full-model baseline, try a concrete fix/refutation loop, and for subtle precision, cache, trace, or routed-expert bugs use $autofix or document why it could not be used. A blocked result requires evidence that continued debugging by the agent is no longer meaningful, not merely that a requirement is still failing.
