This goal creates the datatype-sweep state for the repo-local TTNN autoport pipeline. Start from the completed optimized full model and choose the fastest weight, activation, CCL, and KV-cache datatype configuration that still satisfies the required full-model accuracy. Do not begin vLLM integration work in this goal.

Stage 08 completion requirements:
- The baseline optimized full model accuracy and performance are refreshed using the main AIME24 chat-template readiness reference with 100 generated tokens.
- Full-model candidate results are captured in models/autoports/<model>/doc/datatype_sweep/sweep_results.json and sweep_results.csv, including config id, dtype policy, compute-fidelity policy, top-1/top-5/top-100, TTFT, trace-verified teacher-forcing decode t/s/u, measurement regime, command, hardware, mesh, and pass/fail status.
- The selected precision config is written to models/autoports/<model>/doc/datatype_sweep/selected_precision_config.json and is either the model default or a required config artifact that later full-model/vLLM construction paths actually consume by default. It must include weight groups, layer exceptions, compute fidelities, activation/residual dtype, CCL dtype, KV-cache dtype, and logits/sampling dtype assumptions. The stage evidence must prove these fields are consumed by the measured runtime path; a JSON field ignored by hard-coded model code does not satisfy this requirement.
- Recompute doc/context_contract.json for every selected KV-cache dtype candidate that could change memory capacity. Do not complete after reducing advertised capability unless a hard physical device limit, e.g. DRAM, requires it and evidence records the largest feasible value.
- Datatype changes preserve valid non-aligned prompt-length support; if a KV-cache or trace-buffer dtype/layout changes chunking, rerun the non-aligned prompt check.
- Two pyplot charts exist: top1_perf_pareto.png and top5_perf_pareto.png. Each chart plots evaluated full-model configs, draws the Pareto frontier, marks the selected point in red, and shows a vertical dotted line at the minimum allowed accuracy.
- models/autoports/<model>/doc/datatype_sweep/README.md and work_log.md record thresholds, selected config, rejected configs, Pareto interpretation, commands, plots, performance, accuracy, limitations, and exact artifacts.
- The final selected config is the fastest evaluated config that satisfies the full-model top-1/top-5 gate using traced teacher-forcing decode performance. Eager or untraced decode numbers are not valid for Pareto ranking or final selection. The sweep includes likely lower-precision wins and canonical policies, with evidence for rejected configs. For every material BFP4 matmul group considered or selected, the sweep includes a BFP4+LoFi candidate for that same group, or an exact TTNN/runtime blocker plus `$autofix` evidence.
- After selecting the winning config, run the same warmed token-out no-readback benchmark used by optimized full model through the normal selected-config construction path. Record it separately from teacher-forcing as post-selection token-out TTFT/decode t/s/u. Later reports and vLLM comparisons must use this post-selection token-out number when available.
- $stage-review returns `clean-pass`; findings fixed/rereviewed. Local commit stage-owned touched-repo changes; never push; log SHAs.

Unmet requirements, review findings, failed gates: work. Stop only after `$autofix` fails.
