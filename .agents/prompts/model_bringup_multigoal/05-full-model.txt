/goal Use $full-model for the HF_MODEL optimized multichip decoder in the current tt-metal checkout.

This goal creates the full-model state for the repo-local TTNN autoport pipeline. Start from the completed optimized multichip decoder and work on models/autoports/<model>/tt/model.py and tt/generator.py; do not begin vLLM integration work in this goal.

Goal completion requirements:
- models/autoports/<model>/tt/model.py and tt/generator.py exist and implement the full HF autoregressive path: embeddings, all decoder layers, final norm, LM head or tied embeddings, logits, paged KV-cache exposure, and generator prefill/decode boundaries.
- The generator API follows the standard Metal readiness contract, including build_generator and low-level prefill/decode methods suitable for a later generator_vllm.py adapter.
- The full model uses the optimized multichip decoder strategy where appropriate; if any part falls back to single-chip, replicated, host-side, or less optimized behavior, the report identifies and justifies it with evidence.
- Readiness reference generation succeeds or an existing matching reference is reused with provenance. For instruction/chat models, the main readiness reference must use the DeepSeek AIME24 prompt set with the HF tokenizer chat template, e.g. `python -m models.common.readiness_check.generate --hf-model HF_MODEL --prompt-source aime24 --chat-template --gen-len 100 --top-k 100 --output <model_dir>/readiness_aime24_chat.refpt`; raw Tale-of-Two-Cities book references are only supplemental or fallback evidence.
- run_prefill_check and run_teacher_forcing report top-1, top-5, and top-100 for prefill and decode using the AIME24 chat-template reference; expected bar is top-5 >= 98% and top-100 = 100% unless a model-specific issue or evidence justifies otherwise.
- run_autoregressive produces HF and TT completions; the outputs are read and judged for coherence, repetition, wrong-language drift, and early divergence.
- Full-model prefill and decode performance figures are recorded with the commands and artifact paths that produced them.
- Runtime fallback audit covers model/generator paths, including cache ownership, host-logit boundaries, sampling, and reset behavior.
- models/autoports/<model>/doc/full_model/README.md and work_log.md summarize commands, top-k accuracy, autoregressive qualitative verdict, perf, limitations, and exact artifacts.
- The full model runs end to end on real weights with acceptable accuracy and usable generation evidence. If the full model exposes a decoder, cache, mesh, or optimization gap, keep working.
- Prefill TTFT and decode t/s/u reported for batch 1 inference reported at the top of README.md

Do not mark this goal blocked just because PCC, trace replay, watcher, profiler, CCL, cache, MoE or other validation currently fails. Those failures are expected bringup work. Before marking blocked, isolate the failure to the smallest reproducible component, compare against the optimized single-chip baseline, try a concrete fix/refutation loop, and for subtle PCC, mesh/CCL, cache, or routed-expert bugs use $autofix or document why those skills could not be used. A blocked result requires evidence that continued debugging by the agent is no longer meaningful, not merely that a requirement is still failing.
