# golden: layer_norm_rm
/create-op
Build me a simple TTNN operation layer_norm_rm which takes a row-major interleaved tensor,
and does layer normalization.

First computes the variance of each row. It does so by first calculating the
mean of each row, subtracting it from the relevant elements used to compute
said mean. After that, the centralized values get squared, and their mean is
computed as the variance. After that, a small constant epsilon is added to the
variances for numerical stability. Epsilon should be a parameter of the
operation. After that, the inverse square root of the epsilon + var sum is
computed. That sum multiplies all of the relevant centralized values in order
to do standardization. It takes in a tensor that is at least two dimensional,
and the output tensor should have the same dimensions as the input tensor.

Optionally, the operation applies affine transformations using gamma (scale) and
beta (shift) parameters. Both gamma and beta are optional input tensors in the ROW MAJOR memory layout
tensors with shape (1, 1, 1, W), where W is the input width. When gamma and/or
beta are not provided (i.e., None), the operation returns the pure normalized
output without the affine transform. The function signature must support these
call patterns:
  - layer_norm_rm(input)                           # pure normalization
  - layer_norm_rm(input, gamma)                    # scale only
  - layer_norm_rm(input, gamma, beta)              # full affine
  - layer_norm_rm(input, gamma, beta, epsilon=X)   # custom epsilon

Epsilon should default to 1e-5 and must be a keyword-only argument.

The operation should validate inputs on the Python side and raise ValueError or
RuntimeError for: non-bfloat16 input dtype, tile-layout input, gamma/beta shape
width mismatch with input width.

## Layout handling

All tensors (input, gamma, beta) are in ROW_MAJOR_LAYOUT. The kernels must
handle row-major data natively — do NOT convert to TILE_LAYOUT in the Python
entry point or use ttnn.to_layout/ttnn.tilize as a workaround before calling
the kernel. Any necessary tilize/untilize must happen in-kernel (e.g.,
tilize_block in the reader, untilize_block in the writer). The output must
also be ROW_MAJOR_LAYOUT.

## Dtype and alignment

Inputs should be bfloat16, with the final 2 dimensions being tile aligned to 32. Do not heavily rely on the bfloat16 datatype and instead implement "as if" you had to implement for other dtypes as well.

Run in FULLY AUTOMATED mode. Introduce reasonable assumptions and DO NOT ask
for confirmation or clarifications.

Run with FULL BREADCRUMBS LOGGING ENABLED.
