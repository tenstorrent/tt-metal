# golden: softmax
/create-op
Softmax operation: for each element along a given dimension, compute
exp(x_i) / sum(exp(x_j)) where the sum is over all elements along that
dimension. This is the standard softmax function used in attention mechanisms
and classification heads.

The function signature must be:

    softmax(
        input_tensor: ttnn.Tensor,    # bfloat16, TILE_LAYOUT, 4D
        dim: int = -1,                # dimension along which softmax is computed
        *,
        numeric_stable: bool = True,  # subtract max before exp for numerical stability
    ) -> ttnn.Tensor

This supports these call patterns:
- softmax(t)                                    # default dim=-1, stable=True
- softmax(t, dim=-1)                            # explicit last dim
- softmax(t, dim=-2)                            # height dim
- softmax(t, numeric_stable=False)              # unstable mode (no max subtraction)
- softmax(t, dim=-2, numeric_stable=False)      # all explicit

Only dim=-1 and dim=-2 are supported. All other dim values must be rejected.

The operation must validate inputs on the Python side and raise ValueError or
RuntimeError for: wrong dtype (not bfloat16), wrong layout (not TILE_LAYOUT),
wrong rank (less than 2), invalid dim (not -1 or -2).

Input requirements:
- dtype: bfloat16
- layout: TILE_LAYOUT
- rank: 4 (N, C, H, W)
- H, W: divisible by 32 (tile-aligned)
- memory: DRAM or L1

Output: same shape, dtype, and layout as input.

Run in FULLY AUTOMATED mode. Introduce reasonable assumptions and DO NOT ask
for confirmation or clarifications.

Run with FULL BREADCRUMBS LOGGING ENABLED.
