# Milestone v1.1: Silicon Validation

**Status:** SHIPPED 2026-03-11
**Phases:** 2
**Total Plans:** 3

## Overview

Silicon data-transfer validation for all 9 auto-packetizing wrapper families. Rewrote test runners to use BaseFabricFixture, created device kernels for each family, and verified byte-for-byte data correctness on 4-chip TT hardware.

## Phases

### Phase 2: silicon-data-transfer-validation

**Goal**: Validate all auto-packetizing wrappers deliver correct data on silicon with payloads exceeding FABRIC_MAX_PACKET_SIZE
**Depends on**: Phase 1 (fabric-auto-packetization)
**Plans**: 3 plans

Plans:
- [x] 02-01-PLAN.md -- Rewrite unicast runner for BaseFabricFixture + 3 new unicast kernels + TEST_F silicon cases
- [x] 02-02-PLAN.md -- Rewrite multicast runner + 4 multicast kernels + sparse multicast + 1D fixture tests
- [x] 02-03-PLAN.md -- Silicon test execution, failure triage, and human verification checkpoint

**Details:**
- Rewrote unicast_runner.cpp and multicast_runner.cpp from MeshDeviceFixtureBase to BaseFabricFixture pattern with L1 direct I/O
- Created test_common.hpp with AutoPacketFamily enum (9 families), RawTestParams, family_kernel_path()
- Created 8 device kernels: 4 unicast variants + 4 multicast variants + 1 sparse multicast
- 17 silicon TEST_F cases: 4 unicast (2D) + 4 multicast (2D) + 4 unicast (1D) + 4 multicast (1D) + 1 sparse (skipped)
- 16/17 tests PASSED, 1 SKIPPED (SparseMulticast issue #36581)

## Milestone Summary

**Key Decisions:**
- L1 direct I/O via WriteToDeviceL1/ReadFromDeviceL1 for BaseFabricFixture tests (avoids DRAM-as-L1 bug)
- GlobalSemaphore created fresh per test call for per-chip MeshDevice pattern
- MeshGraph::chip_to_coordinate for bounding-box computation in multicast runner
- Sparse multicast test inlined in test file (different dispatch from multicast runner)
- Linear 1D tests reuse mesh multicast kernels (per-direction fanout degenerates in 1D)
- Phase gate accepted at 8/9 families validated; SparseMulticast deferred pending firmware fix

**Issues Resolved:**
- L1 overflow in multicast runner (reduced buffer sizes)
- 1D/2D kernel routing mismatch (FABRIC_2D ifdef guards)
- Device adjacency detection for Fabric1DFixture

**Issues Deferred:**
- SparseMulticast silicon validation (issue #36581 -- firmware limitation, Ethernet core lockup)

**Technical Debt Incurred:**
- SparseMulticast GTEST_SKIP guard -- must be removed only after firmware fix ships
