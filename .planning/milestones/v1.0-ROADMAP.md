# Milestone v1.0: Fabric Auto-Packetization

**Status:** SHIPPED 2026-03-11
**Phases:** 1
**Total Plans:** 8

## Overview

Enabled all fabric write APIs in `linear/api.h` and `mesh/api.h` to transparently handle payloads larger than `FABRIC_MAX_PACKET_SIZE` by auto-packetizing under the hood. Existing single-packet APIs renamed `_single_packet` for power users. New auto-packetizing wrappers keep original names. Zero caller changes required.

## Milestone Summary

**Key Decisions:**

- _single_packet suffix for renamed APIs; wrappers keep original names
- Breadth-first multi-connection chunking (all connections get chunk N before chunk N+1)
- Scatter wrappers are passthrough (pre-computed NOC addresses can't be independently chunked)
- mesh SetRoute=false pattern: route setup pass before chunk loop
- Fused ops: intermediate chunks as regular writes, atomic_inc only on final chunk
- Tests compiled into fabric_unit_tests binary (not separate target)
- detail::CompileProgram for device kernel compile-only validation

**Issues Resolved:**

- Missing FABRIC_2D define in compile-only test (fixed by using Fabric2DFixture)
- Stale CMake target for separate test binary (removed)
- 7 of 9 wrapper families lacked compile coverage (fixed by plan 01-08)

**Issues Deferred:**

- Hardware data-transfer validation (full end-to-end test on fabric-configured system)
- Raw scatter write auto-packetization (stays passthrough — pre-computed NOC addresses)
- Sparse multicast for mesh/api.h (linear-only concept)

**Technical Debt Incurred:**

- None significant
