<objective>
Rename 4 scatter/fused-scatter function families in `mesh/api.h` to `_single_packet` and add passthrough wrappers.

Purpose: Same as linear/api.h scatter treatment — scatter headers carry pre-computed NOC addresses that cannot be independently chunked. The rename applies for naming convention, but wrappers are passthrough (no chunking). Note that mesh/api.h also has `multicast_fused_scatter_write_atomic_inc` raw-size functions that will also get addrgen overloads added in Plan 06. This plan handles only the raw-size rename + passthrough wrapper.

Output: Modified `tt_metal/fabric/hw/inc/mesh/api.h` with 8 scatter renames + 8 passthrough wrappers.
</objective>

    tt_l1_ptr FabricSenderType* client_interface,
    volatile PACKET_HEADER_TYPE* packet_header,
    uint8_t dst_dev_id,
    uint16_t dst_mesh_id,
    uint32_t src_addr,
    uint32_t size,
    tt::tt_fabric::NocUnicastScatterCommandHeader noc_unicast_scatter_command_header) {
    fabric_unicast_noc_scatter_write_single_packet<FabricSenderType>(
        client_interface, packet_header, dst_dev_id, dst_mesh_id, src_addr, size,
        noc_unicast_scatter_command_header);
}

FORCE_INLINE void fabric_unicast_noc_scatter_write(
    tt::tt_fabric::RoutingPlaneConnectionManager& connection_manager,
    uint8_t route_id,
    uint32_t src_addr,
    uint32_t size,
    tt::tt_fabric::NocUnicastScatterCommandHeader noc_unicast_scatter_command_header) {
    fabric_unicast_noc_scatter_write_single_packet(
        connection_manager, route_id, src_addr, size, noc_unicast_scatter_command_header);
}
