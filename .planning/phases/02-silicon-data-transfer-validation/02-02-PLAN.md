**multicast_runner.cpp:** FULL REWRITE following the same BaseFabricFixture pattern established in Plan 01's unicast_runner.cpp rewrite. Key changes from the broken version:

1. Function signature: `void run_raw_multicast_write_test(BaseFabricFixture* fixture, const RawTestParams& p)` -- takes `BaseFabricFixture*`
2. Remove ALL `MeshBuffer`, `WriteShard`, `ReadShard`, `EnqueueMeshWorkload`, `MeshDeviceFixtureBase` usage
3. Use `fixture->get_devices()` to enumerate all per-chip MeshDevices. Source = `fixture->get_device(p.src_chip)`. Receivers = all devices except source.
4. Use `generate_worker_mem_map()` for L1 layout. Write test data via `detail::WriteToBuffer` to sender's L1 source address. Zero all receiver target addresses via `detail::WriteToBuffer`.
5. Create `GlobalSemaphore` on each receiver's per-chip MeshDevice.
6. Select kernel path based on `p.family` using `family_kernel_path()`.
7. For multicast: compute bounding-box hop counts (E, W, N, S) from sender position relative to all receivers, same as existing kernel's dir_mask pattern.
8. Append per-direction fabric connection RT args using `append_fabric_connection_rt_args` for each active direction, then hop counts.
9. For scatter multicast families: use small payloads only.
10. For fused multicast families: do NOT send separate atomic_inc per direction -- the fused wrapper handles it. But the existing `multicast_tx_writer_raw.cpp` sends separate atomic_inc per direction via manual `send_payload_flush_non_blocking_from_address`. For FUSED multicast kernels, the auto-packetizing wrapper fires the atomic_inc, so the kernel should NOT do manual completion signaling. The new fused multicast kernels must omit the per-direction `send_completion` lambda.
11. For non-fused multicast families: keep existing completion pattern (manual per-direction atomic_inc).
12. Dispatch: `RunProgramNonblocking` for each receiver, then sender, then `WaitForSingleProgramDone` for sender, then each receiver.
13. Read back from each receiver via `detail::ReadFromBuffer` and compare byte-for-byte.

**4 new multicast device kernels:** Each follows existing `multicast_tx_writer_raw.cpp` pattern (per-direction senders, same RT args layout), but calls the corresponding auto-packetizing multicast API:

1. **scatter_multicast_tx_writer_raw.cpp:** Per-direction `fabric_multicast_noc_scatter_write(sender, hdr, 0, 0, ranges, src_l1_addr, scatter_size, NocUnicastScatterCommandHeader{addr0, addr1})`. Plus per-direction manual atomic_inc completion (scatter is not fused).

2. **fused_atomic_inc_multicast_tx_writer_raw.cpp:** Per-direction `fabric_multicast_noc_fused_unicast_with_atomic_inc(sender, hdr, 0, 0, ranges, src_l1_addr, total_size, NocUnicastAtomicIncFusedCommandHeader{dst_noc_addr, sem_noc_addr, 1, 32})`. NO manual completion -- fused wrapper fires atomic_inc on final chunk per direction.

3. **fused_scatter_atomic_inc_multicast_tx_writer_raw.cpp:** Per-direction `fabric_multicast_noc_fused_scatter_write_atomic_inc(sender, hdr, 0, 0, ranges, src_l1_addr, scatter_size, NocUnicastScatterAtomicIncFusedCommandHeader{addr0, addr1, sem_noc_addr, 1, 32})`. No manual completion.

4. **sparse_multicast_tx_writer_raw.cpp:** Uses `linear::experimental` namespace (NOT mesh). Single sender. Calls `fabric_sparse_multicast_noc_unicast_write(&sender, packet_header, sparse_mask, src_l1_addr, total_size, NocUnicastCommandHeader{dst_noc_addr})` where `sparse_mask` is a runtime arg (u32 bitmask of destination devices). After data, sends manual atomic_inc. This kernel does NOT use per-direction fanout -- sparse multicast is linear-only with one sender. Include only `linear/api.h`, do NOT define `FABRIC_2D`.

All multicast kernels: include `mesh/api.h` and `linear/api.h` (except sparse_multicast which only needs `linear/api.h`).
