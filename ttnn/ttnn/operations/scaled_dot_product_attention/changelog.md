# Changelog: scaled_dot_product_attention

## Phase 0 — Core Implementation
- **Date**: 2026-06-10
- **What was done**: Initial Flash-Attention implementation via incremental pipeline (planner → implementer → verifier). Online softmax recurrence, per-block CBs (≤ c_q×c_kv tiles, S_kv-independent), full helper composition, full-grid distribution over (b, h, q_chunk).
- **SUPPORTED at Phase 0**: dtype=[bfloat16], layout=[TILE], alignment=[tile_aligned], attention_kind=[self, cross], kv_heads_mode=[mha], mask_mode=[none, causal], scale_mode=[auto, explicit]
- **Accuracy achieved**: PCC ≥ 0.99995, max_abs_err ≤ 0.0195, rel_rms_err ≤ 0.0152 (4 shapes via test_scaled_dot_product_attention_precision_baseline.py)
- **Golden suite at Phase 0**: 134 / 140 supported cells passing, 604 xfail_expected, 0 xpass_drift / 0 xfail_wrong_mode (per verifier_report.json). 6 supported_fail = long-context unmasked numerical-precision → Refinement 3.
- **Issues encountered**: verifier fixed registry drift (taggers/axes didn't match feature_spec — alignment 3-value, attention_kind/kv_heads_mode/mask_mode/scale_mode), missing registry exports in `__init__.py`, writer per-tile NoC barrier (batched per chunk), reader mask-accessor construction inside KV loop (hoisted); added S_kv alignment gate to validate().
- **Tests added**: test_scaled_dot_product_attention.py, test_scaled_dot_product_attention_debug.py, test_scaled_dot_product_attention_precision_baseline.py

## Refinement 1 — Numerical configurability expansion
- **Date**: 2026-06-10
- **What was done**: Added `ttnn.float32` and `ttnn.bfloat8_b` to SUPPORTED["dtype"]; exposed `compute_kernel_config` (WormholeComputeKernelConfig) on the entry point; derived Q/KT/V/mask/probs/out CB formats + page sizes from input dtype, and stat/accumulator intermediate CB formats from `fp32_dest_acc_en` (Float32 when on, Float16_b when off — Float32 intermediates with a 16-bit DEST corrupted values, pcc ~0.41 at every fidelity, probes 007/008). Defaults preserve Phase 0 behavior for bf16/bf8b (HiFi2 + fp32 dest, no approx); fp32 inputs default to HiFi4 + fp32 dest (helper-documented only-correct pairing). Zero compute/dataflow kernel changes (pass condition held). Guards (config-level, not registry axes): HiFi4 + fp32 dest + bf16/bf8b → ValueError (Wormhole B0 known-bad #38306); fp32/bf8b inputs + fp32_dest_acc_en=False → NotImplementedError (probe_008: fp32 pcc 0.008, bf8b NaN). fp32 D=1024 corner fits L1 (pcc 0.999999).
- **Accuracy achieved**: defaults, randn, shapes (1,1,128,64)/(1,4,256,64): fp32 PCC≥0.999998 rel_rms≤0.0033; bf16 PCC≥0.99997 rel_rms≤0.0132; bf8b PCC≥0.9998 rel_rms≤0.0200. Full sweep in tests/ttnn/unit_tests/operations/scaled_dot_product_attention/precision_matrix_results.md.
- **Golden test progress**: 412/420 supported cells passing (Phase 0: 134/140). 8 fails all severity=precision near-misses, NOT excluded: 6 pre-existing bf16 long-context (Refinement 3) + 2 new bf8b Q1x1x8192x64 mask=none (RMS 0.165 vs 0.12, ulp_p99 9.5e8 — same near-uniform-attention root cause, bf8b cb_probs quantization; lever filed under Refinement 3 inherited notes). Regression suite: 22 passed / 17 failed, all pre-existing buckets (gqa/mqa → R2, uniform/negative/long-context → R3) — no new regressions.
- **Issues encountered**: Float32 stat CBs + 16-bit DEST = silent corruption (fixed via acc_fmt derivation); fp32/bf8b + 16-bit DEST structurally unsupported (guarded); uniform-input PCC degenerates (near-uniform attention) — precision matrix gates uniform cells on max_abs instead.
- **Tests added**: test_scaled_dot_product_attention_precision_matrix.py (precision matrix + config-default equality + guard tests), precision_matrix_results.md, probes 006–008.

## Refinement 2 — GQA / MQA head mapping
- **Date**: 2026-06-10
- **What was done**: Added `gqa` and `mqa` to SUPPORTED["kv_heads_mode"]. Reader-only change per the verifier plan: new CT arg `H_kv` (index 12, TensorAccessorArgs base moved to 13), KV head base now `kv_bh = b*H_kv + h/(H/H_kv)` (HEAD_RATIO compile-time; MHA degenerates to `kv_bh == bh`); mask indexing unchanged (per-Q-head). Zero compute/writer/descriptor-structure changes; SUPPORTED extended; validate() untouched (taggers + `H_q % H_kv == 0` rule already in place).
- **Accuracy achieved**: PCC ≥ 0.995 (bf16), ≥ 0.999 (fp32), ≥ 0.99 (bf8b) across 21 GQA/MQA unit cases (ratios 4:1/8:1/12:1/32:1, self/cross/batch/causal/explicit-scale, all 3 dtypes). Head mapping verified exactly via deterministic constant-V head-identification test (each KV head V = head index; output identifies source head to within bf16 probs quantization ~1.5%).
- **Golden test progress**: 638 passed / 25 failed / 120 xfailed (`--run-all`). GQA/MQA cells: 64/68 passing; all 4 `test_gqa_mqa_forward` regressions now pass; 0 xpass_drift. The 4 GQA/MQA misses are long-context mask=none precision near-misses (Q1x4x4096x64 KV1x1 mqa, Q1x8x4096x128 KV1x2 gqa × scale_modes; RMS 0.064–0.067 vs 0.05) — same near-uniform-attention root cause and magnitude as the MHA siblings at the same shapes; left failing (no EXCLUSIONS), filed under Refinement 3's inherited list. Remaining 21 failures are all pre-existing R3 buckets (8 long-context golden + 13 uniform/negative/long-context regressions).
- **Issues encountered**: None kernel-side. Initial exact-mapping test used atol=0.05, too tight for bf16 probs quantization (~1.5% of the constant, MHA control failed too) — switched to rtol=0.05/atol=0.1 (mapping bugs are off by ≥ 1.0).
- **Tests added**: test_scaled_dot_product_attention_gqa_mqa.py (21 cases: GQA/MQA self/cross/batch/causal/explicit-scale, dtype sweep, constant-V head identification incl. MHA control).

## Refinement 3 — Long-context / near-uniform-attention precision
- **Date**: 2026-06-10
- **What was done**: Host-only fix, zero kernel changes. (1) `cb_probs` format now follows `fp32_dest_acc_en` like the stat CBs (Float32 by default) instead of the input dtype; (2) default `math_fidelity` for bf16/bf8b raised HiFi2 → HiFi3. Actual root cause (probes 009–013) was NOT cb_probs quantization alone: the offline torch model puts bf16-probs noise at rms ≈0.002, far below the observed 0.067–0.158. The dominant term was HiFi2 skipping the SrcB low-mantissa fidelity phase, so rowsum `l` and P@V consumed the same packed probs at different effective precision — the V=ones invariant (exact O==1) was off by up to 0.11 abs at HiFi2 and is ≤0.012 after the fix. Either lever alone is insufficient (HiFi3+bf16 probs: 8192 rms 0.072; fp32 probs+HiFi2: 0.153); together: 0.014.
- **Accuracy achieved**: vs fp32 ref, mask=none auto-scale randn: rel_rms 0.0078 (1,1,4096,64), 0.0142 (1,1,8192,64), 0.0080 MQA 1x4x4096x64, 0.0084 GQA 1x8x4096x128 — all bf16, target 0.05; bf8b (1,1,8192,64) 0.0198, target 0.12.
- **Golden test progress**: 654 passed / 9 failed / 120 xfailed (was 638/25/120). Zero `supported_fail` cells — all 12 R3-named cells pass. `long_context_smoke`, `gqa_mqa_forward` all pass. The 9 remaining are uniform_input (4) / negative_input (5) regressions: rms 0.04–0.21 vs 0.04, every one `ulp_p99=1, median_abs=0` (≤1 bf16-ulp disagreement on ~1–2% of elements). Reference output std ≈ 1 bf16 ulp ⇒ rel-RMS is at the bf16 output quantization floor; pre-fix severity=bug at S=512 (rms 0.50, 16 ulp) is now precision (max 1–2 ulp). Static analyzer: no structural inconsistency remains; residual = FPU 19-bit truncation in final `O·(1/l)` bcast + `recip_tile` ≈1 ulp + pack rounding. Not excluded (no EXCLUSIONS); deferred to new Refinement 5 with SFPU-normalize lever.
- **Issues encountered**: HiFi3+fp32-dest is outside the documented matmul known-bad (HiFi4 only); guard unchanged. `test_default_config_matches_explicit_hifi2` updated → `_hifi3` to track the new default. ~1.5x matmul cycle cost at HiFi3 (precision over throughput per refinement goal).
- **Tests added**: test_scaled_dot_product_attention_long_context.py (14 cases: 6 R3 golden cells incl. GQA/MQA/bf8b vs fp32 ref, V=ones invariant S=128–8192, uniform/negative ulp-floor max-abs gates), probes 009–013.
