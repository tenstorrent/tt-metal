# Changelog: scaled_dot_product_attention

## Phase 0 — Core Implementation
- **Date**: 2026-06-10
- **What was done**: Initial Flash-Attention implementation via incremental pipeline (planner → implementer → verifier). Online softmax recurrence, per-block CBs (≤ c_q×c_kv tiles, S_kv-independent), full helper composition, full-grid distribution over (b, h, q_chunk).
- **SUPPORTED at Phase 0**: dtype=[bfloat16], layout=[TILE], alignment=[tile_aligned], attention_kind=[self, cross], kv_heads_mode=[mha], mask_mode=[none, causal], scale_mode=[auto, explicit]
- **Accuracy achieved**: PCC ≥ 0.99995, max_abs_err ≤ 0.0195, rel_rms_err ≤ 0.0152 (4 shapes via test_scaled_dot_product_attention_precision_baseline.py)
- **Golden suite at Phase 0**: 134 / 140 supported cells passing, 604 xfail_expected, 0 xpass_drift / 0 xfail_wrong_mode (per verifier_report.json). 6 supported_fail = long-context unmasked numerical-precision → Refinement 3.
- **Issues encountered**: verifier fixed registry drift (taggers/axes didn't match feature_spec — alignment 3-value, attention_kind/kv_heads_mode/mask_mode/scale_mode), missing registry exports in `__init__.py`, writer per-tile NoC barrier (batched per chunk), reader mask-accessor construction inside KV loop (hoisted); added S_kv alignment gate to validate().
- **Tests added**: test_scaled_dot_product_attention.py, test_scaled_dot_product_attention_debug.py, test_scaled_dot_product_attention_precision_baseline.py

## Refinement 1 — Numerical configurability expansion
- **Date**: 2026-06-10
- **What was done**: Added `ttnn.float32` and `ttnn.bfloat8_b` to SUPPORTED["dtype"]; exposed `compute_kernel_config` (WormholeComputeKernelConfig) on the entry point; derived Q/KT/V/mask/probs/out CB formats + page sizes from input dtype, and stat/accumulator intermediate CB formats from `fp32_dest_acc_en` (Float32 when on, Float16_b when off — Float32 intermediates with a 16-bit DEST corrupted values, pcc ~0.41 at every fidelity, probes 007/008). Defaults preserve Phase 0 behavior for bf16/bf8b (HiFi2 + fp32 dest, no approx); fp32 inputs default to HiFi4 + fp32 dest (helper-documented only-correct pairing). Zero compute/dataflow kernel changes (pass condition held). Guards (config-level, not registry axes): HiFi4 + fp32 dest + bf16/bf8b → ValueError (Wormhole B0 known-bad #38306); fp32/bf8b inputs + fp32_dest_acc_en=False → NotImplementedError (probe_008: fp32 pcc 0.008, bf8b NaN). fp32 D=1024 corner fits L1 (pcc 0.999999).
- **Accuracy achieved**: defaults, randn, shapes (1,1,128,64)/(1,4,256,64): fp32 PCC≥0.999998 rel_rms≤0.0033; bf16 PCC≥0.99997 rel_rms≤0.0132; bf8b PCC≥0.9998 rel_rms≤0.0200. Full sweep in tests/ttnn/unit_tests/operations/scaled_dot_product_attention/precision_matrix_results.md.
- **Golden test progress**: 412/420 supported cells passing (Phase 0: 134/140). 8 fails all severity=precision near-misses, NOT excluded: 6 pre-existing bf16 long-context (Refinement 3) + 2 new bf8b Q1x1x8192x64 mask=none (RMS 0.165 vs 0.12, ulp_p99 9.5e8 — same near-uniform-attention root cause, bf8b cb_probs quantization; lever filed under Refinement 3 inherited notes). Regression suite: 22 passed / 17 failed, all pre-existing buckets (gqa/mqa → R2, uniform/negative/long-context → R3) — no new regressions.
- **Issues encountered**: Float32 stat CBs + 16-bit DEST = silent corruption (fixed via acc_fmt derivation); fp32/bf8b + 16-bit DEST structurally unsupported (guarded); uniform-input PCC degenerates (near-uniform attention) — precision matrix gates uniform cells on max_abs instead.
- **Tests added**: test_scaled_dot_product_attention_precision_matrix.py (precision matrix + config-default equality + guard tests), precision_matrix_results.md, probes 006–008.
