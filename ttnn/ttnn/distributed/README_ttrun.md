# tt-run: MPI Process Launcher for TT-Metal and TTNN

`tt-run` is a lightweight wrapper around `mpirun` that simplifies launching TT-Metal and TT-NN distributed applications. It maps MPI ranks to logical mesh topology (MeshId, MeshHostRank) and configures per-rank environment variables.

The primary way to use tt-run is **auto allocation mode**: you provide a Mesh Graph Descriptor and host list, and tt-run auto-generates rank bindings and rankfile. Alternatively, **legacy mode** lets you supply pre-built rank bindings and rankfile directly—including files generated by auto allocation mode. This README covers both approaches.

For multi-node cluster setup, hostfiles, rankfiles, and validation, see [tools/scaleout/validation/README.md](../../../tools/scaleout/validation/README.md).

---

## Auto allocation mode (Mesh Graph Descriptor)

> **Note:** Auto allocation mode is not yet implemented. The following describes the intended design. Use legacy mode for now.

In the auto allocation mode, you provide:

1. **`--mesh-graph-descriptor`** – Path to the Mesh Graph Descriptor (MGD) file
2. **`--hosts`** – Comma-separated list of hostnames (e.g., `node1,node2,node3`), **or** a **`--mock-cluster-rank-binding`** mapping file (which makes `--hosts` unnecessary)

tt-run automatically:

1. Runs `generate_rank_bindings` on the specified hosts with the MGD file
2. Writes `generated/ttrun/rank_bindings.yaml` and `generated/ttrun/rankfile` (relative to launch directory)
3. Waits 5 seconds for file sync (e.g., NFS)
4. Launches your program with those generated files

### Basic Usage

```bash
# Auto allocation mode: MGD + hosts
tt-run --mesh-graph-descriptor tt_metal/fabric/mesh_graph_descriptors/my_mesh.textproto \
  --hosts nodeA,nodeB,nodeC \
  ./my_app

# With mock cluster mapping: --hosts not required
# Each rank uses its corresponding mock cluster descriptor (TT_METAL_MOCK_CLUSTER_DESC_PATH).
# All processes run on localhost; process count = number of rankings in the mapping file.
tt-run --mesh-graph-descriptor path/to/mgd.textproto \
  --mock-cluster-rank-binding path/to/mock_mapping.yaml \
  ./my_app
```

### Generated Outputs

Auto allocation mode writes to `generated/ttrun/` (relative to launch directory):

| File | Description |
|------|-------------|
| `generated/ttrun/rank_bindings.yaml` | Rank bindings in tt-run format |
| `generated/ttrun/rankfile` | OpenMPI rankfile |

## Legacy Mode (Explicit Rank Bindings)

In legacy mode, you supply a **rank bindings** YAML file and optionally a **rankfile**. Use legacy mode when you need full control over mesh placement, when you already have these files, or when you want to run using files generated by the auto allocation mode and then modify them.

### Basic Usage

```bash
# Single host, rank bindings only
tt-run --rank-binding path/to/rank_bindings.yaml ./my_app

# Multi-host: pass rankfile via --mpi-args
tt-run --rank-binding path/to/rank_bindings.yaml \
  --mpi-args "--host nodeA,nodeB,nodeC --map-by rankfile:file=path/to/rankfile" \
  ./my_app
```

### Using auto allocation–generated files with legacy mode

You can run **auto allocation mode once** to generate rank bindings and rankfile, then use them with **legacy mode** for tweaks (e.g., env overrides, different host placement, or debugging).

**Step 1: Generate files** (auto allocation mode, with a minimal command; avoid `--dry-run` since it skips execution). Use `--hosts` for real clusters, or `--mock-cluster-rank-binding` for mock (no `--hosts` needed):

```bash
# Real cluster
tt-run --mesh-graph-descriptor path/to/mesh.textproto \
  --hosts nodeA,nodeB,nodeC \
  true

# Mock cluster (no --hosts)
tt-run --mesh-graph-descriptor path/to/mesh.textproto \
  --mock-cluster-rank-binding path/to/mock_mapping.yaml \
  true
```

**Step 2: Use with legacy mode:**

```bash
tt-run --rank-binding generated/ttrun/rank_bindings.yaml \
  --mpi-args "--map-by rankfile:file=generated/ttrun/rankfile" \
  ./my_app
```

**Step 3 (optional):** Edit `generated/ttrun/rank_bindings.yaml` or `generated/ttrun/rankfile` before Step 2 to adjust `env_overrides` (e.g., `TT_VISIBLE_DEVICES`), change host/slot mapping, or add/remove ranks—just keep both files consistent.

### Rank Bindings Format

```yaml
rank_bindings:
  - rank: 0
    mesh_id: 0
    mesh_host_rank: 0
    env_overrides:
      TT_VISIBLE_DEVICES: "0,1,2,3"
  - rank: 1
    mesh_id: 0
    mesh_host_rank: 1
    env_overrides:
      TT_VISIBLE_DEVICES: "4,5,6,7"
mesh_graph_desc_path: "path/to/mesh_graph.textproto"
```

### Rankfile Format (OpenMPI)

The rankfile maps each MPI rank to a host and slot. See [tools/scaleout/validation/README.md](../../../tools/scaleout/validation/README.md) for hostfile/rankfile details.

```text
rank 0=hostname0 slot=0
rank 1=hostname1 slot=0
rank 2=hostname2 slot=0
rank 3=hostname3 slot=0
```

## Comparison

| | Auto allocation mode | Legacy Mode |
|---|--------------|-------------|
| **Required args** | `--mesh-graph-descriptor` and `--hosts` (unless `--mock-cluster-rank-binding`) | `--rank-binding` |
| **Rank bindings** | Auto-generated | You supply |
| **Rankfile** | Auto-generated | You supply via `--mpi-args` |
| **Output location** | `generated/ttrun/` | N/A |
| **Use case** | Quick launch with MGD + host list | Full control, hand-crafted or generated configs |

## Additional Options

Both modes support:

- `--dry-run` – Print the MPI command without executing
- `--verbose` / `-v` – Show path resolution and diagnostics
- `--mpi-args` – Extra MPI arguments (e.g., `--bind-to core`)
- `--tcp-interface` – NIC for MPI TCP (e.g., `cnx1`)
- `--bare` – Disable default multi-host MPI settings
- `--mock-cluster-rank-binding` – Mock cluster descriptor mapping file (rank → descriptor path). When used with auto allocation mode, `--hosts` is not required; processes run locally with one rank per mapping entry, each rank getting its corresponding descriptor via `TT_METAL_MOCK_CLUSTER_DESC_PATH`.

Run `tt-run --help` for the full list.
