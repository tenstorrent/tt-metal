    std::uint8_t input_id{0};

    CircularBuffer cb_x_obj(cb_x);

    std::uint8_t output_id{16};
    const auto cb_y = output_id++;

    constexpr uint32_t dst0 = 0;

        tile_regs_acquire();
        cb_x_obj.wait_front(onetile);

        mul_tiles_bcast_scalar(cb_x, cb_clip_coef_clamped, 0, 0, dst0);

        tile_regs_wait();
        pack_tile(dst0, cb_y);
        cb_y_obj.push_back(onetile);
        tile_regs_release();

  • CB ids → constexpr. Original used std::uint8_t input_id{0}; const auto cb_x = input_id++; — runtime values. Chain template parameters require constexpr CB ids, so the four CB declarations became constexpr uint32_t cb_x = 0; cb_clip_coef_clamped = 1; cb_y = 16;. Values are identical.
  • cb_clip_coef_clamped lifecycle: CallerManaged + Scalar. Original waits once outside the loop (held scalar) and pops once at the very end. Chain with CallerManaged emits no wait/pop, so the explicit wait_front(1) and pop_front(1) stay outside the chain. Per the documented rule (feedback_callermanaged_needs_external_wait.md), CallerManaged needs an external wait — which is exactly the original's behavior.
  • cb_x lifecycle: Streaming. Original waits, uses, and pops once per iteration — straight Streaming.
  • Reconfig flags: both None. Original used mul_tiles_bcast_scalar_init_short (no _with_dt suffix) and plain pack_tile — neither does a per-iter format reconfig. Initial state set by compute_kernel_hw_startup(cb_x, cb_clip_coef_clamped, cb_y) matches the original's binary_op_init_common. Chain emits BinaryDataFormatReconfig::None + PackTileReconfig::None to match.
  • Init choice: compute_kernel_hw_startup vs binary_op_init_common. Original used binary_op_init_common(cb_x, cb_clip_coef_clamped, cb_y). The chain's first call still does its own per-element init internally, so either kernel-level init works — but compute_kernel_hw_startup is the convention in already-migrated kernels (moreh_dot_backward etc.) and is the minimal correct init.
