Decisions — hover for tooltip, click the buttons above to focus the code on a single decision

  • 1 · CB ids → constexpr. Chain template parameters require constexpr CB ids; std::uint8_t input_id{0}; const auto cb_x = input_id++; is a runtime value. Same final values (0, 1, 16), just baked at compile time.
  • 2 · cb_clip_coef_clamped lifecycle: Bulk + Scalar. Held scalar across the whole loop. window_1d<Scalar> collapses Bulk's window to 1 tile, so the chain emits exactly cb_wait_front(1) at the head and cb_pop_front(1) at the tail — no external scaffolding needed. The old feedback_bulk_scalar_overwait gotcha was fixed in 14a5a61e462; CallerManaged would also work but forces the caller to keep the wait/pop, which this version removes (review feedback from the initial migration in 98c5ebb783b).
  • 3 · cb_x lifecycle: Streaming. Original waits, uses, and pops once per iteration — straight Streaming, chain owns it.
  • 4 · Reconfig flags: both None. Original used mul_tiles_bcast_scalar_init_short (no _with_dt) and plain pack_tile — no per-iter format reconfig. Chain emits BinaryDataFormatReconfig::None + PackTileReconfig::None to match.
  • 5 · Init: compute_kernel_hw_startup. Convention in already-migrated kernels (e.g. moreh_dot_backward). The chain's first call does its own per-element init internally, so either kernel-level init works — compute_kernel_hw_startup is the minimal correct one.

Decisions / suspected root cause

  • 4 · Reconfig flag mismatch under BF16. Original _with_dt helpers only reconfig when FP32_DEST_ACC_EN is defined — under BFLOAT16 (the failing test config) they're a no-op. The migration emitted BinaryDataFormatReconfig::Input + CopyTileReconfig::Input + PackTileReconfig::Output unconditionally, perturbing format state across the recip → mul → mul chain in a way that broke numerics.
  • 5 · Init swap. Original init_sfpu(cb_output_grad, cb_input_grad) = unary_op_init_common. Migration swapped to binary_op_init_common(cb_tmp_weight, cb_output_grad, cb_input_grad). The first chain (recip) is unary-shaped; binary init leaves different initial unpacker state that the chain may rely on.
  • 6 · HeldStream choice on cb_divisor / cb_tmp1. Both are single-tile held CBs popped at the very end (cb_divisor) or never popped (cb_tmp1). HeldStream emits per-iter wait_front, never pops — matches the lifecycle semantics on paper. May still over-wait subtly.
  • Next steps: retry with all reconfig flags ::None (no format reconfig under BF16); revert init to compute_kernel_hw_startup(cb_divisor, cb_divisor, cb_tmp1) matching the first chain; bisect by migrating only the recip pre-loop stage and leaving the per-iter blocks raw, then add muls one at a time.

template <
    bool WeightHas,
    bool BiasHas,
    bool NeedsTypecast,
    uint32_t TcInFmt,
    uint32_t TcOutFmt,
    uint32_t cb_bcast,
    uint32_t cb_other,
    uint32_t cb_batch_var,
    uint32_t cb_eps,
    uint32_t cb_den,
    uint32_t cb_weight,
    uint32_t cb_bias,
    uint32_t cb_output_0,
    uint32_t cb_output_final>
ALWI void batchnorm_bcast_tiles(uint32_t freq, uint32_t tile_start) {
    using namespace compute_kernel_lib;

    // Stage 1: cb_den = rsqrt(cb_batch_var + cb_eps).
    eltwise_chain(
        1,
        CopyTile<cb_batch_var, Dst::D0, Bulk, OperandKind::Scalar, CopyTileReconfig::Input>{},
        CopyTile<cb_eps, Dst::D1, CallerManaged, OperandKind::Scalar, CopyTileReconfig::Input>{},
        AddBinary<Dst::D0, Dst::D1, Dst::D0>{},
        Rsqrt<>{},
        PackTile<cb_den, Dst::D0, OutStreaming, OperandKind::Scalar, PackTileReconfig::Output>{});

    const uint32_t inner_count = freq - tile_start;

    constexpr uint32_t cb_final_out = NeedsTypecast ? cb_output_final : cb_output_0;

    // Stage 2..5 fused — DEST[0] threaded through Sub → Mul(den) → [Mul(weight)] → [Add(bias)] → [Typecast] → Pack.
    eltwise_chain(
        inner_count,
        CopyTile<cb_other, Dst::D0, Streaming, OperandKind::Scalar, CopyTileReconfig::Input>{},
        CopyTile<cb_bcast, Dst::D1, Bulk,      OperandKind::Scalar, CopyTileReconfig::Input>{},
        SubBinary<Dst::D0, Dst::D1, Dst::D0>{},
        CopyTile<cb_den,   Dst::D1, Bulk,      OperandKind::Scalar, CopyTileReconfig::Input>{},
        MulBinary<Dst::D0, Dst::D1, Dst::D0>{},
        OptionalChainElement<
            WeightHas,
            CopyTile<cb_weight, Dst::D1, Bulk,    OperandKind::Scalar, CopyTileReconfig::Input>>{},
        OptionalChainElement<WeightHas, MulBinary<Dst::D0, Dst::D1, Dst::D0>>{},
        OptionalChainElement<
            BiasHas,
            CopyTile<cb_bias,   Dst::D1, Bulk,    OperandKind::Scalar, CopyTileReconfig::Input>>{},
        OptionalChainElement<BiasHas, AddBinary<Dst::D0, Dst::D1, Dst::D0>>{},
        OptionalChainElement<NeedsTypecast, Typecast<TcInFmt, TcOutFmt, Dst::D0>>{},
        PackTile<cb_final_out, Dst::D0, OutStreaming, OperandKind::Scalar, PackTileReconfig::Output>{});
}

    constexpr bool bias_has_value = get_compile_time_arg_val(1) == 1;

    constexpr auto cb_batch_mean   = get_compile_time_arg_val(3);
    constexpr auto cb_output_0     = get_compile_time_arg_val(4);
    constexpr auto cb_batch_var    = get_compile_time_arg_val(5);
    constexpr auto cb_eps          = get_compile_time_arg_val(6);
    constexpr auto cb_den          = get_compile_time_arg_val(7);
    constexpr auto cb_weight       = get_compile_time_arg_val(8);

    constexpr uint32_t tc_out_fmt  = get_compile_time_arg_val(14);

    cb_wait_front(cb_eps, 1);

    const uint32_t complete_iterations  = (num_tiles + tile_start) / tile_freq;
    const uint32_t remaining_iterations = (num_tiles + tile_start) % tile_freq;

    for (uint32_t i = 0; i < complete_iterations; ++i, tile_start = 0) {
        batchnorm_bcast_tiles<
            weight_has_value, bias_has_value,
            needs_output_typecast, tc_in_fmt, tc_out_fmt,
            cb_batch_mean, cb_input, cb_batch_var, cb_eps, cb_den,
            cb_weight, cb_bias, cb_output_0, cb_output_final>(tile_freq, tile_start);
    }
    if (remaining_iterations > 0) {
        batchnorm_bcast_tiles<…>(remaining_iterations, tile_start);
    }
    cb_pop_front(cb_eps, 1);
}

Decisions — hover for tooltip, click chips above to focus the code

  • 1 · Function CB args → template params. Chain template parameters require constexpr CB ids; the original passed all 11 CBs as uint32_t runtime args. Lifted everything (WeightHas, BiasHas, NeedsTypecast, TcInFmt, TcOutFmt, and 9 CBs) into the function template. Calls from kernel_main pass them as template args.
  • 2 · Held-tile wait/pop absorbed via Bulk + Scalar. Every held single-tile operand (cb_bcast, cb_den, cb_weight, cb_bias) uses Bulk + Scalar — window_1d<Scalar> collapses Bulk's window to 1, so each CopyTile emits exactly cb_wait_front(cb, 1) at the chain head and cb_pop_front(cb, 1) at the tail. The initial migration kept these as CallerManaged with 8 external wait/pop lines around the chain; the simplification (50d82ae55a9) drops all eight. cb_batch_var (used only in Stage 1) was already Bulk + Scalar.
  • 3 · 5 stages → 1 fused chain. Original ran each of Sub, Mul(den), Mul(weight), Add(bias), Typecast in a separate tile_regs_acquire / commit / release window and staged the running result through cb_tmp_1. The fused chain threads DEST[0] across all stages: CopyTiles load the new operand into D1, SFPU binaries consume (D0, D1) and write D0. Single DEST window. Stage 1 (rsqrt) stays its own chain — it produces cb_den which is then consumed across the iter loop.
  • 4 · cb_tmp_1 dropped from the function. No longer needed because DEST holds the running result. The host's CT-arg slot 9 is preserved as a no-op for ABI compatibility with the program factory.
  • 5 · Init: compute_kernel_hw_startup. Convention in already-migrated kernels (batch_norm_kernel.cpp FPU FULL). Original used unary_op_init_common(cb_other, cb_output_0) — unary-shaped init that didn't match the actual binary-Sfpu work. The chain emits its own per-element inits, so kernel-level startup just needs to set the initial unpacker/pack formats.
  • 6 · OptionalChainElement gates weight / bias / typecast — AND absorbs their wait/pop. Three independent compile-time booleans (WeightHas, BiasHas, NeedsTypecast) collapse to no-op tags when false — eliminates an 8-way constexpr-if branching in the chain body. Crucially, OptionalChainElement<false, Inner> inherits a tag with cb_a_id() = 0 and a_policy() = CallerManaged, so the chain emits ZERO wait/pop on the inactive branch — same compile-time gating as the original constexpr-if'd cb_wait_front(cb_weight, 1), just absorbed into the chain.

Decisions — hover for tooltip, click chips above to focus the code

  • 1 · cb_tmp1/2/3 dropped (ABI slot kept). Original used three intermediate CBs (cb_tmp1, cb_tmp2, cb_tmp3) to bridge separate tile_regs_acquire windows. The fused chain keeps the running result in DEST across all stages, so the temps are no longer needed. CT-arg slots 11..13 are kept as no-op placeholders for program-factory ABI compatibility.
  • 2 · 4 stages → 1 chain across 3 DEST slots. Original sequenced (1-momentum), (momentum × batch), ((1-momentum) × old), (sum) as four separate tile_regs_acquire windows with intermediate CB writes. The chain version uses D0/D1/D2: D0 accumulates the (1-momentum)·old branch, D1 accumulates the momentum·batch branch (with D2 holding momentum), then a final Add. DEST_AUTO_LIMIT >= 4 so the 3-slot fusion is always safe.
  • 3 · Per-stat CB lifecycles. cb_one and cb_momentum are held by kernel_main across all iters (CallerManaged + Scalar inside the chains, external wait/pop in kernel_main). cb_batch_mean is wait/popped per iter regardless of branch (original reader contract — one tile per output even when the mean block is skipped). cb_old_running_* and cb_batch_var are Bulk + Scalar (chain emits 1-tile wait+pop via window_1d<Scalar>).
  • 4 · Dual-pack via constexpr-if (not OptionalChainElement). cb_out0 receives the "last computed stat" — var if both, else mean. The chain has TWO PackTile<…, D0> elements: one to cb_updated_running_mean (OutStreaming = reserve+pack+push) and one to cb_out0 (OutCallerManaged = pack only, kernel_main does reserve+push around both stats). Discovered quirk: wrapping a PackTile in OptionalChainElement<false, …> breaks chain_pack_writes_collide's SFINAE probe — fell back to constexpr-if at the call site to build two chain shapes (with vs without the cb_out0 pack).
  • 5 · maybe_typecast_stat inlined as a chain. Original used a maybe_typecast_stat helper that did its own tile_regs_acquire / commit. In the migrated version it's a separate eltwise_chain call gated by the same constexpr boolean, with the same CopyTile + Typecast + PackTile shape — but built from chain primitives, no helper indirection.
  • 6 · Init: compute_kernel_hw_startup. Replaces the original unary_op_init_common(cb_batch_mean, cb_out0) + the manual last_srca_cb = cb_batch_mean tracking (chain handles all per-element reconfig internally).
