01 The measured perf diff

KV-gather all-gathers the latent cache [1,1,T,576] every layer — bytes grow with context. qr keeps KV stationary and rings the fixed queries [1,128,512,576] + reduces the output. On the SP=4 ring, KV cost doubles per doubling of T while qr stays flat. This is the classic per-head-Q MVP wire (the pessimistic one); the qr-latent format (Phase 4) is smaller still.

02 Gates

G1top-k=2048 qualitypass

Faithful G1 needs the 671B DeepSeek checkpoint (out of window); random weights spuriously fail. Synthetic mechanism check clears the bar in the trained/peaked regime and correctly fails on uniform. Authoritative: prod KV-gather already ships top-k=2048 — the scheme's numerical basis is established. Not a blocker.

G2host flash-merge numericspass

Per-shard (O,m,l) partials merged with raw-max online softmax reproduce the full-T golden: fp32 = 1.000000, bf16 = 0.99998, and the zero-hit-shard identity partial (O=0,m=−∞,l=0) merges at 1.0 — across sp = 2/4/8. Reduce formula + hang-safety identity proven before any fabric.

G3zero-hit / empty-shard rowopen

Not needed with no fabric collective (no hop-count deadlock class). But the op still requires ≥1 valid key per row, so a query whose top-k all miss a shard needs an identity partial emitted in-kernel — the one remaining op-hardening item before the model wiring.

G4Galaxy break-evenpartial

Transport crossover measured on SP=4 (~150K). Full SP=8×TP=4 Galaxy timing + indexer-included break-even remain — this box tops out at 8 chips.

03 Fabric-bringup risk — hit, as predicted

First run opened an ad-hoc (1,4) mesh → Fabric Router Sync timeout (ethernet handshake incomplete on that device subset). It self-aborted at the 10 s router-sync guard — a clean failure, not a silent hang. Fixed by opening the native (2,4) LoudBox mesh + 2D sharding along the SP axis (the known-good smoke-test topology). Lesson for Phase 3: pin fabric to the native 2×4 / its submeshes; don't hand-roll device subsets.
