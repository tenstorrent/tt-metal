# AI Development Guidelines

This document outlines the standards for using AI agents and tools (such as Cursor) within the `tt-metal` repository, as established in Issue #38803.

## Standardized File Locations

To maintain repository hygiene and ensure that AI-specific configurations are easily discoverable and maintainable, the following directory structure is enforced:

### 1. Project-Specific Context (`.cursor/commands/`)
Files that contain context specific to `tt-metal` (e.g., build targets, directory layouts, hardware-specific dispatch logic) should reside in the `.cursor/commands/` directory within this repository.

- **Usage**: Use this for recipes, migration scripts, and operational shortcuts that only apply to `tt-metal`.
- **Format**: `.md` files containing instructions for the Cursor agent.

### 2. Generic AI Agents (`tenstorrent/.github-private`)
Generic, reusable AI agent definitions that can be applied across multiple Tenstorrent repositories (e.g., generic PR review agents, coding style checkers) should be proposed to the centralized `tenstorrent/.github-private` repository.

- **Usage**: Use this for high-level "persona" definitions or logic that is shared across the organization.

### 3. Local Experimental Context (`.agent/`)
For contributors using advanced agentic workflows (like the Antigravity agent), temporary research caches, bounty notes, and local implementation plans should be kept in a `.agent/` directory at the root of the repository.

- **Note**: The `.agent/` directory is typically ignored by `.gitignore` to prevent polluting the repository with transient research data.

## Bug Bounty Restrictions

As per the [CONTRIBUTING.md](../CONTRIBUTING.md#bug-bounty-program---ai-tool-restrictions), the use of autonomous AI agents to claim or request assignment of bug bounty issues is **strictly prohibited**.

- Contributors are encouraged to use AI for development assistance (code generation, refactoring, documentation).
- All AI-assisted content must be personally reviewed and manually posted by the human contributor.

## Best Practices

- **Transparency**: If a significant portion of a PR was generated by an AI, it is good practice to mention it in the PR description.
- **Verification**: Never commit AI-generated code without local verification (builds and tests).
- **Human Responsibility**: The human contributor takes full responsibility for the quality and correctness of the code submitted.
