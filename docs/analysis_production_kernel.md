### Mutual exclusivity and co-occurrence rules

1. **PACK_RELU and SFPU_OP_* are mutually exclusive.** The factory uses an if/else: RELU -> PACK_RELU; other activation -> SFPU_OP_*. However, the kernel code does NOT enforce this -- it tests them independently.
2. **FP32_DEST_ACC_EN and PACKER_L1_ACC often co-occur** in the kernel code (combined in `#if defined FP32_DEST_ACC_EN or defined PACKER_L1_ACC` guards). They're set independently but trigger the same `pack_reconfig_data_format` call.
3. **MATMUL_DRAM_SHARDED only appears with SKIP_COMPUTE** in the same two DRAM-sharded factories. However, they're independently conditioned.
4. **FUSE_BIAS requires either no activation, PACK_RELU, or SFPU_OP_*** -- all three combinations are valid.
5. **SKIP_COMPUTE + FUSE_BIAS is a valid combination** (bias-only path, no matmul computation).

## Section B -- Code Path Decision Tree

### High-level kernel structure

```
kernel_main():
  [MATMUL_DRAM_SHARDED: early exit if not worker core]

  Parse compile-time args (lines 148-178)
  Set up CB IDs:
    in0_cb = in0_transpose_tile ? cb_in0_transposed : cb_in0
    in1_cb = cb_in1
    out_cb = cb_out
    mm_partials_cb = cb_intermed0
    untilize_mode_out_cb = untilize_out ? mm_partials_cb : out_cb
    [FUSE_BIAS: mm_out_cb = mm_partials_cb]
    [!FUSE_BIAS: mm_out_cb = untilize_mode_out_cb]
    [FUSE_BIAS: bias_cb = cb_bias]

  [SFPU_OP_INIT_ACTIVATION: SFPU_OP_INIT_ACTIVATION macro call]
  [IN1_TRANSPOSE_TILE: in1_transpose_tile = true; else false]

  mm_block_init(in0_cb, in1_cb, mm_partials_cb, in1_transpose_tile, osw, osh, bw)

  FOR b in batch:
    [get_batch_from_reader: mailbox check, skip if invalid]
    FOR bh in num_blocks_h_dim:
      FOR bw in num_blocks_w_dim:
        ---- INNER COMPUTE BLOCK (see tree below) ----
        ---- BIAS PHASE (if FUSE_BIAS) ----
        ---- UNTILIZE PHASE (if untilize_out) ----
        ---- RECONFIGURE (if more iterations) ----
```

### Inner compute block decision tree

```
ENTRY: enable_reload=false, out_num_tiles_to_wait=out_subblock_num_tiles

[PACK_RELU && (batch>1 || H>1 || W>1)]:
  PACK(llk_pack_relu_config(NO_RELU))    -- disable relu for intermediates

[(batch>1 || H>1 || W>1)]:
  PACK(pack_reconfig_data_format(mm_partials_cb))

FOR block in num_blocks_inner_dim:
  last_out = (block == K_blocks - 1)

  [!FUSE_BIAS && PACK_RELU && last_out]:
    PACK(llk_pack_relu_config(ZERO_RELU))    -- enable relu for final output

  [in0_transpose_tile]:  -- (compile-time arg, not #ifdef)
    transpose_wh_init_short(in0_transpose_cb)
    PACK(pack_reconfig_data_format(in0_cb))
    [PACKER_L1_ACC]: PACK(llk_pack_reconfig_l1_acc(0))    -- disable L1 acc during transpose packing
    transpose_tile_block<in0_block_num_tiles>(in0_transpose_cb, in0_cb)
    mm_block_init_short(in0_cb, in1_cb, in1_transpose, osw, osh, bw)
    PACK(pack_reconfig_data_format(mm_partials_cb))

  cb_wait_front(in0_cb, in0_block_num_tiles)
  cb_wait_front(in1_cb, in1_block_num_tiles)

  FOR in0_subblock in in0_num_subblocks:
    FOR in1_subblock in in1_num_subblocks:
      tile_regs_acquire()

      [enable_reload]:  -- Reload partials from interm CB to dst
        copy_tile_to_dst_init_short_with_dt(in1_cb, mm_partials_cb)
        cb_wait_front(mm_partials_cb, out_subblock_num_tiles)
        copy_block_matmul_partials(mm_partials_cb, 0, 0, out_subblock_num_tiles)
        cb_pop_front(mm_partials_cb, out_subblock_num_tiles)
        mm_block_init_short_with_dt(in0_cb, in1_cb, mm_partials_cb, ...)

      [!SKIP_COMPUTE]:  -- Actual matmul
        FOR k in in0_block_w:
          matmul_block(in0_cb, in1_cb, in0_idx, in1_idx, dst_idx, in1_transpose, osw, osh, bw)

      IF last_out:
        [!FUSE_BIAS && SFPU_OP_INIT_ACTIVATION]:
          FOR i in out_tiles: SFPU_OP_FUNC_ACTIVATION
        tile_regs_commit()
        cb_reserve_back(mm_out_cb, out_tiles)
        tile_regs_wait()
        [FP32_DEST_ACC_EN || PACKER_L1_ACC]:
          PACK(pack_reconfig_data_format(mm_out_cb))
        [PACKER_L1_ACC]:
          [FUSE_BIAS]:
            block==0? llk_pack_reconfig_l1_acc(0) : llk_pack_reconfig_l1_acc(1)
          [!FUSE_BIAS]:
            llk_pack_reconfig_l1_acc(0)
        pack_tile_block(0, mm_out_cb, out_tiles)
        tile_regs_release()
        cb_push_back(mm_out_cb, out_tiles)

      ELSE (not last_out):
        tile_regs_commit()
        if block==0: cb_reserve_back(out_cb, out_num_tiles_to_wait); out_num_tiles_to_wait += out_tiles
        cb_reserve_back(mm_partials_cb, out_tiles)
        tile_regs_wait()
        [PACKER_L1_ACC]:
          block==0? llk_pack_reconfig_l1_acc(0)
          block==1? llk_pack_reconfig_l1_acc(1)
          in0_transpose_tile? llk_pack_reconfig_l1_acc(1)  -- re-enable after transpose disabled it
        pack_tile_block(0, mm_partials_cb, out_tiles)
        tile_regs_release()
        cb_push_back(mm_partials_cb, out_tiles)

  -- After subblock loops, before next K block:
  [PACKER_L1_ACC]:
    [FUSE_BIAS]:
      if block < K-1: cb_wait_front(interm, out_block_tiles); cb_pop_front(interm, out_block_tiles)
      enable_reload = false    -- NEVER reload with L1_ACC + FUSE_BIAS
    [!FUSE_BIAS]:
      if block < K-2: cb_wait_front(interm, out_block_tiles); cb_pop_front(interm, out_block_tiles)
      if block == K-2: enable_reload = true    -- reload only on final block
  [!PACKER_L1_ACC]:
    if spill: enable_reload = true    -- reload every block

  cb_pop_front(in0_cb, in0_tiles)
  cb_pop_front(in1_cb, in1_tiles)
```

### Bias phase decision tree (after K-loop, only when FUSE_BIAS)

```
[FUSE_BIAS]:
  [PACK_RELU]: PACK(llk_pack_relu_config(ZERO_RELU))
  [FP32_DEST_ACC_EN || PACKER_L1_ACC]: PACK(pack_reconfig_data_format(out_cb))
  [PACKER_L1_ACC]: PACK(llk_pack_reconfig_l1_acc(0))

  reconfig_data_format(in1_cb, mm_partials_cb, in0_cb, bias_cb)
  add_bcast_rows_init_short(mm_partials_cb, bias_cb)
  cb_wait_front(bias_cb, in1_block_w)

  FOR in0_subblock in in0_num_subblocks:
    FOR in1_subblock in in1_num_subblocks:
      cb_wait_front(mm_partials_cb, out_tiles)
      tile_regs_acquire()
      FOR j in out_subblock_h:
        FOR k in out_subblock_w:
          add_tiles_bcast_rows(mm_partials_cb, bias_cb, tile_i, bias_tile, tile_i)

      [!SFPU_OP_INIT_ACTIVATION]:
        tile_regs_commit()

      cb_pop_front(mm_partials_cb, out_tiles)

      [SFPU_OP_INIT_ACTIVATION]:
        FOR i in out_tiles: SFPU_OP_FUNC_ACTIVATION
        tile_regs_commit()

      cb_reserve_back(untilize_mode_out_cb, out_tiles)
      tile_regs_wait()
      FOR i in out_tiles: pack_tile(i, untilize_mode_out_cb)
      tile_regs_release()
      cb_push_back(untilize_mode_out_cb, out_tiles)

  [num_blocks_w > 1]: cb_pop_front(bias_cb, in1_block_w)
```

### Untilize phase decision tree (after bias or matmul, when untilize_out)

```
[untilize_out]:
  [PACK_RELU]: PACK(llk_pack_relu_config(NO_RELU))
  [!FUSE_BIAS]:
    reconfig_data_format_srca(in1_cb, mm_partials_cb)
    [FP32_DEST_ACC_EN || PACKER_L1_ACC]: PACK(pack_reconfig_data_format(out_cb))
    [PACKER_L1_ACC]: PACK(llk_pack_reconfig_l1_acc(0))
  pack_untilize_dest_init<osw, out_block_w>(out_cb)
  copy_tile_to_dst_init_short(mm_partials_cb)
  FOR in0_subblock_i in in0_num_subblocks:
    reblock_and_untilize<osw, out_block_w>(in1_num_subblocks, out_tiles, osh, mm_partials_cb, out_cb)
  pack_untilize_uninit(mm_partials_cb)
```

### Reconfigure at end of bh/bw iteration

```
[(batch>1 || W>1 || H>1)]:
  [FUSE_BIAS]:
    reconfig_data_format(mm_partials_cb, in1_cb, bias_cb, in0_cb)
  [!FUSE_BIAS]:
    reconfig_data_format_srca(mm_partials_cb, in1_cb)
  mm_block_init_short(in0_cb, in1_cb, in1_transpose, osw, osh, bw)
```

## Section C -- LLK Call Sequences

Each path below shows the LLK API calls in order for one full (bh, bw) iteration (one output block). Intermediate loop variables are omitted for clarity.

### Path 1: Base (no defines, single K-block)

```
mm_block_init(in0, in1, interm, false, osw, osh, bw)
...
  cb_wait_front(in0, tiles)
  cb_wait_front(in1, tiles)
  tile_regs_acquire()
  FOR k: matmul_block(in0, in1, ...)
  tile_regs_commit()
  cb_reserve_back(out, tiles)
  tile_regs_wait()
  pack_tile_block(0, out, tiles)
  tile_regs_release()
  cb_push_back(out, tiles)
  cb_pop_front(in0, tiles)
  cb_pop_front(in1, tiles)
```

### Path 2: Base with spill/reload (no defines, K>1)

```
mm_block_init(in0, in1, interm, false, osw, osh, bw)
...
  # Block 0:
  cb_wait_front(in0, tiles); cb_wait_front(in1, tiles)
  tile_regs_acquire()
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(out, out_tiles)       # prevent overlap
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  pack_tile_block(0, interm, out_tiles) # spill
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_pop_front(in0, tiles); cb_pop_front(in1, tiles)

  # Block 1..K-2: (same + reload)
  cb_wait_front(in0, tiles); cb_wait_front(in1, tiles)
  tile_regs_acquire()
  copy_tile_to_dst_init_short_with_dt(in1, interm)  # reload
  cb_wait_front(interm, out_tiles)
  copy_block_matmul_partials(interm, 0, 0, out_tiles)
  cb_pop_front(interm, out_tiles)
  mm_block_init_short_with_dt(in0, in1, interm, ...)
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  pack_tile_block(0, interm, out_tiles) # spill
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_pop_front(in0, tiles); cb_pop_front(in1, tiles)

  # Block K-1 (last):
  cb_wait_front(in0, tiles); cb_wait_front(in1, tiles)
  tile_regs_acquire()
  [reload from interm]
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(out, out_tiles)
  tile_regs_wait()
  pack_tile_block(0, out, out_tiles)    # final output
  tile_regs_release()
  cb_push_back(out, out_tiles)
  cb_pop_front(in0, tiles); cb_pop_front(in1, tiles)
```

### Path 3: PACKER_L1_ACC, no FUSE_BIAS

```
mm_block_init(in0, in1, interm, false, osw, osh, bw)
...
  # Block 0:
  tile_regs_acquire()
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(out, out_tiles)
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  PACK(llk_pack_reconfig_l1_acc(0))     # no accumulation first block
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_pop_front(in0); cb_pop_front(in1)

  # Block 1..K-3: (L1 accumulates, no explicit reload)
  tile_regs_acquire()
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  PACK(llk_pack_reconfig_l1_acc(1))     # enable L1 accumulation
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_wait_front(interm, out_block_tiles) # advance FIFO pointer
  cb_pop_front(interm, out_block_tiles)
  cb_pop_front(in0); cb_pop_front(in1)

  # Block K-2: same as above but sets enable_reload=true (no cb_wait/pop)

  # Block K-1 (last): reload + pack to output
  tile_regs_acquire()
  copy_tile_to_dst_init_short_with_dt(in1, interm)
  cb_wait_front(interm, out_tiles)
  copy_block_matmul_partials(interm, 0, 0, out_tiles)
  cb_pop_front(interm, out_tiles)
  mm_block_init_short_with_dt(in0, in1, interm, ...)
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(out, out_tiles)        # changed: pack to out_cb (mm_out_cb = untilize_mode_out_cb)
  tile_regs_wait()
  PACK(pack_reconfig_data_format(out))
  PACK(llk_pack_reconfig_l1_acc(0))      # disable L1 acc for final output
  pack_tile_block(0, out, out_tiles)
  tile_regs_release()
  cb_push_back(out, out_tiles)
  cb_pop_front(in0); cb_pop_front(in1)
```

**Key difference from base spill/reload**: With PACKER_L1_ACC, intermediate blocks don't need explicit reload into DST. The packer accumulates in L1 directly. Only the last block reloads (to pack to the output CB instead of interm CB). The cb_wait_front/cb_pop_front between blocks just advances the FIFO pointer for the L1 accumulator.

### Path 4: PACKER_L1_ACC + FUSE_BIAS

```
mm_block_init(in0, in1, interm, false, osw, osh, bw)
...
  # Block 0:
  tile_regs_acquire()
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)     # mm_out_cb = interm (FUSE_BIAS)
  tile_regs_wait()
  PACK(pack_reconfig_data_format(interm))
  PACK(llk_pack_reconfig_l1_acc(0))      # no accumulation first
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_wait_front(interm, out_block_tiles) # advance FIFO pointer
  cb_pop_front(interm, out_block_tiles)
  cb_pop_front(in0); cb_pop_front(in1)

  # Block 1..K-2:
  tile_regs_acquire()
  FOR k: matmul_block(...)               # enable_reload is always false!
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  PACK(pack_reconfig_data_format(interm))
  PACK(llk_pack_reconfig_l1_acc(1))      # enable L1 accumulation
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  cb_wait_front(interm, tiles); cb_pop_front(interm, tiles)
  cb_pop_front(in0); cb_pop_front(in1)

  # Block K-1 (last): L1 acc packs final matmul result to interm (NOT output)
  tile_regs_acquire()
  FOR k: matmul_block(...)
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)     # still interm! (mm_out_cb = interm)
  tile_regs_wait()
  PACK(pack_reconfig_data_format(interm))
  PACK(llk_pack_reconfig_l1_acc(1))      # still accumulating!
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)
  # NO cb_wait/cb_pop for last block (data stays for bias phase)
  cb_pop_front(in0); cb_pop_front(in1)

  # Bias phase:
  PACK(llk_pack_relu_config(ZERO_RELU))  # [PACK_RELU]
  PACK(pack_reconfig_data_format(out_cb))
  PACK(llk_pack_reconfig_l1_acc(0))
  reconfig_data_format(in1, interm, in0, bias)
  add_bcast_rows_init_short(interm, bias)
  cb_wait_front(bias, in1_block_w)
  FOR subblocks:
    cb_wait_front(interm, out_tiles)
    tile_regs_acquire()
    FOR tiles: add_tiles_bcast_rows(interm, bias, i, bias_i, i)
    [SFPU_OP: FOR tiles: SFPU_OP_FUNC_ACTIVATION]
    tile_regs_commit()
    cb_pop_front(interm, out_tiles)
    cb_reserve_back(out, out_tiles)
    tile_regs_wait()
    FOR tiles: pack_tile(i, out)
    tile_regs_release()
    cb_push_back(out, out_tiles)
```

**Critical difference**: With PACKER_L1_ACC + FUSE_BIAS, `enable_reload` is ALWAYS false. The L1 accumulator builds up partial results across all K blocks, and the last block's pack is the final accumulated result. This goes to interm_cb, and the bias phase then reads from interm_cb. This is different from the non-FUSE_BIAS L1_ACC path, where the last block reloads to pack to output.

### Path 5: FUSE_BIAS without PACKER_L1_ACC

```
mm_block_init(in0, in1, interm, false, osw, osh, bw)
...
  # K-loop: same as base spill/reload, but mm_out_cb = interm
  # All blocks pack to interm (including last_out)
  # enable_reload = true after first block (standard spill/reload)

  # Last block packs to interm (same as non-last blocks):
  tile_regs_commit()
  cb_reserve_back(interm, out_tiles)
  tile_regs_wait()
  pack_tile_block(0, interm, out_tiles)
  tile_regs_release()
  cb_push_back(interm, out_tiles)

  # Bias phase: same as Path 4 bias section, minus L1_ACC reconfig
  reconfig_data_format(in1, interm, in0, bias)
  add_bcast_rows_init_short(interm, bias)
  cb_wait_front(bias, in1_block_w)
  FOR subblocks:
    cb_wait_front(interm, out_tiles)
    tile_regs_acquire()
    FOR tiles: add_tiles_bcast_rows(...)
    tile_regs_commit()
    cb_pop_front(interm, out_tiles)
    cb_reserve_back(out, out_tiles)
    tile_regs_wait()
    FOR tiles: pack_tile(i, out)
    tile_regs_release()
    cb_push_back(out, out_tiles)
```

### Path 6: SFPU_OP_INIT_ACTIVATION without FUSE_BIAS

```
# Same as base, but on last block after matmul:
  FOR k: matmul_block(...)
  # Apply SFPU activation before packing
  FOR i in out_tiles: SFPU_OP_FUNC_ACTIVATION
  tile_regs_commit()
  cb_reserve_back(out, out_tiles)
  tile_regs_wait()
  pack_tile_block(0, out, out_tiles)
  tile_regs_release()
  cb_push_back(out, out_tiles)
```

### Path 7: SFPU_OP_INIT_ACTIVATION with FUSE_BIAS

```
# Matmul K-loop: identical to FUSE_BIAS path (no activation during matmul)
# Bias phase: activation is applied AFTER bias add:
  cb_wait_front(interm, out_tiles)
  tile_regs_acquire()
  FOR tiles: add_tiles_bcast_rows(...)
  # Note: tile_regs_commit() is NOT called before SFPU (different from !SFPU path)
  FOR i in out_tiles: SFPU_OP_FUNC_ACTIVATION
  tile_regs_commit()      # commit AFTER activation
  cb_pop_front(interm, out_tiles)
  cb_reserve_back(out, out_tiles)
  tile_regs_wait()
  FOR tiles: pack_tile(i, out)
  tile_regs_release()
  cb_push_back(out, out_tiles)
```

**Subtle detail**: The `tile_regs_commit()` placement differs based on SFPU_OP_INIT_ACTIVATION:
- Without SFPU: commit immediately after bias add (line 435-436)
- With SFPU: commit AFTER activation (lines 444-445)
This is because SFPU operates on DST registers, so the commit must happen after both bias-add and activation are done.

### Path 8: SKIP_COMPUTE

```
# Same structure as other paths, but the matmul_block calls are skipped:
  tile_regs_acquire()
  [enable_reload: reload from interm]
  # NO matmul_block calls
  [last_out: pack to mm_out_cb]
  [!last_out: spill to interm]
```

This is used for bias-only paths where the matmul result is pre-computed or where only bias needs to be applied.

### Path 9: in0_transpose_tile (compile-time arg)

```
# Before each K-block's matmul, transpose in0 tiles:
  transpose_wh_init_short(in0_transpose_cb)
  PACK(pack_reconfig_data_format(in0_cb))
  [PACKER_L1_ACC]: PACK(llk_pack_reconfig_l1_acc(0))  # must disable L1 acc to pack transpose result
  transpose_tile_block<in0_block_num_tiles>(in0_transpose_cb, in0_cb)
    # internally: for blocks of 4 tiles:
    #   cb_wait_front(transpose_cb, 4)
    #   tile_regs_acquire()
    #   transpose_wh_tile(transpose_cb, i, i) x4
    #   tile_regs_commit()
    #   cb_pop_front(transpose_cb, 4)
    #   cb_reserve_back(in0_cb, 4)
    #   tile_regs_wait()
    #   pack_tile(i, in0_cb) x4
    #   tile_regs_release()
    #   cb_push_back(in0_cb, 4)
  mm_block_init_short(in0_cb, in1_cb, ...)
  PACK(pack_reconfig_data_format(mm_partials_cb))
  # Then proceed with normal matmul on transposed data

# PACKER_L1_ACC interaction: because transpose_tile_block packs to in0_cb with
# L1_ACC disabled, the non-last block path must re-enable L1_ACC:
  [PACKER_L1_ACC, !last_out, in0_transpose_tile]:
    PACK(llk_pack_reconfig_l1_acc(1))  # re-enable after transpose disabled it
```

### Path 10: untilize_out

```
# After matmul (and optional bias phase):
  [PACK_RELU]: PACK(llk_pack_relu_config(NO_RELU))
  [!FUSE_BIAS]:
    reconfig_data_format_srca(in1_cb, mm_partials_cb)
    [FP32_DEST_ACC_EN || PACKER_L1_ACC]: PACK(pack_reconfig_data_format(out_cb))
    [PACKER_L1_ACC]: PACK(llk_pack_reconfig_l1_acc(0))
  pack_untilize_dest_init<osw, out_block_w>(out_cb)
  copy_tile_to_dst_init_short(mm_partials_cb)
  FOR in0_subblock_i:
    reblock_and_untilize<osw, out_block_w>(...)
      # internally: reads tiles from interm, copies to dst, pack_untilize_dest
  pack_untilize_uninit(mm_partials_cb)
```

Note: when untilize_out + FUSE_BIAS, the bias phase packs to `untilize_mode_out_cb = mm_partials_cb`, and then the untilize phase reads back from mm_partials_cb. When untilize_out + !FUSE_BIAS, the matmul last_out also packs to `mm_partials_cb` (via untilize_mode_out_cb), then untilize reads from there.

## Section D -- What the Current Helper Covers

### Current `matmul_block` helper (on this branch)

**File**: `ttnn/cpp/ttnn/kernel_lib/matmul_block_helpers.hpp` + `.inl`

**What it covers**:
- Sub-blocked matmul with K-dimension spill/reload
- PostComputeFn for fused SFPU activation on last block
- B-matrix transpose (template param)
- Batch dimension
- Data format reconfiguration at init

**What it DOES NOT cover (compared to production kernel)**:

| Production kernel feature | Helper support | Gap description |
|--------------------------|---------------|-----------------|
| PACKER_L1_ACC | None | Completely different spill/reload strategy. L1_ACC uses `llk_pack_reconfig_l1_acc(0/1)` to toggle hardware accumulation. No CB-based reload needed for intermediate blocks. |
| FP32_DEST_ACC_EN | None | Requires `pack_reconfig_data_format` calls at specific points |
| FUSE_BIAS | None | Requires: (1) routing all matmul output to interm_cb, (2) separate bias-add phase with `add_bcast_rows`, (3) different data format reconfiguration between phases |
| PACK_RELU | None | Requires `llk_pack_relu_config` toggling: disable at start of each (bh,bw) iteration, enable on last K-block |
| SKIP_COMPUTE | None | Skip the matmul_block inner loop entirely |
| in0_transpose_tile | None | Requires `transpose_tile_block` phase before matmul, with pack format reconfig and L1_ACC disable/re-enable |
| untilize_out | None | Requires `reblock_and_untilize` + `pack_untilize_dest` phase after matmul |
| MATMUL_DRAM_SHARDED | None | Early exit for non-worker cores |
| H/W block loops | None | Helper only has K-loop (inner dim). Production kernel has outer bh/bw loops with reconfiguration between iterations |
| PACK_RELU + FUSE_BIAS interaction | None | Relu is delayed until after bias phase when both are active |
| SFPU + FUSE_BIAS interaction | None | SFPU applied after bias add, not after matmul |
| PACKER_L1_ACC + FUSE_BIAS interaction | None | enable_reload is always false; L1_ACC accumulates across all K blocks; final result goes to interm for bias phase |
| PACKER_L1_ACC + in0_transpose interaction | None | Must disable L1_ACC during transpose packing, re-enable after |
| `tile_regs_acquire/commit/wait/release` | Simplified | Helper uses `acquire_dst()`/`release_dst()` (wrappers) but doesn't handle the production kernel's split `commit`/`wait` pattern where commit and wait are separated by CB operations |
| `pack_tile_block` vs `pack_tile` loop | Different | Helper uses individual `pack_tile` calls; production uses `pack_tile_block` |

### Specific line coverage in production kernel

The current helper covers approximately the `#ifndef PACKER_L1_ACC` + `#ifndef FUSE_BIAS` + `#ifndef SKIP_COMPUTE` + `!in0_transpose_tile` + `!untilize_out` path, which corresponds to:
- Lines 203-204 (init)
- Lines 255-301 (wait, subblock loops, matmul_block, non-L1_ACC packing)
- Lines 395-398 (spill enable_reload = true)
- Lines 400-401 (pop inputs)
- This is roughly **100 lines** of the 500-line kernel.

### Failed `matmul_block_fused_bias` helper (llk3 branch)

**File**: `origin/wransom/llk3:ttnn/cpp/ttnn/kernel_lib/matmul_block_fused_bias_helpers.hpp/inl`

**What it attempted to cover**:
- Sub-blocked matmul + fused bias-add in a single helper
- PostComputeFn for SFPU activation after bias add
- B-matrix transpose
- InitUninitMode and ReconfigureRegisterDatatypeMode enums
- Param structs (In0BlockParams, In1BlockParams, OutSubblockParams)

**Why it failed (device hangs)**:

1. **Duplicated matmul code instead of composing with matmul_block helper.** The fused_bias .inl file contains a complete copy of the matmul K-loop (lines 85-160), not a call to the matmul_block helper. This means any fix to the base matmul logic would need to be duplicated.

2. **Missing PACKER_L1_ACC support.** The helper used standard spill/reload (`enable_reload = true` after every block, line 152). It did NOT implement the L1 accumulation path, which fundamentally changes how intermediate results are managed. Since the production deployment likely had PACKER_L1_ACC enabled, this mismatch caused hangs.

3. **Missing data format reconfiguration between phases.** The production kernel has explicit `PACK(pack_reconfig_data_format(...))` calls at specific transitions (matmul -> bias, matmul -> output, etc.). The helper's phase 2 bias-add section (lines 165-211) does reconfigure, but the matmul phase (lines 85-160) lacks the `pack_reconfig_data_format(mm_partials_cb)` that the production kernel does when `batch > 1 || H > 1 || W > 1`.

4. **Incorrect CB memory sharing assumption.** The helper pops interm_cb before reserving out_cb (line 198: `cb_pop_front(interm_cb, out.num_tiles)` before `cb_reserve_back(out_cb, out.num_tiles)`). The production kernel's non-bias path does this differently: it reserves out_cb BEFORE packing to interm_cb (line 343-348), creating a different memory flow.

5. **Param struct design (PR feedback).** Used `In0BlockParams`, `In1BlockParams`, `OutSubblockParams` structs, which were explicitly rejected by PR reviewer. The reviewer said: take independent params, derive computed values internally.

6. **Unused enums (PR feedback).** `InitUninitMode` and `ReconfigureRegisterDatatypeMode` had no call sites using non-default values.

**What it got right**:
- The bias-add phase structure (lines 165-211) correctly implements the `add_bcast_rows` pattern with row broadcast
- The PostComputeFn placement after bias-add is correct
- The data format reconfiguration between matmul and bias phases is approximately correct

### Structural differences from base kernel

**1. Core type system (lines 14, 218-221)**:

**2. Per-batch output and partials CBs (lines 132-179, 210-211)**:
```cpp

constexpr std::array<uint32_t, batch> mm_partials_cb_ids = fill_named_cb_array<batch>(mm_partials_cb_names);
// Each batch gets its own CB pair (cb_mm_out_0..15, cb_mm_partials_0..15)
// Base kernel uses a single out_cb and mm_partials_cb for all batches
```

**3. Different compile-time args (lines 183-203)**:
- `in1_block_size_bytes` (arg 6) -- replaces `in1_block_w`
- `in1_tensor_size_bytes` (arg 7) -- new
- `in1_per_core_w` (arg 8) -- replaces derived value

**4. Ring-based in1 weight gathering (ENABLE_GLOBAL_CB, lines 243-256, 304-315, 429-462)**:

The gathered variant implements a ring-based all-gather for in1 (weight) data across devices:
```cpp
// Per-batch setup:
UNPACK((in1_cb_start_addr = get_local_cb_start_addr(in1_cb_id)));
UNPACK((in1_rd_ptr_start_addr = get_local_cb_rd_ptr(in1_cb_id)));
UNPACK((curr_in1_block_index = ring_idx));
UNPACK((in1_tensor_split = is_tensor_split(in1_cb_id, in1_tensor_size_bytes)));
UNPACK((update_rd_ptr_to_ring_index(in1_cb_id, in1_block_size_bytes, ring_idx, in1_tensor_split)));

// Per K-block: calculate next block index and update rd pointer
UNPACK((calculate_next_block_index_and_update_rd_ptr(...)));

// After all K-blocks:
cb_reserve_back(sync_cb, 1); cb_push_back(sync_cb, 1);  // release in1
UNPACK((update_local_cb_rd_ptr(in1_cb_id, in1_rd_ptr_start_addr)));  // reset
UNPACK((update_rd_ptr_to_ring_index(in1_cb_id, in1_block_size_bytes, ring_size, in1_tensor_split)));
```

This directly manipulates CB FIFO read pointers (`LocalCBInterface.fifo_rd_ptr`) to navigate a ring buffer of weight blocks across multiple devices. This is fundamentally different from the base kernel's simple `cb_wait_front`/`cb_pop_front` pattern.

**5. Sync CBs (lines 207-208, 275-276)**:
```cpp
constexpr uint32_t sync_cb = get_named_compile_time_arg_val("cb_sync");
constexpr uint32_t sync_cb2 = get_named_compile_time_arg_val("cb_sync2");
// sync_cb2: wait before first K-block (in1 received signal)
// sync_cb: signal after all K-blocks (in1 released)
```

**6. Variable in0 block width (lines 223-224, 280)**:
```cpp
const uint32_t* unpadded_in0_shard_widths_in_tiles = (uint32_t*)get_arg_addr(rt_args_idx);
// Per K-block: unpadded_in0_block_w = unpadded_in0_shard_widths_in_tiles[curr_ring_idx]
// This allows different in0 block widths per ring position (for padding)
```

**7. Input CB switching (lines 287, 302)**:
```cpp
const uint32_t input0_cb_id = block == 0 ? in0_cb_id : in2_cb_id;
// First K-block reads from in0_cb_id (local shard)
// Subsequent K-blocks read from in2_cb_id (gathered data)
```

**8. No H/W block loops**: The gathered variant has only `for (b in batch)` > `for (block in num_blocks)`. No bh/bw nesting. This means only one output block per batch.

**9. Untilize is inline in subblock loop (lines 373-399)**:
```cpp
if (last_out) {
    if constexpr (untilize_out) {
        pack_untilize_dest_init<out_subblock_num_tiles>(mm_out_cb_id);
    }
    tile_regs_commit();
    cb_reserve_back(mm_out_cb_id, out_subblock_num_tiles);
    tile_regs_wait();
    if constexpr (untilize_out) {
        pack_untilize_dest<out_subblock_num_tiles>(mm_out_cb_id);
    } else {
        pack_tile_block(start_dst_index, mm_out_cb_id, out_subblock_num_tiles);
    }
    tile_regs_release();
    if constexpr (untilize_out) {
        pack_untilize_uninit(mm_out_cb_id);
    }
    cb_push_back(mm_out_cb_id, out_subblock_num_tiles);

```
This is simpler than the base kernel's separate `reblock_and_untilize` function -- it untilizes per-subblock directly during the pack phase.

**10. PACKER_L1_ACC is simpler (lines 409-415, 429-438)**:
- No FUSE_BIAS interaction (no bias at all)
- No `in0_transpose_tile` interaction
- Standard pattern: `block==0 -> l1_acc(0)`, `block==1 -> l1_acc(1)`, last block reload

**11. Spill condition is different (line 238)**:
```cpp

// Base kernel: spill = num_blocks_inner_dim > 1
// Gathered: also requires multiple subblocks (avoids spill for single-subblock case)
```

**12. in1 indexing differs for non-DRAM, non-global-CB path (lines 319-324)**:
```cpp
#ifdef ENABLE_GLOBAL_CB
    int in1_index_subblock_offset = 0;
#else

#endif
```
When in1 is NOT from DRAM and NOT using global CB, the in1 offset includes the ring index, meaning the full weight tensor is in L1 and indexing navigates to the correct ring position.

### Summary of gathered variant delta

The gathered variant is NOT a simple extension of the base kernel. It is a substantially different kernel that shares the core matmul + pack loop structure but differs in:
1. **Data orchestration**: Ring-based weight gathering with direct CB pointer manipulation
2. **Memory model**: Per-batch CB pairs instead of shared CBs
3. **Feature scope**: No bias fusion, no in0 transpose, no DRAM-sharded mode
4. **Untilize approach**: Inline per-subblock instead of separate phase
5. **Synchronization**: Cross-device sync via sync CBs

A helper redesign should treat the gathered variant as a separate migration target, not as a minor variant of the base kernel. The core matmul loop (subblock iteration, K-blocking, L1_ACC/spill-reload) is shared, but the surrounding orchestration is different enough that the gathered variant would need its own kernel file calling shared helper functions.
