# LLK ISA-Grounded Bug Audit — WH B0 + BH A0 — FINAL

**Status:** COMPLETE (Waves 0–3). **Scope:** WH B0 + BH A0 LLK library + common headers + compute/dataflow API, ISA docs (tenstorrent/tt-isa-documentation) as ground truth; kernel *usage* over a representative sample (eltwise, matmul, reduce, tilize/untilize) — NOT the full ttnn/models corpus; Quasar excluded (no ISA docs); SFPU ops in `tt_metal/hw/ckernels/*/llk_api/llk_sfpu/` excluded (only the tt-llk `common/inc/sfpu/` subtree audited).

**Priority:** P0 confirmed-live · P1 clear defect/very likely · P2 latent (safe only under a sync/caller/HW assumption) · P3 defensive/informational. **Conf:** High/Med/Low (Low = candidate needing maintainer/HW confirmation).

## Executive summary
- **P0:** 0 confirmed always-live bugs.
- **P1 (5):** strongest = **U1** (BH 8-bit tilize emits wrong MOP body via C++ shadowing — High conf logic bug) and **U2** (BH untilize-uninit state leak). Plus **C1** (pack relu full-word clobber), **C3** (`set_packer_strides` non-self-draining root cause), **F1** (WH 32-bit broadcast datacopy may drop low-16 bits — high impact, needs HW confirm).
- **P2 (~21):** led by **S1** (BH `_llk_math_set_fp32_dest_acc_`/`_hw_configure_` omit `WAIT_SFPU` when writing the SFPU-read `SFPU_Fp32_enabled`; the int8-reconfig sites correctly do **not** need it — ISA SFPLOAD/SFPSTORE) and **S2** (BH SFPU `done_with_addrmod_reset` writes the wrong cfg register). Many are Fp8_e4m3 / int8-reconfig / arch-divergence latents.
- **P3 (~14):** dead-code-without-rationale, misleading comments, cosmetic SHAMT pairing, redundant writes.
- **Strong negatives (high confidence "no bug"):** register-encoding census (~270 RMW + ~117 raw writes) clean; ISA contracts (fidelity phases, ADCs, REPLAY/MOP) conform; matmul-reuse loop resolved correct; API-usage sample clean; most WH/BH SFPU divergences are correct-by-design (ADDR_MOD base remap; SM vs 2's-comp int layout).
- **Cross-cutting fix with highest leverage:** **C3** — making `set_packer_strides` self-draining collapses 6 of the 7 reconfig hardening-gaps.

## Resolved / verified-correct (not bugs — recorded to avoid re-chasing)
- **M4 — matmul reuse loop (WH `llk_math_matmul.h:844-975`): NOT A BUG** (High). Dual-`t` step-2 unroll emits 2nd MVMUL only under `(t+1)<t_dim` (odd t_dim correct); SET/CLR(`FlipSrcA/B`)+`SET_ABD`(RWC reset)+`CLEARDVALID` match ISA; reused-operand auto-clear suppressed via `CLR_DVALID_Src*_Disable` for the block, restored in uninit.
- **R1 — superseded by S2** (the BH `SETC16(2,0)` is not merely a magic literal; it's a wrong-register write).
- **ISA contracts CONFORM:** fidelity phases (1 MVMUL/GAPOOL per phase, matmul+reduce); ADCs/SETADCXX/SETADCXY encodings; REPLAY (≤32, no wrap) + MOP bounds.
- **Register-encoding census CLEAN:** 130 WH + 140 BH `cfg_reg_rmw_tensix`, 65 WH + 52 BH raw `cfg[]=`, ~270 REG2FLOP/SETC16/WRCFG/RMWCIB, ~115 SETADC/SETDMAREG — no field-overflow, SHAMT/MASK mismatch (R3 benign), SEC swap, space confusion, REG2FLOP out-of-range, or immediate overflow. Stride values fit field widths both arches.
- **API-usage sample CLEAN:** compute API + ~40 ttnn kernels; all candidates (missing init, unpopped scaler, tile_regs ordering, missing reduce_uninit, missing reconfig) were false positives (correct-by-design/macro-split/resident-scaler/fused-init). Only 2 P3/Low same-dtype *fragility* notes (`rotary_embedding_llama_sharded.cpp`, `ternary_addc_ops_fpu.cpp`).
- **WH/BH SFPU divergences confirmed CORRECT-by-design:** ADDR_MOD_SET_Base 2-bit→4-7 remap (WH) vs 3-bit direct (BH); SM (WH) vs 2's-comp (BH) int DEST layout driving different negate/compare/add-sub; WH manual SFPNOP vs BH auto-interlock; transpose_dest-32b & SFPU addr-mod-base divergences.
