## Test Design Notes

### GAP-59 (FIX QB) — Reset Loop Early Break
- **Differentiator from GAP-58**: GAP-58 tests FIX QC (skip entire safe_assert body for non-MMIO). GAP-59 tests FIX QB (break the inner ETH-core-specific `assert_risc_reset_at_core` loop after first failure). Complementary defenses at different loop levels.
- **Budget**: 60s TESTEE budget catches the 60s+ regression (4 ETH cores × 5s × 3 non-MMIO devices) while passing with the fix (~15-25s).

### GAP-60 (FIX PY+PZ) — Phase 2.5 + Topology Timeout
- **Two-create pattern**: Unique among GAP tests — creates MeshDevice TWICE in the same TESTEE process. First create pays full probe cost; second create should be faster due to FIX PZ's topology cache. Tests both FIX PY (inner-loop relay skip) and FIX PZ (static ASIC cache).
- **Budget**: 90s total for two creates. Without fixes: first create alone could take 150s+.

### GAP-61 (FIX QD) — Dead Router MMIO Skip
- **Crash prevention test**: Primary assertion is no SIGABRT (exit 134). The pre-fix failure mode was AllGather hanging on semaphore wait → TT_THROW TIMEOUT → SIGABRT. FIX QD sets `fabric_channels_not_ready_for_traffic_` so guards can SKIP.
- **Flag verification**: TESTEE checks that at least one device has degraded flags set after init, confirming FIX QD fired.

### GAP-62 (FIX QU) — Flag Re-assertion
- **State management test**: Verifies that `Device::configure_fabric()` resetting flags doesn't lose the degraded state. FIX QU re-asserts flags in `FabricFirmwareInitializer::configure()`.
- **Subtle regression**: Without FIX QU, flags are false after configure_fabric() → test guards see healthy cluster → dispatch to devices with no dispatch kernel → 120s hang → TIMEOUT.
