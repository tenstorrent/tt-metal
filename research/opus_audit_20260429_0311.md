## Gap Identified: FIX NY Has No Test (GAP-46)

### What FIX NY Does

FIX NY (`8b3017d4f2b`, `tt_cluster.cpp/hpp`) adds `relay_broken_chips_` — an `unordered_set<ChipId>` member of `Cluster` — to eliminate per-channel 5s UMD relay timeout stalls.

**Root cause** (CI run 25086219070, job 73503180670 — t3k_ttnn_udm_tests):
- FIX NX (GAP-44) wraps `write_to_device()` in try/catch — prevents exception propagation.
- BUT: UMD's `write_to_non_mmio` does NOT check `relay_broken_` before the 5s polling loop.
- `set_internal_routing_info_for_ethernet_cores` iterates 6 ETH channels per non-MMIO chip.
- 6 channels × 5s UMD timeout = 30s serial stall → GHA 5-minute action timeout → exit 124.

**FIX NY mechanism**:
1. First `write_core(chip N)` fails → FIX NX catches exception → FIX NY inserts `N` into `relay_broken_chips_`.
2. Channels 2–6 for chip N: `relay_broken_chips_.count(N) == 1` → return immediately (0ms).
3. Total stall for T3K (3 non-MMIO chips): 3 × 5s ≈ 15s (vs 90s without FIX NY).

### Why No Prior Test Existed

GAP-44 (`WriteCorRelayGuardFixture`) tests FIX NX with a 45s budget — generous enough to pass even without FIX NY (since exceptions are suppressed by FIX NX, the test just takes longer). GAP-44's purpose was exception propagation; timing was secondary. FIX NY requires a **tighter** timing budget to detect the per-channel stall regression.

## GAP-46 Implementation

**File**: `tests/tt_metal/distributed/test_gap46_fixny_relay_broken_chips_cache.cpp`
**Fixture**: `RelayBrokenChipsCacheFixture`

**Test design**:
1. Fork PREDECESSOR: FABRIC_2D + blank workload → SIGKILL → non-MMIO ERISCs in FABRIC state.
2. 2s settle.
3. Fork TESTEE: open full MeshDevice → MetalContext::initialize() (triggers set_internal_routing_info + watcher init).
4. Check: testee exits within **35s** (with FIX NY: ~27s; without FIX NY: ~102s → timeout).
5. Check: testee exits 0 (secondary — FIX NX regression shows as exit 1).

**Timing budget rationale**:
- With FIX NY: 3 chips × 5s + ~12s normal init = ~27s. Budget margin: +8s.
- Without FIX NY: 18 channels × 5s + ~12s init = ~102s. Regression margin: -67s (clear fail).

**Key distinction from GAP-44**: GAP-44 catches exception path (generous 45s); GAP-46 catches timing path (tight 35s). Both are needed — a FIX NY revert breaks GAP-46 while GAP-44 still passes.
