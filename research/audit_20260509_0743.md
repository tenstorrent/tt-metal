## GAP A: FIX XY-2 clear_relay_broken() fires silently (no Metal-level log)

**File**: `tt_metal/impl/device/firmware/fabric_firmware_initializer.cpp`, lines ~877-884

The `clear_relay_broken(ch.dev->id())` call (FIX XY-2) is wrapped only in a try/catch with
a comment. No `log_info` or `log_warning` is emitted when the relay_broken flag is cleared.

```cpp
// FIX XY-2 (#42429): ...
try {
    cluster_.clear_relay_broken(ch.dev->id());
} catch (...) {
    // Best-effort: if clear fails (e.g. chip not remote), harmless.
}
```

UMD's `Cluster::clear_relay_broken` does log `"FIX XY-2: Clearing relay_broken for chip N"`, but
`tt::Cluster::clear_relay_broken()` (tt_cluster.hpp:94-96) is silent.

**Impact**: When relay is restored after force-reset, there is no Metal-level trace of the
recovery event. If AllGather still hangs AFTER clear_relay_broken(), the log gives no indication
that the restoration was attempted or succeeded.

**Fix**: Add `log_info(tt::LogMetal, "FIX XY-2: relay_broken cleared for Device {} — relay restored after force-reset", ch.dev->id());`
immediately before the `clear_relay_broken` call (Metal source), and add a corresponding log
in tt_cluster.hpp `clear_relay_broken()`.

## GAP B: analyze_fabric_hang_log.sh stale patterns for FIX AX / FIX AU

**File**: `scripts/analyze_fabric_hang_log.sh`, lines 824-1287

The script counters for `FIX_AX_FIRES` and `FIX_AU_FIRES` match OLD behavior:
```bash
FIX_AU_FIRES=$(grep -cE 'FIX AU|bypassed the poll loop.*relay-broken' ...)
FIX_AX_FIRES=$(grep -cE 'FIX AX|relay confirmed dead.*skipping assert_risc_reset' ...)
```

After FIX AX-2 and FIX AU-2, the log messages changed:
- FIX AU-2 emits: `"FIX AU-2 (#42429): Device {} (non-MMIO) relay path broken — ..."` (attempt, not skip)
- FIX AX-2 emits: `"FIX AX-2 (#42429): Device {} chan={} relay confirmed dead — attempting assert_risc_reset_at_core anyway"`

The old `skipping assert_risc_reset` pattern no longer appears. The script's `FIX_AX_FIRES`
counter will always be 0 for recent CI runs. Similarly, FIX SB2-R has no counter in the
AGGREGATE_COUNTERS section, nor does FIX XY-2 relay-restored events.

**Fix**:
1. Update `FIX_AU_FIRES` to match `FIX AU-2` log pattern
2. Update `FIX_AX_FIRES` to match `FIX AX-2` log pattern and update annotation (now "attempted" not "skipped")
3. Add `FIX_XY2_RELAY_RESTORED` counter matching `"FIX XY-2.*relay_broken cleared"` or `"FIX XY-2: Clearing relay_broken"`
4. Add `FIX_SB2R_CLEAN_BOOT` counter matching `"relay_broken NOT set \(FIX SB2-R"` pattern

## GAP C: No unit test for AllGather after relay_broken set-then-cleared

**Directory**: `tests/nightly/t3000/ccl/`

There is `test_cross_device_relay_broken_race.py` (GAP_12) which tests Phase 5 concurrent quiesce
with relay_broken. There are GAP_36, GAP_38 tests for relay_dead per-device skip and AllGather
correctness after cleanup. However, there is NO test that:

1. Marks relay_broken=true for one or more non-MMIO devices
2. Runs a teardown (force-reset path that calls clear_relay_broken via FIX XY-2)
3. Verifies that relay_broken is cleared (`is_relay_broken()` returns false)
4. Runs AllGather and verifies it completes successfully (not hanged)

This is the *exact* recovery path that FIX XY-2 enables. Without a test, a regression that
breaks `clear_relay_broken()` or skips the FIX XY-2 path would not be caught until a CI hang.

**Fix**: Create `tests/nightly/t3000/ccl/test_gap79_fixxy2_relay_broken_cleared_allgather.py`
(GAP_79) that:
- Opens a T3K mesh
- Injects relay_broken via conftest fixture (calls cluster.mark_relay_broken on non-MMIO devices)
- Runs teardown → verifies `is_relay_broken()` cleared after teardown
- Re-opens mesh and runs AllGather → verifies correctness

## GAP D: FIX SB2-R "probe was clean" log not captured in TIMELINE filter

**File**: `scripts/analyze_fabric_hang_log.sh`, line 46 (timeline grep pattern)

The FIX SB2-R log: `"{} FIX M channel(s) but probe was clean — relay_broken NOT set (FIX SB2-R"`
does NOT appear in the timeline grep pattern (line 46). This means a clean-boot SB2-R skip is
invisible in the TIMELINE section, making it hard to distinguish "relay_broken was NOT set (clean)"
from "relay_broken WAS set" in a CI log parse.

**Fix**: Add `FIX SB2-R` to the timeline and phase-summary grep patterns.
