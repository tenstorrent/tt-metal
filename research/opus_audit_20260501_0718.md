### `8d5e9820b73` — FIX TH: relay_mux preflight check

**File**: `tt_metal/impl/dispatch/kernel_config/relay_mux.cpp`

**Root cause**: On a progressively-degraded T3K cluster (each SIGKILL teardown leaves more
ERISC L1 channels corrupt), the second+ session finds ALL ETH channels between two in-cluster
MMIO devices dead. Existing guards (FIX NY/NY+) only check whether chips are excluded from the
fabric cluster — not whether ETH links between them are exhausted.

Without FIX TH:
```
get_dispatch_link_index()
  → get_forwarding_link_indices() returns {}
  → TT_FATAL: "No links available from (M0,D2) to (M0,D3)"
```
This TT_FATAL fires BEFORE `CustomMeshGraphFabric2DFixture::SetUp()` can reach the FIX SA
degraded-cluster skip guard.

**Fix (preflight block in `GenerateStaticConfigs()`)**:
```cpp
// FIX TH (#42429)
const auto& available_links =
    tt_fabric::get_forwarding_link_indices(src_fabric_node_id, dst_fabric_node_id);
if (available_links.empty()) {
    log_warning(tt::LogMetal, "FIX TH (#42429): ... Marking device {} "
        "fabric_channels_not_ready_for_traffic ...", device_id_);
    device_->set_fabric_channels_not_ready_for_traffic();
    return;
}
```

**Impact**: Converts TT_FATAL crash during device init into graceful skip via FIX SA.

**Coverage**: **GAP-69 added** (see below). ✓

#### GAP-69 Regression Test (FIX TH)

**Mechanism** (matches GAP-67/68 pattern):
1. **PREDECESSOR**: fork, open FABRIC_2D on all T3K devices, dispatch blank workload, signal ready → SIGKILL
2. **TESTEE**: fork, install SIGABRT handler (exits 69), try `MeshDevice::create()` with T3K custom mesh descriptor
3. **PARENT**: wait ≤90s for testee. If testee exits 69 → `FAIL()` (regression: FIX TH absent/reverted)

**Key sentinel**:
```cpp
static const int kGap69RegressionExit = 69;
static void gap69_sigabrt_handler(int) {
    const char msg[] = "GAP-69 TESTEE SIGABRT: TT_FATAL triggered ...";
    (void)::write(STDERR_FILENO, msg, sizeof(msg) - 1);
    ::_exit(kGap69RegressionExit);
}
```

Without FIX TH: relay_mux TT_FATAL → SIGABRT → sighandler → `_exit(69)` → FAIL
With FIX TH: preflight returns early → exit 0 → PASS

**Fixture**: `FixThRelayMuxNoLinksGuardFixture` — requires ≥8 devices (T3K).

**Assessment**: Well-structured. Same pattern as GAP-67/68. Covers the exact failure mode from
CI run 25204526485.

#### FIX TI: `is_control_plane_initialized()` guard

**Root cause**: `set_default_fabric_topology()` resets `control_plane_` to null (in
`CustomMeshGraphFabric2DFixture::TearDown`). If `risc_firmware_initializer.cpp::teardown()` then
calls `get_control_plane_()` (bound to `MetalEnvImpl::get_control_plane()`), that function
lazy-initializes control_plane_ → runs topology discovery on degraded hardware → throws
`unordered_map::at`.

**Fix**: Add `is_control_plane_initialized()` (non-initializing nullptr check):
```cpp
bool is_control_plane_initialized() const { return control_plane_ != nullptr; }
```
Then guard all three `get_control_plane_` checks in teardown:
```cpp
const bool cp_ready = get_control_plane_ && descriptor_->env_impl().is_control_plane_initialized();
// Use cp_ready instead of get_control_plane_ in all three guards
```

**Coverage**: No dedicated regression test (GAP-70). The crash occurs during destructor
teardown AFTER GTest has finished observation — making a fork-based test difficult.
This is a defensive measure for a known-bad-state case, not an observable test-level failure.
**Verdict: defer GAP-70**. The fix is correct and audited. Risk of regression: low.

### `2e8cb93304` — FIX TJ: topology_mapper WH_B0 shape prefilter

**File**: `tt_metal/fabric/topology_mapper.cpp`

**Root cause (CI run 25205399359)**:
After prior test suites degrade the T3K cluster to 4 accessible chips,
`generate_possible_cluster_shapes(4)` produces: `[2x2, 4x1, 3x1, 2x1, 1x1]`.
Both 2x2 and 4x1 fail to map on degraded hardware. The mapper then tries 3x1:

```
MeshGraph::generate_mesh_graph_of_shape(3x1, ...)
  → line 942: TT_FATAL(!both_odd || (shape[0]==1 && shape[1]==1), "got 3x1")
```

This TT_FATAL fires inside `SetUpTestSuite()` for three fixture classes:
`Fabric2DFixture`, `Fabric2DUDMModeFixture`, `Fabric2DMuxFixture` — all three fail.

**Fix (prefilter in topology_mapper shape loop)**:
```cpp
// FIX TJ (#42429)
if (cluster.arch() == tt::ARCH::WORMHOLE_B0) {
    bool both_odd = (mesh_shape[0] % 2 != 0) && (mesh_shape[1] % 2 != 0);
    if (both_odd && !(mesh_shape[0] == 1 && mesh_shape[1] == 1)) {
        log_warning(tt::LogFabric, "FIX TJ (#42429): TopologyMapper: skipping shape {}x{} "
            "— invalid for WORMHOLE_B0 ...", mesh_shape[0], mesh_shape[1]);
        continue;
    }
}
```

**Impact**: Converts TT_FATAL crash in `SetUpTestSuite()` into a graceful fallthrough to 2x1 or 1x1.

**Coverage**: No dedicated regression test (GAP-71). A fork test could be written:
- Predecessor: degrade to 4-chip cluster
- Testee: install SIGABRT handler, try `MeshDevice::create()` with T3K descriptor
- Parent: exit 71 = regression

**However**: GAP-69 already exercises a similar degradation sequence. If FIX TJ regresses
and FIX TH is still present, we'd see `Fabric2DFixture` SetUpTestSuite fail with TT_FATAL in
CI (very visible — kills 3 fixture suites). **Verdict: defer GAP-71** for now. If FIX TJ is
reverted accidentally, CI will catch it loudly.

**Script coverage**: `FIX_TJ_FIRES` counter + `[FIX TJ]`/`[FIX TJ MISSING]` output blocks added.
`invalid for WORMHOLE_B0` tracked in main filter (line 46).
