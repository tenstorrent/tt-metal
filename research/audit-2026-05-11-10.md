### 4. No test for FIX P25-CLEAN
**Severity**: High
**Impact**: Phase 3 quiesce launch on already-terminated channels causes cascade hang
**Details**: Created GAP-78 test. Fork-based: PREDECESSOR opens full mesh + clean close, TESTEE opens sub-mesh. Channels to out-of-mesh devices should be skipped. Requires >= 4 devices.

### 5. No test for FIX AP
**Severity**: High

**Details**: Created GAP-79 test. Fork-based: PREDECESSOR gets SIGKILL'd, TESTEE checks relay_path_broken flag on non-MMIO devices. Requires >= 2 devices.

### 6. No test for FIX EV
**Severity**: High
**Impact**: EventSynchronize infinite spin on dead relay → 8-minute CI hang
**Details**: Created GAP-80 test. Fork-based: PREDECESSOR gets SIGKILL'd, TESTEE attempts dispatch + EventSynchronize. Verifies no infinite hang. Requires >= 2 devices.

## Remaining Gaps (Out of Scope)

1. **No unit test for FIX RR-NM (non-MMIO relay-assisted reset)**: Requires specific hardware topology (non-MMIO ERISC channels in ROM-boot state) that cannot be simulated without hardware mock framework. Recommend adding when mock infrastructure is available.

2. **No test for FIX BT (terminate_stale_erisc_routers ROM postcode detection)**: The ROM postcode polling logic in terminate_stale_erisc_routers is deeply integrated with hardware state. Fork tests can exercise the path but cannot guarantee ROM boot state on demand.

3. **No test for FIX FQ-1/2/5 (quiesce firmware launch timeout paths)**: These are timeout recovery paths that require specific timing-dependent hardware conditions.

4. **EDM firmware audit items (MEDIUM)**: WFEWS (unbounded spin in mux forward_data) and MUX-NOWH (legacy mux no-termination-check) identified in v3 audit but not fixed — require performance benchmarking before changing hot-path code.

5. **analyze_fabric_hang_log.sh FIX label drift**: As new FIX labels are added to code, the script needs manual updates. Consider auto-generating the pattern from `grep -oP 'FIX [A-Z0-9_-]+' *.cpp`.
