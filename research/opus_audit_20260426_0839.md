## 2. Logical Correctness

### FIX AK + AM (partial-mesh quiesce)

The logic chain in Phase 5b is:

```
truly_unhealthy not empty?
  → all at 0x0/DEAD5B5B/DEADECE7? (all_dead)
    → YES: FIX W path — return true, set relay_broken if non-MMIO
    → NO: all at {dead values} ∪ {STARTED, REMOTE_HANDSHAKE, LOCAL_HANDSHAKE}? (all_handshake_incomplete)
      → YES: FIX AK path — return true, set channels_not_ready (FIX AM)
      → NO: TT_THROW (unexpected state)
```

This is **logically correct**. The `!all_dead` guard on `all_handshake_incomplete` prevents double-entry. The mixed case (some dead + some handshake-incomplete) correctly falls into FIX AK, not FIX W — this is the right call because the relay IS functional if at least one channel reached STARTED.

**One edge case to note**: If exactly one channel is at 0x0 (dead) and another is at STARTED, FIX AK fires and sets `fabric_channels_not_ready_for_traffic_` but does NOT set `fabric_relay_path_broken_`. The test then skips AllGather. This is correct behavior for partial-mesh, but the logging says "peer device(s) not in quiesce set" which is misleading for the 0x0 channel — that channel might genuinely be dead, not just handshake-incomplete. Low priority but worth noting.

### FIX AL (dispatch hang regression)

The fix reads PREFETCH_Q_RD from hardware at `reset()` time instead of hardcoding the sentinel. This is sound:
- Case A (kernel reloaded): hw contains sentinel → same as before.
- Case B (kernel NOT reloaded): hw contains actual fence → correct delta arithmetic.

**Risk**: The read occurs via `cluster.read_core()` which goes through NOC. If the prefetcher core is mid-reset or hung, this read could itself hang. The code does not have a timeout wrapper. This is a latent hang path, but it's the same risk as every other NOC register read in the codebase, so it's not a regression.

### FIX AA test skip (test file)

The skip logic is clean:
```cpp
for (auto* idev : mesh_device_->get_devices()) {
    if (!idev->is_mmio_capable() &&
        (idev->is_fabric_relay_path_broken() ||
         idev->is_fabric_channels_not_ready_for_traffic())) {
        GTEST_SKIP() << ...;
    }
}
```

This correctly checks both flags and only on non-MMIO devices. Appears in 3 test methods — consistent application.

**Concern**: GTEST_SKIP is a band-aid. If the partial-mesh condition fires on every CI run for a given topology, the test never actually exercises AllGather. There is no counter or CI visibility for "how often does this skip fire?" This should be tracked.

## 3. Risk Surface

### What could still go wrong

1. **TOCTOU on `fabric_channels_not_ready_for_traffic_`**: The flag is set in `wait_for_fabric_workers_ready()` and read by the test between quiesce and AllGather. If another thread calls `configure_fabric()` between those two points, the flag gets cleared and AllGather runs on a not-yet-ready fabric. The comment at device_impl.hpp:320-323 notes that MeshDevice serializes these calls, but the invariant is documented, not enforced.

2. **Partial-mesh + MMIO device**: FIX AK only fires for non-MMIO devices (the test skip checks `!is_mmio_capable()`). If an MMIO device's ETH channels are stuck at handshake-incomplete (e.g., peer chip outside mesh), the code would hit the TT_THROW at the bottom of Phase 5b. This scenario hasn't been observed but isn't explicitly guarded.

3. **FIX AL register read during device reset**: As noted above, `cluster.read_core()` in `reset()` has no timeout. If the prefetcher core is in a bad state (e.g., post-quiesce with ETH relay broken), the NOC read could hang.

4. **Flag accumulation across quiesce cycles**: `fabric_channels_not_ready_for_traffic_` is only cleared in `configure_fabric()`. If a quiesce cycle completes with the flag set and the next operation is another quiesce (not configure_fabric), the flag remains stale. The current call sequence is quiesce → configure → quiesce, so this shouldn't happen in practice, but the invariant is implicit.

5. **Dead channel set confusion**: `fabric_pre_dead_channels_` is cleared then re-set in `configure_fabric()` (lines 450-455). If `configure_fabric()` is called with an empty `pre_dead_channels`, previously dead channels are silently forgotten. This is intentional (fresh firmware = fresh state) but means a transiently-dead channel that recovered during configure_fabric won't be tracked.

## 4. Complexity Debt

**This is the biggest concern.**

```
Files changed vs main:    86 files, +13702 -384
Key files alone:          +4270 lines
device.cpp:               3316 lines (was ~987 before this branch)
Unique FIX tags:          34 (A through AM, with gaps)
fabric_firmware_init.cpp: 1836 lines
```

The fix is a **layer cake** — each iteration adds a new early-return path, a new flag, or a new status check to an increasingly deep conditional tree. The Phase 5b health check alone has 4 tiers of conditional logic (pre-dead filtering → all_dead check → all_handshake_incomplete check → TT_THROW).

**Is a refactor cleaner?** Yes, but not now. The fixes are individually well-tested against specific CI failure modes. Refactoring mid-investigation risks losing the mapping between "FIX X prevents failure Y." The right time to refactor is after the branch merges and the fix is stable for a sprint.

**Recommended refactor targets** (post-merge):
1. Extract the entire quiesce flow from `device.cpp` into a dedicated `fabric_quiesce.cpp` with a clear phase-based structure.
2. Replace the 34 FIX tags with a structured enum or state machine for quiesce phases.
3. Consolidate the 5 `fabric_relay_path_broken_` SET sites into a single `mark_relay_broken()` method that handles logging and flag-setting.
4. Move the Phase 5b health-check branching into a dedicated `classify_unhealthy_channels()` function that returns an enum (DEAD, HANDSHAKE_INCOMPLETE, UNEXPECTED).

## 5. Next Steps / Most Likely Next Failure Mode

1. **MMIO device with partial-mesh ETH stuck at handshake**: FIX AK only handles non-MMIO via the test skip. If T3K topology puts an MMIO device at a mesh edge with out-of-mesh peers, Phase 5b will TT_THROW. Fix: extend FIX AK guard to MMIO devices, or prove it can't happen topologically.

2. **FIX AL register read hang on broken device**: If a device's dispatch core is non-responsive after quiesce, the `read_core()` in `reset()` will hang. Fix: add a timeout wrapper or skip the read when `fabric_relay_path_broken_` is set on the device.

3. **GTEST_SKIP masking real regressions**: If the partial-mesh skip fires too frequently, real AllGather bugs will be hidden. Fix: add a CI metric for skip frequency per test and alert if it exceeds a threshold.

4. **Stale `fabric_channels_not_ready_for_traffic_` across test boundaries**: If test fixture TearDown triggers quiesce but SetUp for the next test does not call configure_fabric(), the flag persists. Fix: verify that the test fixture lifecycle always pairs quiesce with configure_fabric.
