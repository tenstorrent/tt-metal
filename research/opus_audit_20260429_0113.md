## New Fixes Found (Since Iteration 24)

### FIX NV (`03e4d397bea`) — Skip `get_device_aiclk` for non-MMIO chips

**Location**: `tt_metal/impl/device/firmware/risc_firmware_initializer.cpp`,
`RiscFirmwareInitializer::run_async_build_phase()`, line ~141

**Root cause**: `get_device_aiclk(device_id)` for a remote (non-MMIO) chip calls:
```
RemoteChip::get_clock()
  → WormholeArcMessenger::send_message()
    → wait_for_non_mmio_flush()  [5s timeout when relay dead]
      → throws "Timeout waiting for Ethernet core service remote IO request."
```
This is an async task. The throw propagates: `fut.get()` → `run_async_build_phase()` →
`MetalContext::initialize()` → test `SetUp()` → **test FAILS with exception**.

With N non-MMIO chips (T3K: 3), hang is N×5s = 15s before the throw.

**Fix**: gate call on `mmio_ids_set.count(device_id)`. The aiclk value is `[[maybe_unused]]`
debug-only; zero functional impact on skipping.

**No prior test existed.**

### FIX NW (`a2fd50af057`) — Skip `clear_launch_messages_on_eth_cores` for non-MMIO chips

**Location**: same file, `run_async_build_phase()`, line ~176

**Root cause**: `clear_launch_messages_on_eth_cores(device_id)` calls in sequence:
1. `cluster_.write_core()` — throws on dead relay, but **FIX AE catches it** in `tt_cluster.cpp`,
   marks relay broken, and returns normally. ✓
2. `cluster_.l1_barrier()` → `driver_->l1_membar()` → `wait_for_non_mmio_flush()` — throws on dead
   relay. **FIX AE does NOT wrap `l1_barrier()`** — this is the unclosed hole. ✗

The second throw propagates identically to FIX NV → **test FAILS**.

**Key insight**: FIX AE (commit `561f7abd505`) wraps `write_core()`, `write_reg()`, and
`noc_multicast_write()` in tt_cluster.cpp, but NOT `l1_barrier()`. FIX NW closes this by
skipping the call entirely for non-MMIO chips.

**Safe to skip**: `run_launch_phase()` calls `terminate_active_ethernet_cores_on_all_chips()`
unconditionally (MMIO + non-MMIO), performing full ETH core reset which clears any stale
launch messages.

**CI evidence**: run 25083588469 (t3k-06), TestMeshWidthShardedCopy2D_Large FAILED.
Log: "FIX AE (flush guard) fired at 00:04:24 but the write_core/l1_barrier calls still threw"

**No prior test existed.**

## Gaps NOT Addressed (Justified)

1. **`l1_barrier()` FIX AE hole in general**: FIX AE doesn't wrap `l1_barrier()` in
   tt_cluster.cpp. This affects callers other than `clear_launch_messages_on_eth_cores`.
   However, fixing the general hole would be a larger change requiring broader analysis.
   FIX NW's targeted skip is the safe minimal fix. No new test needed beyond GAP-43.

2. **`run_launch_phase` relay guards**: `run_launch_phase` also calls `reset_cores()` and
   `initialize_and_launch_firmware()` for each chip including non-MMIO. These have their
   own relay guards (FIX AS canary path). Not a new gap.

3. **Other async build tasks** (`generate_device_bank_to_noc_tables`,
   `generate_worker_logical_to_virtual_map`, `build_firmware`): these are relay-independent
   (use PCIe-direct structures). Not a new gap.
