### d4df01b3853 — Timing window fix (GAP-23)

**Type**: Test-only behavior fix.

FIX W topology_discovery probes during `set_fabric_config(FABRIC_2D)` can take
30-60s per dead gateway ETH core on a degraded cluster. The timing guard was meant
to detect FIX AE (STARTED-deadlock) and FIX AK (stuck-channel) regressions — both
of which occur during `synchronize_device + set_fabric_config(DISABLED)` (teardown),
not during FABRIC_2D re-init.

Fix: moved `set_fabric_config(FABRIC_2D)` outside the timed block. Only teardown
is now timed; re-init is excluded. A real FIX AE/AK regression still exceeds 30s
because the teardown hangs.

**Decision: No new GAP test needed.** The existing GAP-23 test IS the regression
test; this commit just tightens its precision. Writing a test for "timing window
excludes FABRIC_2D re-init" would be circular.

### 74cb2aa7591 — is_fabric_degraded() guard before AllGather in GAP-23 Python test

**Type**: Python test guard.

Adds a skip guard in `test_gap23_partial_mesh_quiesce_cycling.py` using
`mesh_device.is_fabric_degraded()`. When FIX RZ sets
`fabric_stale_base_umd_channels_=true` (base-UMD channels from prior SIGKILL),
`is_fabric_degraded()` returns true and AllGather is skipped to avoid hang.

**Decision: No new GAP test needed.** The underlying predicate
(`fabric_stale_base_umd_channels_`) is already tested by GAP-66
(`test_gap66_fixrz_stale_base_umd_flag.cpp`). The Python guard is a downstream
user of that flag. Coverage is sufficient.

### 10297772581 — FIX QW-B: missing stale_base_umd_channels check in C++ fixtures

**Type**: HIGH PRIORITY — state management gap in production fixture code.

`Fabric2DFixture::SetUp()`, `CustomMeshGraphFabric2DFixture`, and
`MultiCQFabricMeshDevice2x4Fixture::SetUp()` skip guards previously checked:
```cpp
relay_broken || channels_not_ready
```
This misses the FIX M transition path: after a SIGKILL, base-UMD firmware is
retained on non-MMIO ERISCs, configure_fabric() uses skip_soft_reset (FIX M), and
FIX RZ sets `fabric_stale_base_umd_channels_=true`. But `relay_broken=false` and
`channels_not_ready=false` in this path → old guard returns false → AllGather
proceeds → dispatch core timeout + device hang (~100s).

FIX QW-B adds `|| dev->is_fabric_stale_base_umd_channels()` to all three fixtures.

**Decision: NEW GAP TEST IMPLEMENTED — GAP-73.**
