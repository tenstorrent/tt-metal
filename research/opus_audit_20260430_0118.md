## Commit Triage

### Priority 1 — fw_launch_addr Cascade (GAP-48, GAP-49, GAP-50)

These three fixes address the **dominant CI regression**: stale `fw_launch_addr` in ERISC L1 after hardware reset causes `erisc_app_still_running()` false positives, triggering 500ms `wait_until_cores_done` timeouts per core. On T3K (8 devices, ~6 ETH cores each), this cascades to 12-24s of wasted time per test open. Observed 140+ times in CI runs 25096771728 and 25110543943.

| Commit | FIX | Location | GAP Test |
|--------|-----|----------|----------|
| `1111316e84f` | FIX PA | `risc_firmware_initializer.cpp reset_cores()` — init force-reset path | **GAP-48** |
| `136dd144275` | FIX PB | `device.cpp rescue_stuck_dispatch_cores()` — rescue path | **GAP-49** |
| `3e2614543f3` | FIX PD | `device.cpp quiesce_and_restart_fabric_workers()` Pass-0 + `launch_eth_cores_for_quiesce()` Pass-0 | **GAP-50** |

All three write `0` to `fw_launch_addr` via `write_core_immediate` after `deassert_risc_reset_at_core`, wrapped in try/catch for non-MMIO dead-relay channels.

**Test pattern**: Fork PREDECESSOR (opens FABRIC_2D, dispatches blank workload, SIGKILL'd to leave stale state) -> Fork TESTEE(s) with timing budgets. Budget calibrated so test passes with fix (~25s) and fails without (~37s+).

### Priority 2 — UMD Heartbeat Skip (GAP-51)

| Commit | FIX | Location | GAP Test |
|--------|-----|----------|----------|
| `f0e3d677935` | FIX PF | `risc_firmware_initializer.cpp reset_cores()` — heartbeat detection | **GAP-51** |

**Optimization**: After `erisc_app_still_running()==true` on MMIO devices, reads UMD base firmware heartbeat register. If `(hb >> 16) == 0xABCD`, clears `fw_launch_addr` directly and skips the Metal exit signal + 500ms wait. This is the fast path; FIX PA is the safety net.

**Test pattern**: Same predecessor SIGKILL pattern. Tight budget (30s) catches the 12s overhead from 500ms x 24 cores that occurs without the fix.

### Priority 3 — Relay Retry + YAML Graceful Skip (GAP-52, GAP-53)

| Commit | FIX | Location | GAP Test |
|--------|-----|----------|----------|
| `07bda1e2d2f` | FIX PG | `device.cpp quiesce_and_restart_fabric_workers()` Phase 2.5 | **GAP-52** |
| `9ef7acee300` | FIX PH | `tt_cluster.cpp` + `mesh_socket_test_context.cpp` + `tt_fabric_test_common.hpp` | **GAP-53** |

**FIX PG**: Retries Phase 2.5 L1 reads for non-MMIO devices (max 2 retries, 3s sleep between) to handle transient relay congestion after AllGather traffic. MMIO devices get zero retries (no relay involved).

**FIX PH**: Adds `try_get_physical_chip_id_from_eth_coord()` returning `std::optional` instead of `TT_FATAL` when YAML hardcoded EthCoords don't match the runner's actual chip layout. Tests use exit(0) skip instead of SIGABRT crash.

**GAP-53 test pattern**: Direct API unit tests (no fork needed) — verifies nullopt for bogus coords and valid result for real coords.

## Test Design Notes

1. **No hardcoded device numbers** — all tests use `MetalContext::instance().get_cluster().number_of_devices()` and reason at the level of MMIO vs non-MMIO characteristics.

2. **Self-documenting** — each test file has a 40+ line header comment explaining root cause, fix location, test strategy, and timing budget rationale.

3. **Shared memory pattern** — `MAP_SHARED | MAP_ANONYMOUS` mmap with `std::atomic<int>` for predecessor->parent signaling. Properly munmapped in all exit paths (including timeout and skip).

4. **Budget calibration** — budgets are set so tests pass with the fix (measured ~25s) and fail without (measured ~37s+ from 500ms cascade). GAP-52/54 have generous budgets (90s) because they test crash/hang prevention rather than timing.

5. **Skip guards** — all fork-based tests skip if `number_of_devices() < 2` (need MMIO + non-MMIO for relay paths). GAP-53 skips if `< 1` device.
