### 3ee311c83f7 — GAP-38 Testee-2 is_fabric_degraded() skip guard (FIX BB)

**Type**: Python test fix — subprocess degraded-cluster bail-out.

Root cause: GAP-38 runs two subprocess workers (Testee-1, Testee-2). Prior test session leaves
non-MMIO devices 4-7 in base-UMD relay firmware (FIX RX skips quiesce teardown for broken-fabric
mesh). Testee-2 opens MeshDevice → configure_fabric() takes FIX M path → FIX RZ sets
`fabric_stale_base_umd_channels_=true` → `is_fabric_degraded()` returns true (FIX QW fires).

Without FIX BB: Testee-2 had no degraded guard → AllGather hung 60s → timeout → parent:
`pytest.fail("GAP-38 HANG")`. This was a **false test failure** — the test was not testing
what it's meant to test (correctness after cleanup), but was instead failing due to degraded
pre-condition from prior run.

FIX BB changes:
- `_testee2_script()`: after `set_fabric_config()`, calls `mesh.is_fabric_degraded()`.
  If degraded, writes `{"error": "SKIP:fabric_degraded..."}` to result JSON, closes mesh,
  `sys.exit(2)`.
- Parent test: detects `result.get("error", "").startswith("SKIP:")` → `pytest.skip()`.

Confirmed firing in CI run on t3k-03: `FIX QW (#42429): cluster degraded (device 4...)` fires
4 times in `ci_analysis_latest.txt` (devices 4-7, one call per test run with degraded state).

**Decision: No new GAP test needed.**

The underlying `is_fabric_degraded()` predicate is already covered by GAP-66 and GAP-73.
FIX BB is a test-harness fix, not a production code fix. The scenario being guarded
(Testee-2 subprocess encountering degraded cluster) is only testable in a fully-degraded
hardware environment — a unit test for this would require a real degraded cluster.
