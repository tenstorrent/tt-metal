## New Fixes Since Last Audit (Iterations 21-22)

### FIX NS — Eliminate redundant topology discovery (commit f80f0295263)

**CI failure**: run 25074797042 — 14m40s SIGALRM before any test ran in `unit_tests_ttnn`

**Root cause**:
`MetalEnvImpl::initialize_base_objects()` called `get_cluster_type_from_cluster_desc()`
BEFORE creating the Cluster object. This triggered **TopologyDiscovery #1** — depositing
one stuck relay CMD per MMIO gateway (4 gateways × 1 cmd = 4/4 capacity approaching).
Then `Cluster::Cluster()` ran **TopologyDiscovery #2** which pushed relay queues to 4/4
on FABRIC-state ERISC systems → `while(full)` no-timeout spin → infinite hang.

**Fix**: Create Cluster first in `initialize_base_objects()`, derive cluster type from
`cluster_->get_cluster_type()`. Eliminates TopologyDiscovery #1 entirely.

### FIX AE — Catch flush timeouts + mark relay broken (commit 561f7abd505)

**Problem 1**: `write_core`, `write_core_immediate`, `write_reg`, `noc_multicast_write`
called `wait_for_non_mmio_flush()` without try/catch. On dead relay: 5s timeout per call,
then exception. If the caller caught and retried → cascade of 5s hangs.

**Problem 2**: `~Cluster()` with old FIX AW called `close_device()` in a background thread
with 5s timeout. This created a race: new `Cluster()` could start construction while old
`~Cluster()` was still holding UMD global state in `close_device()` → heap corruption →
SIGSEGV/SIGABRT in the new process.

**Fix**:
1. Wrap each `wait_for_non_mmio_flush()` call in try/catch. On timeout: log FIX AE warning
   + call `mark_relay_broken()` → all future flushes for that chip instant (0ms).
2. In `~Cluster()`: mark ALL remote chips relay-broken before `close_device()`. This makes
   `close_device()` return instantly, closing the race window. Supersedes FIX AW.

## Gaps Identified and Implemented

### GAP-39: FIX NS regression test

**File**: `tests/nightly/t3000/ccl/test_gap39_fixns_single_topology_discovery.py`

**What it catches**: A FIX NS regression where `get_cluster_type_from_cluster_desc()` is
re-added before Cluster creation, causing double topology discovery → relay queue overflow
→ 14+ min hang on systems with FABRIC-state ERISCs.

**Test pattern**:
1. Predecessor: SIGKILL after FABRIC_2D open (non-MMIO ERISCs in FABRIC state)
2. Testee: open FABRIC_2D + AllGather (or degraded) + close. Must complete < 90s.
   - With FIX NS: single discovery + FIX AQ 5s catches → ~25s
   - Without FIX NS: double discovery → relay queue overflow → timeout assertion

**Budget**: 90s (4 chips × 5s FIX AQ + init overhead + AllGather)

### GAP-40: FIX AE regression test

**File**: `tests/nightly/t3000/ccl/test_gap40_fixae_flush_timeout_catch.py`

**What it catches**:
- FIX AE mark-before-close regression (Testee-1 budget: 60s)
- UMD global state heap corruption from racing ~Cluster()/Cluster() (Testee-2 crash check)

**Test pattern**:
1. Predecessor: SIGKILL after FABRIC_2D open
2. Testee-1: open + quiesce (relay-dead path) + close. Must complete < 60s.
   - With FIX AE: mark_relay_broken before close_device → instant → ~20s
   - Without FIX AE: close_device blocks 5s × 4 chips = 20s+ → ~40s → times out
3. Testee-2: open immediately after Testee-1 exits. Must not crash (SIGSEGV/SIGABRT).
   - With FIX AE: ~Cluster() returned quickly → no race window → Testee-2 safe
   - Without FIX AE: ~Cluster() held UMD global state for 20s → Testee-2 may crash

**Crash detection**: checks exit code -11 (SIGSEGV), -6 (SIGABRT), "Segmentation fault" in stderr
