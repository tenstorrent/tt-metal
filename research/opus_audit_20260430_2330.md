## Remaining Gaps

### GAP-A: SIGSEGV in close_mesh_device() while AllGather in-flight (CRITICAL)

**Status**: Open, unfixed
**Test**: `test_gap22_allgather_inflight_close.py` — hits SIGSEGV on iteration 0
**Root cause**: `close_mesh_device()` during in-flight AllGather triggers null dereference or use-after-free. Frames 0-9 in `ttnn.so` are truncated in CI logs.
**Cascade**: SIGSEGV leaves ETH gateways dead → iteration 1/2 fail with `unordered_map::at` during topology discovery.
**Next step**: Debug build with symbols to get full crash stack. Look at `terminate_active_ethernet_cores_on_all_chips` / `FabricFirmwareInitializer::teardown` code paths.

### GAP-B: FIX NY/NY+ lacks regression test

**Status**: No GAP test
**Risk**: Without the guard, `RelayMux::GenerateStaticConfigs` accesses chips excluded from the fabric cluster → out-of-bounds / crash in dispatch kernel config.
**Recommendation**: GAP-63 — fork-based test that damages ETH to create a degraded topology, then opens device and verifies dispatch kernel config generation doesn't crash.

### GAP-C: FIX TB (topology_mapper graceful unknown ASIC) lacks regression test

**Status**: No GAP test
**Risk**: Without FIX TB, an ASIC ID not in `chip_topology_mapping_` during UMD topology re-discovery triggers `TT_FATAL` → SIGABRT. This happens after SIGSEGV recovery.
**Recommendation**: GAP-64 — unit test that directly calls topology_mapper with a bogus ASIC ID and verifies graceful handling (skip + warning) instead of fatal.

### GAP-D: FIX QV (Phase 4 MUX poll skip) lacks regression test

**Status**: No GAP test, but tested in CI via UDM test suite
**Risk**: Without FIX QV, Phase 4 times out on MMIO devices with dead master router → TT_THROW in TearDown → 5 UDM test failures.
**Recommendation**: GAP-65 — fork-based test that creates degraded fabric with dead MMIO master router, then verifies `wait_for_fabric_workers_ready()` returns without timeout.

### GAP-E: FIX SA/TC interaction with GTEST_SKIP propagation

**Status**: FIX TC moves `CustomMeshGraphFabric2DFixture::SetUp(desc, mapping)` into the GTest `SetUp()` hook so `GTEST_SKIP()` propagates correctly. The `custom_setup_initialized_` flag prevents double-init.
**Risk**: If `custom_setup_initialized_` is not reset between tests (it IS reset in `TearDown`), tests after a skip could have stale state. Currently mitigated by the `TearDown` override.
**Recommendation**: Low priority — the code is correct. No GAP test needed unless TC regresses.

### GAP-F: ETH channels 14/15 probe bounds check

**Status**: Open, no fix
**Symptom**: `No core type found for system TRANSLATED at location: (0, 14)` in fabric_eth_health probe on devices 1 and 3 channels 14/15.
**Risk**: Low — the error is caught and logged, doesn't crash. But it pollutes logs.
**Recommendation**: Bounds check in `coordinate_manager.cpp` or the health probe to skip channels beyond the valid TRANSLATED range.

### GAP-G: `unordered_map::at` cascade in topology_discovery.cpp

**Status**: Open, triggered by GAP-22 SIGSEGV cascade
**Symptom**: After SIGSEGV kills a subprocess, the next subprocess's `open_mesh_device` crashes with `std::out_of_range` from `topology_discovery.cpp:390` or `:524` — `.at()` on `remote_asic_id_to_mmio_device_id` or `asic_id_to_chip_id` when ETH gateways are dead.
**Risk**: Medium — only triggers after SIGSEGV, but makes GAP-22 iterations 1/2 non-diagnostic.
**Recommendation**: Replace `.at()` with `.find()` + graceful skip in `topology_discovery.cpp` for the 6 call sites identified. This is in `tt_metal/third_party/umd/` (third-party code — may need upstream fix).

## GTEST_SKIP Misuse Audit

### Fixed in this batch:
- **FIX TC** addresses the primary GTEST_SKIP propagation bug: `GTEST_SKIP()` from a helper called in the test body (not from SetUp) only returns from the helper — the test body continues. FIX TC moves the setup call into `T3kCustomMeshGraphFabric2DFixture::SetUp()` (GTest hook) so the skip propagates.
- **FIX QE** returns `bool` from the degraded-check helper instead of calling `GTEST_SKIP()` inside it — the caller in `SetUp()` calls `GTEST_SKIP()` based on the return value.
