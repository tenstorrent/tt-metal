# Audit: nsexton/0-racecondition-hunt — 2026-05-09 09:50Z

## Context
Task 1's most recent commit (5466be3482b — FIX DT-1) extended four WARM_RING_TIMEOUT grep
patterns in run_t3000_unit_tests.sh to also match dispatch-ERISC markers:
- "Timeout (N ms) waiting for physical cores"
- "rescue of stuck dispatch cores"

This prevents stale go_msg=0x02 (left after rescue_stuck_dispatch_cores hard-resets ERISCs)
from propagating into GTest and causing channels_not_ready → AllGather SKIPPED.

## Gaps Found

### Gap 1: No [FIX DT-1] log marker in dispatch_kernel_initializer.cpp
**File**: `tt_metal/impl/device/firmware/dispatch_kernel_initializer.cpp:298-303`
**Problem**: The existing warning log ("Attempting rescue of stuck dispatch cores") does not
include a FIX DT-1 label. This makes it impossible to distinguish this scenario from unrelated
dispatch rescues when parsing CI logs.
**Impact**: analyze_fabric_hang_log.sh cannot count FIX DT-1 occurrences; root cause analysis
requires manual grep of "rescue_stuck" which conflates warm-up vs GTest rescues.

### Gap 2: analyze_fabric_hang_log.sh missing FIX DT-1 coverage
**File**: `scripts/analyze_fabric_hang_log.sh`
**Problems**:
1. "FIX DT-1" not in TIMELINE grep → events don't appear in timeline section.
2. "FIX DT-1" not in PHASES grep → events don't appear in phases section.
3. No `FIX_DT1_FIRES` counter → AGGREGATE COUNTERS can't show dispatch ERISC timeout count.
4. No output section explaining FIX DT-1 → no guidance when counter > 0.

### Gap 3: No targeted test for FIX DT-1 dispatch ERISC timeout → AllGather correctness
**File**: `tests/nightly/t3000/ccl/` — no test for this path.
**Problem**: No test verifies that after a dispatch-ERISC timeout in warm-up (rescue fires,
go_msg=0x02 stale), a subsequent AllGather session produces correct output.
