## Gaps Identified

### 1. analyze_fabric_hang_log.sh missing FIX AM detection
FIX AM log: `"…skipping Phase 5b. (FIX AM: #42429)"`
Impact: CI log analysis would not detect FIX AM firing, making triage harder.

### 2. analyze_fabric_hang_log.sh missing FIX AZ+ detection
FIX AZ+ log: `"teardown: FIX AZ+ — assert_cores threw for MMIO device…"`
FIX AZ+ is distinct from FIX AZ (non-MMIO). Without separate detection,
a t3k-05 CI failure from the MMIO path would look identical to non-MMIO path.

### 3. GAP-30 not registered in run_t3000_unit_tests.sh
test_gap30_fixal_started_early_exit_timing.py existed but was never added
to the test execution script. FIX AL regressions would not be caught.

### 4. No timing test for FIX AM (GAP-34)
FIX AL test (GAP-30) only checks that Phase 5 polling exits at 3000ms.
It does NOT test that Phase 5b is skipped (FIX AM).
A regression removing FIX AM would pass GAP-30 but cause ~8s extra overhead
on T3K quiesce, causing TestMeshWidthShardedCopy3D to time out (54s budget).
CI run 25048641877 was the exact scenario.
