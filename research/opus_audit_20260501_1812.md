### `c59d888dfb5` — FIX TK: guard DoSetUpTestSuite against chips not in fabric cluster

**File**: `tests/tt_metal/tt_fabric/common/fabric_fixture.hpp` (+44 lines)

**Root cause (CI run 25206124487, job 73907738880)**:
After progressive SIGKILL teardowns on T3K, all ETH links between MMIO chips go dead.
`SetFabricConfig(FABRIC_2D)` triggers topology discovery → mapper degrades to 1x1 (only chip 0).
`DoSetUpTestSuite` still collected all chip IDs [0..3] and called `create_unit_meshes({0,1,2,3})`.
`get_fabric_node_id_from_physical_chip_id(1)` TT_FATALs: chip 1 not in 1x1 cluster → SIGABRT.

**Fix mechanics**:
- After `SetFabricConfig()`, iterate ids through `control_plane.is_physical_chip_in_fabric_cluster(id)`
- Remove absent chips; if any removed: `cluster_degraded_skip_ = true`
- If all removed: return early (skip `create_unit_meshes`)
- Reset `cluster_degraded_skip_` in `DoTearDownTestSuite()`

**Test priority**: HIGH — crash in `SetUpTestSuite` kills entire fixture suite (3+ suites).
**Coverage**: **GAP-70 implemented** (`FixTkDegradedClusterChipFilterFixture`).

## Design Notes

**Why GAP-70/71 fork the testee to simulate the UNFIXED path**:
FIX TK and FIX TL are in `BaseFabricFixture::DoSetUpTestSuite()`. Since the fix is NOW in
place, calling `DoSetUpTestSuite()` directly would pass (fixed code returns early). To detect
regression (fix accidentally removed), the testee BYPASSES the fixture and calls the lower-level
APIs directly — simulating what the unfixed code did. SIGABRT from TT_FATAL = regression.

**Why GAP-71 uses UDM mode**:
FIX TL's regression is specifically triggered when `create_unit_meshes` is called with a
partial chip list in UDM mode. Regular FABRIC_2D mode with 1 chip might succeed; UDM mode
TT_FATALs because the tensix/UDM builder has a strict topology requirement. Using UDM mode
in GAP-71 makes the regression more reliably detectable.

**On GAP-72 deferral**:
FIX TM is in `tt_fabric_test_config.cpp::expand_one_or_all_to_all_unicast()`. The function
is only called during perf benchmark config expansion, not during the core ETH initialization
hang path. The guard is 4 lines; it's not on the critical code path; and CI would clearly
show the crash in perf benchmark logs if regressed. Low risk, high test complexity → defer.
