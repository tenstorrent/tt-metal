## Code correctness findings

**No critical correctness issues found.** The code is defensively programmed with appropriate guards:

1. **Thread safety**: `fabric_relay_path_broken_`, `fabric_channels_not_ready_for_traffic_`, and `fabric_teardown_timed_out_` are all `std::atomic<bool>` with documented thread-safety rationale. `fabric_pre_dead_channels_` is non-atomic but has a documented serialization invariant (configure always completes before quiesce).

2. **Exception safety in destructors**: `~MetalEnvImpl` wraps `teardown_fabric_objects()` and `cluster_.reset()` in try/catch blocks with separate handlers for `std::exception` and `...` (for UmdException). This prevents SIGABRT during teardown.

3. **Dead channel cascade prevention**: Dead channels propagate correctly through the pipeline: `terminate_stale_erisc_routers` -> `probe_dead_channels` -> `configure_fabric_cores` (skip soft reset + L1 clear) -> `Phase 5b` (skip health check). Each stage handles both pre-known and newly-discovered dead channels.

4. **FIX W heartbeat check**: The UMD topology discovery now reads `get_eth_heartbeat()` before relay attempts. The `(heartbeat >> 16) != 0xABCD` check is correct for identifying non-live firmware. The `continue` skip prevents the relay hang that our timeout catch cannot intercept.

5. **Off-by-one / null checks**: The `edm_status_name()` function handles all 15 EDMStatus enum values plus 2 sentinel values (0xDEAD5B5B, 0xDEADECE7). The `is_known_edm_status()` function properly delegates to `edm_status_name()`.

6. **Minor note**: `topology_mapper.cpp` changed the odd-shape filter from `||` to `&&` (line 1594). This is intentional per the comment — mixed shapes like 3x2 are now valid for 6-chip runners. This is correct behavior.

## 0x49706550 / dead channel fix status

The 0x49706550 ("iPeP") handling is comprehensive:

1. **Detection** (`terminate_stale_erisc_routers`): Reads `edm_status_address` via relay. Value 0x49706550 is recognized as base-UMD relay firmware sentinel. These channels are NOT added to `probe_dead_channels` (the relay is alive), but ARE tracked in `base_umd_channels` for FIX M soft-reset skip.

2. **Configure phase** (`configure_fabric_cores`): FIX M skips `assert_risc_reset_at_core()` for `skip_soft_reset_channels` (0x49706550 channels) — halting their BRISC would kill the ETH relay. L1 clear still runs for these channels.

3. **Teardown** (`teardown_fabric_config`): Polls for TERMINATED on all active ETH channels. Timed-out channels get force-reset (ALL RISCs per FIX AI). Clean-terminated channels get ERISC0-only reset to restart the heartbeat.

4. **UMD FIX W** (topology_discovery.cpp): Heartbeat check `(heartbeat >> 16) != 0xABCD` before relay — dead firmware channels are skipped entirely. This prevents the infinite hang that FIX AQ's catch block cannot intercept (because the relay write completes via PCIe but no ACK ever returns).

5. **0xDEADECE7 sentinel**: Used in Phase 5b when a relay read throws an exception. Properly recognized in `edm_status_name()` and in the Phase 5b health-check logic where channels with this value are classified as "all-dead" (triggering clean return path FIX W/AK).

6. **0xDEAD5B5B sentinel**: Used in Phase 5b when the per-iteration deadline is exceeded before a channel can be read. Also properly handled.
