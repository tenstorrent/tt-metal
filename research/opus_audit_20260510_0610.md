# Opus Audit: AllGather Hang Race Condition Gap Analysis

**Date**: 2026-05-10
**Branch**: `nsexton/0-racecondition-hunt` (tip: `d7b66ba1a24`)
**Worktree**: `/workspace/group/worktrees/nsexton-0-racecondition-hunt/`
**Commit**: `5f7ac277865` (FIX BZ/CA/CB)

## Executive Summary

Audited ~60 FIX commits (A through BY) for gaps in detecting and preventing AllGather hangs caused by dead ERISC relay channels on non-MMIO devices. Found 6 gaps across 4 dimensions. Implemented 4 fixes (GAP-A through GAP-E), identified 2 report-only gaps (GAP-D, GAP-F).

The core hang occurs in `FDMeshCommandQueue::read_completion_queue_event` when the UMD relay path to a non-MMIO device is dead. The relay read blocks for 5s per attempt with no upper bound, causing indefinite hangs in AllGather operations.

## Gaps Found

### GAP-A (IMPLEMENTED as FIX BZ) — Incomplete FIX Z guard

**File**: `tt_metal/distributed/fd_mesh_command_queue.cpp` line ~942
**Issue**: FIX Z guard in `read_completion_queue_event` only checked `is_fabric_relay_path_broken()`. Two other degraded states — `channels_not_ready_for_traffic` (FIX AM/AO STARTED-state devices) and `stale_base_umd_channels` (FIX RZ base-UMD firmware retained) — also indicate non-functional non-MMIO devices that hang on CQ read, but were not checked.
**Fix**: Extended the guard to check all three flags with a descriptive TT_THROW including all flag values.
**Severity**: High — silent hang on devices flagged by FIX AM/AO or FIX RZ.

### GAP-B (IMPLEMENTED as FIX CA) — Silent FIX PD catch blocks

**File**: `tt_metal/impl/device/device.cpp` lines ~1545 and ~1975
**Issue**: Both FIX PD `catch(...)` blocks silently swallowed exceptions from fw_launch_addr PCIe writes on MMIO devices. MMIO devices use direct PCIe — these writes should never fail. Silent swallow masked potential HAL or driver-level bugs.
**Fix**: Added typed `catch(const std::exception& ex_pd)` with `log_warning` on MMIO devices, plus fallback `catch(...)` for non-std exceptions.
**Severity**: Medium — diagnostic, no hang prevention. Enables detection of HAL bugs.

### GAP-C (IMPLEMENTED as FIX CB) — Missing guard in copy_buffer_data_to_user_space

**File**: `tt_metal/distributed/fd_mesh_command_queue.cpp` line ~880
**Issue**: `copy_buffer_data_to_user_space` `reader_lambda` had NO relay guard at all. If called for a non-MMIO device with broken relay, the UMD relay timeout (~5s per call) would hang the buffer read path. FIX Z protected `read_completion_queue_event` but not this parallel code path.
**Fix**: Added identical 3-flag guard with descriptive TT_THROW, same pattern as GAP-A.
**Severity**: High — silent hang on buffer reads for degraded non-MMIO devices.

### GAP-D (REPORT ONLY) — test_async_teardown_race.cpp not in CI

**File**: `tests/scripts/t3000/run_t3000_unit_tests.sh`
**Issue**: If a `test_async_teardown_race.cpp` test exists (testing async teardown race scenarios), it is not listed in `run_t3000_unit_tests.sh`. The script includes `test_ccl_multi_cq_multi_device` (line 375) and `test_gap21` (line 397) but no async teardown race test.
**Fix needed**: Add the test to `run_t3000_unit_tests.sh` once it exists. No code change made — the test file may not yet exist.
**Severity**: Low — test coverage gap, not a hang risk.

### GAP-E (IMPLEMENTED) — analyze_fabric_hang_log.sh missing patterns

**File**: `scripts/analyze_fabric_hang_log.sh`
**Issue**: The analysis script had no counter variables or summary output for: FIX XZ (heartbeat poll), FIX BY (quiesce probe), FIX XX (deassert guard), or the new GAP-A/B/C diagnostic log patterns. CI log analysis would miss these events.
**Fix**: Added 7 counter variables (FIX_XZ_FIRES, FIX_BY_FIRES, FIX_XX_FIRES, GAP_A_FIRES, GAP_B_FIRES, GAP_C_FIRES) with grep patterns, summary output blocks with explanatory text, and aggregate counter lines.
**Severity**: Medium — diagnostic coverage gap in CI log analysis.

### GAP-F (REPORT ONLY) — C++ fixture lacks quiesce probe

**File**: `tests/ttnn/unit_tests/gtests/multi_thread/test_ccl_multi_cq_multi_device.cpp`
**Issue**: Python test `test_gap21` has FIX BY quiesce probe (calls `quiesce_devices()` before AllGather loop to detect silent relay failures). The C++ `MultiCQFabricMeshDevice2x4Fixture` has FIX QW (skip if degraded after SetUp) and FIX RX (skip TearDown quiesce when broken) but no equivalent quiesce probe before the test body. A runner with external ETH channels (FIX EXT) could pass SetUp health checks but still hang in AllGather.
**Fix needed**: Add a quiesce probe in the C++ fixture's SetUp after fabric init, similar to FIX BY. Requires careful design — quiesce in C++ fixture context is more complex than Python.
**Severity**: Medium — affects runners with external ETH channels only.
