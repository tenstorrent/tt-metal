## Root Cause Analysis

### [1] Build wheel -103s faster on branch
Runner difference (branch: `z86dq-runner-4kwjc`, main: `z86dq-runner-4c8g5`). Normal variation for cloud build runners. No code change impact.

### [2] t3k_grok_tests -60s faster on branch
Both passed all tests. The 60s difference is infrastructure overhead (docker pull, git checkout). Different runners (branch: t3k-06, main: t3k-08). Test execution time was nearly identical. **No regression.**

### [3] t3k_tt_metal_multiprocess_tests — FAIL on both
Both runs failed with identical error: `IntermeshSplit2x2FabricFixture.MultiMeshEastMulticast_0` and Ethernet core timeout issues. **Pre-existing flake, not a branch regression.**

### [4] t3k_ttmetal_tests — branch 793s FASTER (1801s vs 2594s)
- **Main (t3k-01)**: Hit 2x ARC startup timeouts of 300s each (`Timed out after 300000 ms`) in `QuiesceSubmeshesAllowsAlternatingWorkloads/1` and `MeshTraceSweepTest2x4.Sweep/0`. These 600s of timeouts account for most of the difference.
- **Branch (t3k-06)**: The pre-flight fabric health check detected degraded remote devices and **skipped** 13 test suites instead of letting them hang for 5 minutes each. This is the health-check mitigation working as designed.
- Both failed — main with hard ARC timeouts, branch with `MeshSocketTest` failures (3 tests).
- Branch failed tests: `MeshSocketTest.SingleConnectionSingleDeviceSocketWithCBs`, `MeshSocketTest.SingleConnectionSingleDeviceSocketWithWorkersLoopAck`, `MeshSocketTest.MultiConnectionSingleDeviceSocketWithWorkersLoopAck`

### [5] t3k_ttnn_multiprocess_slow_tests — branch 451s SLOWER (735s vs 284s)
**Root cause: Infrastructure overhead, not test regression.** Detailed breakdown:

```
Phase                    Branch (t3k-03)    Main (t3k-09)    Diff
-----------------------------------------------------------------
Setup + Docker                 87s              117s         -30s
Git checkout                  341s               44s        +297s  <-- KEY
Build artifact download       156s               65s         +91s
Wheel download + install      105s               15s         +90s
Actual test execution          11s               10s          +1s
-----------------------------------------------------------------
Total overhead                707s              259s        +448s
```

The branch run on `t3k-03` had a 341-second git checkout (vs 44s on main). This suggests `t3k-03` has poor network connectivity to GitHub or a cold git cache. The artifact downloads were also 2-6x slower on that runner. **Test execution itself was identical (~11s, 1 test passed on both).**

### [6] t3k_ttnn_tests — FAIL on both (identical failures)
Both crashed with exit code 139 (SIGSEGV) after identical `DistributedTensorOpIfTest` failures:
- `AllGatherWithShardedTopology`
- `ReduceScatterWithShardedTopology`
- `AllReduceWithShardedTopology`
- `AllBroadcastWithShardedTopology`
- `BroadcastWithShardedTopology`
- `FusedRmsMinimalWithShardedTopology`

All hit `TT_FATAL @ fabric.cpp:163: link_idx < candidate_eth_chans.size()`. **Pre-existing failure on main, not a branch regression.**

### [7] t3k_ttnn_udm_tests — branch 206s FASTER (326s vs 532s)
- **Branch (t3k-12)**: Fabric health check detected degraded mesh — all 60 UDM tests were **skipped**. 0 tests passed.
- **Main (t3k-05)**: All 60 tests ran and passed normally.
- The "speedup" is because the branch skipped all tests. **This is the health-check mitigation avoiding hangs on a degraded runner.**

**None.** Every job that failed on the branch also failed on main (or had identical pre-existing failures).

## Summary

1. **No branch-introduced regressions.** All failures exist on main.
2. **t3k_ttnn_multiprocess_slow_tests +451s** is entirely infrastructure overhead — runner `t3k-03` had a 341s git checkout and slow artifact downloads. Test execution was identical.
3. **t3k_ttmetal_tests -793s** is the health-check mitigation working correctly: branch skips degraded suites fast instead of hitting 300s ARC timeouts.
4. **t3k_ttnn_udm_tests -206s** is also the health-check mitigation: branch skipped all 60 tests on degraded `t3k-12`.
5. The `t3k_ttnn_tests` SIGSEGV crash (`fabric.cpp:163: link_idx < candidate_eth_chans.size()`) exists on both branch and main — pre-existing bug.
6. Runner `t3k-03` needs investigation for poor GitHub/artifact network performance.
