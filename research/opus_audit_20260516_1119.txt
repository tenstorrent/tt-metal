CODE CORRECTNESS ISSUES
=======================

1. [FIXED] FIX DI-B — Missing PCIe read-barrier in FIX DI retry path
   - File: tt_metal/fabric/fabric_init.cpp (lines 442-454)
   - Issue: FIX DI writes fw_launch_addr_value via write_core_immediate (posted
     PCIe TLP) then immediately calls deassert_risc_reset_at_core (also posted).
     No read-barrier between them — same race condition that FIX DG addresses
     in the main FIX AC path.
   - Impact: Deassert can arrive before fw_launch_addr write. ERISC ROM reads
     fw_launch_addr=0, enters link-training at 0x49705180, channel stays stuck.
   - Fix: Added read_reg barrier matching FIX DG pattern. Logs warning on mismatch.
   - Risk: LOW — this is MMIO-direct PCIe where read_reg is always available.
   - Note: FIX RR-NM step 2 (fabric_firmware_initializer.cpp) uses write_only
     variants for non-MMIO relay where read_reg is impossible — no fix needed.
