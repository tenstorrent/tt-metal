## Primary Finding: FIX TG Root Cause

The `bad optional access` in CI run 25197685800 was traced to:

```
control_plane.cpp — configure_routing_tables_for_fabric_ethernet_channels()
  auto host_rank_for_chip = topology_mapper_->get_host_rank_for_chip(mesh_id, chip_id);
  TT_ASSERT(host_rank_for_chip.has_value(), ...);  // NO-OP IN RELEASE BUILDS!
  auto connected_host_rank_id = host_rank_for_chip.value();  // throws std::bad_optional_access
```

**Why `TT_ASSERT` is a no-op in Release**: `tt_stl/tt_stl/assert.hpp` defines:
```cpp
#else  // not DEBUG
#define TT_ASSERT(condition, ...) \
    do { (void)(condition); } while (0)  // no-op — avoids unused var warnings in Release
#endif
```

**Why `host_rank_for_chip` is nullopt**: On a degraded T3K cluster where FIX TB has excluded
some chips from the topology mapper (corrupt ERISC L1), `get_host_rank_for_chip()` returns
`std::nullopt` for excluded chips. The `else` branch in `configure_routing_tables` reaches
this path when a connected chip falls outside `local_mesh_coord_range` (cross-host routing
path, entered for inter-mesh connections in the custom mesh graph descriptor).

**Evidence chain from CI log** (run 25197685800):
1. HEAD at `2f828dc16d9` (FIX TF deployed) — confirmed at log line 419
2. FIX AE marks chips 4-7 relay broken
3. FIX TB excludes chips 4-7 from topology mapper (corrupt ERISC L1)
4. ControlPlane ctor calls `configure_routing_tables_for_fabric_ethernet_channels`
5. For chip in mesh 0 connected to excluded chip in mesh 1: `get_host_rank_for_chip` → nullopt
6. `TT_ASSERT` (no-op) → `.value()` throws `std::bad_optional_access`
7. GTest catches: "bad optional access" thrown in SetUp() (line 2713)

**Fix (commit `5dbd5423f41`)**:
```cpp
// FIX TG (#42429): TT_ASSERT is a no-op in Release builds
if (!host_rank_for_chip.has_value()) {
    log_warning(
        tt::LogFabric,
        "FIX TG (#42429): Mesh {} Chip {} has no host rank in topology mapper "
        "(chip excluded by degraded-topology guard). "
        "Skipping cross-host routing table entry from FabricNodeId (M{}, D{}).",
        *mesh_id, logical_connected_chip_id, *mesh_id, fabric_chip_id);
    continue;
}
auto connected_host_rank_id = host_rank_for_chip.value();
```

**Impact**: Converts opaque `bad_optional_access` thrown during ControlPlane ctor into
a skipped routing entry with a clear warning. Matches FIX TE pattern for excluded chips.
Session may still fail (no routes to degraded chips) but with a clear diagnostic message.

## Key Insight: TT_ASSERT vs TT_FATAL

Any use of `TT_ASSERT` as a guard before a `.value()` call in Release-built code is a latent
bug of this class. A systematic audit of `TT_ASSERT` + `.value()` patterns in the fabric
initialization path would be a valuable hardening sweep.

Known safe locations (TT_FATAL used correctly):
- `relay_mux.hpp:152` — FIX TF
- `control_plane.cpp:1207` — **FIX TG** (this audit)
- `fabric.cpp:155,165,210,438` — all guarded
- `compute_mesh_router_builder.cpp:711,850` — all guarded
