## Primary Finding: FIX TF Root Cause

The `T3kCustomMeshGraphFabric2DFixture.TestUnicastRaw/0` CI failure (run `25195921333`, job `73878392973`) was traced to:

```
relay_mux.hpp:154 — assemble_2d_fabric_packet_header_args()
  forwarding_direction.value()   // unconditional — throws std::bad_optional_access
                                  // when inter-mesh relay broken (chips 4-7 degraded)
```

**Evidence chain from CI log**:
1. ERISC L1 corrupt on 28+ channels at session open (edm_status=0x49705180)
2. "Could not find a route between (M0, D0) and (M1, D3)" at startup
3. FIX AE marks chips 4-7 relay broken → control plane has no inter-mesh routes
4. `assemble_2d_fabric_packet_header_args()` called for cross-mesh pair → `get_forwarding_direction()` returns `nullopt` → `.value()` throws

**Fix (commit `2f828dc16d9`)**:
```cpp
TT_FATAL(
    forwarding_direction.has_value(),
    "FIX TF: No forwarding direction from physical chip {} (fabric node {}) to "
    "physical chip {} (fabric node {}). Inter-mesh relay path is broken.",
    my_device_id, src_fabric_node_id, destination_device_id, dst_fabric_node_id);
```

**Impact**: Converts opaque `bad_optional_access` into a `TT_FATAL` with full chip/mesh context. Session still fails when relay is broken (no route exists), but the error is now diagnostic instead of cryptic.
