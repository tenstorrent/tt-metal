## Recommended Execution Order

### Phase A: Validate pipeline (pick 1-2 from Tier 1)

Start with the simplest kernels to prove codegen works end-to-end:
```
Generate abs for Quasar
Generate negative for Quasar
Generate fill for Quasar
```
These are trivial SFPU ops. If codegen can't handle these, fix the pipeline before scaling.

### Phase B: Complete Tier 1 (remaining simple SFPU)

All similar patterns — batch these. After Phase A confirms the pipeline is solid, these should be straightforward.

### Phase C: Tier 2 medium SFPU (14 kernels)

Some depend on existing SFPU (elu→exp, tanh_derivative→tanh). All dependencies are already on Quasar so no ordering constraint.

### Phase D: Tier 3 complex SFPU (12 kernels)

Integer ops, binary ops, conditional ops. More architectural discovery needed.

### Phase E: Tier 4 specialized SFPU (6 kernels)

Unusual patterns — welfords, reduce, pooling. Most complex SFPU work.

### Phase F: Tier 5 LLK submodule — core (10 files)

Math SFPU wrappers depend on SFPU kernels from earlier phases. Pack/unpack are independent.

### Phase G: Tier 6 LLK submodule — experimental (13 files)

Low priority. Assess need based on tt-metal integration requirements.

## Notes

### 1. Existing implementations are available as style examples
With 16 SFPU kernels already on Quasar, agents have on-target style references. This was the main concern before — now solved.

### 2. SFPU kernels are standalone
Each SFPU kernel is a single header file with no compilation dependencies on other SFPU kernels (they're dispatched at a higher level). Safe to generate in any order.

### 3. Math SFPU wrappers #include SFPU kernels
Tier 5 math wrappers (llk_math_eltwise_binary_sfpu.h etc.) will #include the SFPU kernels. Generate SFPU kernels before their math wrappers.

### 4. Params files are thin
The `_params.h` files are typically just parameter struct definitions with minimal logic. They're paired with their main file and can be generated together.

### 5. llk_math_eltwise_unary_sfpu is done (renamed)
The old plan listed this as missing. It exists on Quasar as `llk_math_eltwise_unary_sfpu_common.h`. Only the `_params.h` counterpart is still needed.

### 6. Experimental files are low priority
The 13 experimental files in `llk_lib/experimental/` on Blackhole are custom/specialized variants. They are unlikely to be needed for initial Quasar bring-up and can be deferred.

### 7. API files live in tt-metal
Intermediate API and Compute API files are in the tt-metal repository, not tt-llk. They are tracked in the Excel but cannot be generated from this repo. See the dashboard Coverage tab for details.
