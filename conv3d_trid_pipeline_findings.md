# Conv3D gather trid-ring autotune — performance report

Production change introducing an intensity-and-burst-gated NoC trid ring on
`gather_rows_to_shard` DRAM reads in conv3d.  See `conv3d_trid_pipeline_findings.md`
for design rationale and alternatives explored.

**Branch:** `pjosipovic/conv3d-gather-trid-autotune` (2 commits on top of `main`).

**Methodology:**
- Test: `tests/ttnn/unit_tests/operations/conv/test_conv3d.py::test_conv3d_wan_vae_profile` (94 WAN-VAE shapes).
- Metric: `DEVICE KERNEL DURATION [ns]` for `Conv3dDeviceOperation` from tracy CSV.
- Each value is the **median of 3 independent test runs** (single-trial sweeps had ±5–10% noise).
- Baseline = original code (no ring): kernel takes the issue-all + single-trailing-barrier path,
  identical to `main` HEAD before this branch.  Implemented in the kernel by setting
  `gather_trids = 0`, which constexpr-elides all ring code.
- New = same kernel binary; the program factory's classifier
  `(intensity ≥ 128 B/tile) AND (inner_burst ≥ 8)` selects which shapes get `gather_trids = 8`.
