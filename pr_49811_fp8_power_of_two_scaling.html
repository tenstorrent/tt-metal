Device packer

Truncates

Values from 448 up to—but not including—480 encode as 0x7E. At 480 the dropped mantissa reaches the 0x7F bucket.

Host encoder

Rounds + saturates

Finite overflow and infinities saturate to ±0x7E. A NaN input remains the canonical 0x7F NaN.

Host decoder

Extends exponent 15

0x78 through 0x7E decode as finite. Only mantissa 7 is treated as NaN.

E4M3FN contractRepresentable max 448 · packer’s next bucket begins at 480
Do not confuse maximum representable value with conversion threshold. 448 is the largest finite code point. Because the Blackhole packer truncates three mantissa bits, values in [448, 480) still land on 448; 480 is the first input whose top three mantissa bits form 111.

What this PR actually emits

The scale tensor remains FP32. Its values are constrained to powers of two; they are not packed into one-byte UE8M0.

Binary exactness

Multiplying by a normal power of two changes the exponent without changing significand bits, barring range limits.

Consumer contract

Sparse MLA and FlashMLA can rely on exponent-grid scales consistent with their expected format.

Predictability

Every emitted scale is an exact FP32 power of two. Device reduction near a threshold can select an adjacent power while remaining safe.

Rounding detail

The operation documentation notes that the Blackhole E4M3 packer truncates the mantissa. Results may differ from a round-to-nearest reference by up to one E4M3 ULP.

448 is still the maximum finite decoded value. The 480 figure is only the Blackhole truncating packer’s first input bucket that produces byte 0x7F; it is not another representable finite value.

Zeros and tiny blocks

The 10−4 clamp is part of the format contract

a = max(amax, 0.0001)
raw = a / 448
s = 2ceil(log2(raw))

For an all-zero block, the chosen scale is 2−22 ≈ 2.384×10−7, while every quantized value remains exactly zero.

  • Prevents division by zero when computing 1/scale.
  • Provides a valid positive input to the exponent-based power-of-two rounding helper.
  • Applies the lower bound before multiplying by the fixed 1/448 constant.
  • Keeps scale metadata finite and deterministic for zero-filled cache regions.

Host-side correction

The bug was one reserved-exponent assumption

Before
// Treated all E=15 encodings as NaN FP8_EXP_MAX = 14 if (exp == 15) return NaN overflow → 0x77 // 240
Now
// E4M3FN extends E=15 through M=6 FP8_EXP_MAX = 15 if (exp == 15 && man == 7) return NaN overflow → 0x7E // 448

Encode

RNE carry from 240 can enter exponent 15; finite values that would land on NaN saturate to 0x7E.

Decode

Bytes 0x78–0x7E and 0xF8–0xFE now decode as finite positive and negative values.

Three CPU-only tests cover exponent-15 encoding, saturation/NaN, and decoding.
