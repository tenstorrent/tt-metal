Log Probs Update Report
=======================

Summary of Changes
------------------

The log_probs behavior has been updated so that LogProbsCalculator.calculate_log_probs()
returns None instead of a device tensor (output_tensor) when:
  - Log probs are not requested (enable_log_probs=False)
  - The device count is not 8 or 32 (unsupported configuration)
  - The number of devices for sharding is less than 2

Previously, a tensor of ones was returned in all cases, wasting device-to-host bandwidth
and making it impossible for consumers to distinguish "not computed" from actual values.

Files Changed
-------------

1. models/common/sampling/tt_log_probs.py
   - calculate_log_probs() now returns None for disabled/unsupported cases
   - Added explicit num_devices check for 8 or 32

2. models/tt_transformers/tt/generator.py
   - read_decode_output(): Added None check in async path for log_probs
   - process_decode_output_host(): Returns None for log_probs when not available
     (instead of creating torch.ones dummy tensor)

3. models/demos/llama3_70b_galaxy/tt/generator.py
   - process_decode_output_host(): Returns None for log_probs when not available
     (instead of creating torch.ones dummy tensor)

4. /localdev/divanovic/vllm/vllm/v1/worker/tt_model_runner.py
   - _get_output_tokens(): Removed assert for tt_log_probs is not None
   - Now gracefully handles None tt_log_probs when logprobs were requested
     but device doesn't support them (appends None to logprobs_per_dp)

5. models/common/tests/test_sampling.py
   - Added test_log_probs_returns_none_when_disabled: Verifies that
     calculate_log_probs returns None when enable_log_probs is False,
     and returns a tensor when enabled on supported devices.
