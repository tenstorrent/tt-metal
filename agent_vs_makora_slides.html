Where did Makora's wins come from?

Same mechanism (fusion vs TTNN composite) — two outcomes depending on whether the composite still exists today

Bucket A — Composite still exists (claims hold)

TTNN baseline is an N-op composite; each step writes to DRAM. Fusion collapses N programs into 1, eliminating N-1 DRAM round-trips. TTNN has not (yet) replaced the composite.

OpMakora claimMeasured today

multigammaln: TTNN composite calls a 6-term Lanczos polynomial 4× (~150 programs). multigammaln@main:196 _lgamma@main:32
glu: TTNN composite calls 2× ttnn.slice + sigmoid + multiply. Slices are real DRAM copies; reader-level split eliminates them. glu@main:338

Bucket B — TTNN replaced the composite (claims inverted)

Same fusion mechanism as bucket A at Makora's benchmark time — TTNN was a multi-program composite chain. TTNN then replaced these composites with single-program SFPU primitives in March–April 2026. The primitives are also algorithmically better than Makora's general kernel.

atan2 (2026-04-07): commit message: "atan2 was composite and would benefit from a dedicated SFPU implementation." Replaced an 11-op chain (reciprocal + multiply + atan + 8 quadrant-fixup ops). commit aed2421
lgamma (2026-03): replaced the same _lgamma Lanczos chain that still serves multigammaln. Makora ships that chain fused; TTNN now ships a Stirling SFPU primitive. commit 41c953b (bf16) commit 22ad2b3 (fp32)

After attribution, roughly half of Makora's published wins evaporate or invert on current TTNN main — bucket A's fusion math is durable; bucket B's algorithmic gap closed without anyone replicating Makora's kernel approach.

Two agents, same problem

Bucket A is the question we can ask: can our pipeline match Makora's fused kernels?

  • Makora — external set of 19 AI-generated fused TTNN kernels, each produced by an unknown agent process. Published per-op benchmarks against TTNN composites and chains.
  • Our pipeline — incremental agent flow: planner → implementer → verifier → refinements. Generates a complete TTNN op (Python entry point + program descriptor + reader / compute / writer kernels) from a single prompt, with tests and refinement-tracking docs as a byproduct.

Load-bearing structural choices discovered independently from the prompt

  • glu_fused: invented the "tile-pair reader" (read tile(t) and tile(t + Wt/2) from one tensor, no slice op) from the prompt constraint "no ttnn::slice allowed". glu_fused_reader.cpp
  • multigammaln_lanczos: independent register-pressure trick — spill DST[1] to a CB, run the inner multiply, reload — to keep the Lanczos accumulator pinned in DST[0] across iterations. multigammaln_lanczos_compute.cpp
  • atan_mean: one parameterised kernel covers both of Makora's shape-specialised variants (tall + high_channel) — the split had no structural justification. atan_mean_compute.cpp
  • Agent builds on the kernel_lib helper layer (sfpu_atan, reduce<AVG, REDUCE_ROW>, calculate_and_prepare_reduce_scaler). Makora hand-rolls raw LLK calls.

Takeaway

  • Two agent stacks at parity. Across four compute-kernel head-to-heads, Makora is at most 3% faster; the agent is 7% faster on one of them.
  • End-to-end cost per op: one working session. Planner ~12 min, implementer ~9 min, verifier ~5 min. Refinements 5–10 min each. Tests, capability docs, and refinement checklist are byproducts.
  • The compounding wins are below the kernel. The agent's edge comes from the kernel_lib helpers absorbing complexity, and from a prompt + refinement protocol that captures hardware constraints precisely. The kernel-level work is straightforward; the leverage is in the layers around it.
  • Helpers are an LLM-friendly surface. Writing kernels through the kernel_lib helper layer is materially easier for the agent to get right than hand-rolling raw LLK calls. backward_softmax (slide 4) — taken through Phase 0 + four refinements for $60.70 / ~2h of agent time — lands at gmean 1.02× of Makora's hand-tuned bf16 kernel on compute kernel duration.

Bucket B (TTNN-primitive baseline) was not part of this comparison — TTNN's own primitives were uplifted in March–April 2026 and now beat Makora 2–4× without any agent involvement; those kernels are obsolete.
